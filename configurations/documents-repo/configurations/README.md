# Config Templates

产品配置模板仓库，按语言、产品、案例分层组织配置文件与文档。

## 目录结构

```
config-templates/
├── zh/                              # 语言：中文
│   ├── ER805/                       # 产品型号
│   │   ├── 案例A/                   # 案例名称
│   │   │   ├── 案例手册.md          # 案例说明文档
│   │   │   ├── 配置手册.md          # 配置操作指南
│   │   │   ├── config/             # 配置文件目录
│   │   │   │   ├── config.json
│   │   │   │   └── ...
│   │   │   └── assets/             # 相关资源（图片等）
│   │   └── 案例B/
│   │       └── ...
│   └── ER806/
│       └── ...
├── en/                              # Language: English
│   ├── ER805/
│   │   ├── CaseA/
│   │   │   ├── case-manual.md       # Case manual
│   │   │   ├── config-manual.md     # Configuration guide
│   │   │   ├── config/
│   │   │   │   └── ...
│   │   │   └── assets/
│   │   └── CaseB/
│   │       └── ...
│   └── ER806/
│       └── ...
└── README.md
```

## 层级说明

| 层级 | 说明 | 示例 |
|------|------|------|
| 第 1 层 | 语言代码 | `zh`、`en` |
| 第 2 层 | 产品型号 | `ER805`、`ER806` |
| 第 3 层 | 案例名称 | 以业务场景或客户需求命名 |
| 第 4 层 | 具体文件 | 案例手册、配置手册、配置文件等 |

## 命名规范

- **语言目录**：使用 ISO 639-1 语言代码小写，如 `zh`、`en`、`ja`
- **产品目录**：使用产品型号，如 `ER805`
- **案例目录**：使用简洁明了的案例名称
- **配置文件**：放在案例目录下的 `config/` 子目录中
- **资源文件**：图片等放在案例目录下的 `assets/` 子目录中

## 贡献指南

1. 新增案例时，请按上述目录结构创建对应的文件夹和文件
2. 每个案例至少包含案例手册和配置文件
3. 配置手册为可选，但建议提供以方便使用者理解配置内容
