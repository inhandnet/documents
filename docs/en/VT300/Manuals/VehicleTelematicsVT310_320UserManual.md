## Chapter I Product Introduction and Preparation

## 1. Introduction

### 1.1 Overview

The VT300/320 series vehicle tracking gateway is an asset tracking product that features cost-effectiveness, rich interfaces and strong performance. It is suitable for industries such as logistics and transportation, engineering vehicle monitoring and so on. It offers precise positioning with GNSS, tracking and monitoring the status, history track, geofencing, abnormity alarm and other functions of vehicles and drivers, combined with the vehicle network cloud platform, can realize remote vehicle management, asset tracking, preventive maintenance, helping fleet operators save costs and improve efficiency. The device provides sub-models that support wireless network access of various speeds such as LTE CatM1, Cat1, Cat4, etc.

## 2. SIM and Cable Installation

For general cases, the device can be used after mounted onto the vehicle, with SIM card inserted and cable installed.

### 2.1 Install SIM Card

In the case of dial-up Internet access, a SIM card is needed. The VT will dial automatically once it is powered and started. Open the waterproof baffle on the downside of the VT and insert the SIM card into the slot in the direction shown in the picture.

Customers can fix the VT onto the vehicle with installation bolts. It is recommended that the tracker be installed under the front windshield of the vehicle, where GPS signal is better received and connection to the OBD-II diagnostic interface is easier.

### 2.4.2 RS232 Serial Port

The RS232 serial port is used for debugging. Connect the RS232_RX, RS232_TX, and GND of the VT to TXD, RXD, and GND of the DB-9 serial port. Use RS232 to USB cable to connect with DB-9 serial port.

### 2.4.3 Digital Input (DI)

The DI can detect the switching value, such as whether the button is pressed or bounced, and whether the switch is on or off. The VT provides configurable pull-up. The DI has a default 10kΩ resistor pulled down to GND. When the DI is configured to pull up, there is a 20kΩ resistor pull up to the power supply voltage. When using DI, it is necessary to distinguish between pull-up and no pull-up.

### 2.4.6. 1-Wire

The 1-Wire is usually used for small communication equipment, such as digital thermometers and iButton devices. Before use, the customer needs to connect the DQ pin (signal line) of the 1-Wire device to the VT PIN8, and connect the VDD and GND pins of the 1-Wire device to the GND of the VT. The sensor is the less02b type. The following picture shows the water temperature detection wires of the 32 digital temperature sensor probe.

### 2.4.7 Ignition Sense

IGT(Ignition sense): IGT is used to connect to the Ignition switch of the vehicle. When IGT enable, the VT/VT320 can detect whether the connected vehicle is ignited. When using the 20PIN cable for testing, connect the IGT cable and V+ cables to DC power supply.

## 3. Start the VT300/200

After the customer completes the installation according to the above steps, the device can be started for debugging. The condition of the device can be told through the status indicator. To avoid consumption of battery power during transportation, the device is under transportation mode in the factory state, so the VT battery needs to be activated by external power supply or the vehicle diagnostic interface and the VT320 battery need to open battery switch to activate.

### 3.1 GNSS Status Light

| Indicator Status | Function status |

## Status

Basic device information and operating status of the device.

### Summary

The summary page displays the basic information and status information of InVehicle Telematics equipment, including: device model, serial number, firmware version, Bluetooth version, device time, battery level, network connection status and network signal value. At present, the configuration tool supports connected InVehicle Telematics including VT300 series, VT200 series; Serial number is set when the equipment is delivered, as the unique identification of the equipment; Firmware version for the current device firmware version number, available from [Website](https://www.inhand.com/en/resources-center/#/Mobility/InVehicleTelematicsGateways/VT300) The latest firmware version is recommended. The Bluetooth version is the Bluetooth firmware version of the device. If the device model does not support Bluetooth, it is unknown. The battery power is the remaining power of the current device. If no battery is installed, it will be displayed as unknown.

## System Settings

Set sleep mode and wake up working modes of InVehicle Telematics. Set Wakeup interval, Wakeup Running time and Sleep Delay time to meet different low-power operation modes, which can ensure normal and stable operation of basic functions of the system and save power.

You need to enter the correct user name and password to log in to the device through this tool. You can change the user name and password through the account setting.

### Sleep Mode

The sleep mode ensures the battery life after flameout, providing continuous guarantee for special environments. The state machine is as follows:

![1681197836972-0bb9ca70-0855-452b-9004-47539a3a3ef2.png](./img/JTyXJeajJoIDZj8-/1681197836972-0bb9ca70-0855-452b-9004-47539a3a3ef2-794073.png)Description of the state machine:

Run, Sleep, and Temp run represent normal running status, sleep status, and temporary running status respectively.

① Corresponding to the state machine, the condition from Run to Temp run is that the power supply voltage is less than sleep voltage (6V by default) or IGT OFF (IGT needs to be enabled in the configuration), by default, the device continues to run for 15S temp (for reporting information) and then enters Sleep;

② Corresponding to the state machine, the condition of entering Sleep from the Temp run is that after the device runs a wake-up runtime cycle in the Temp run or after the device runs Temp Run for 15s from run;

③ Corresponding to the state machine, the condition from Sleep to Run is that the power supply voltage is greater than Sleep voltage or IGT ON (IGT needs to be enabled in configuration);

④ Corresponding to the state machine, the condition of entering the Temp run from Sleep is that after the device runs a wake-up interval in Sleep;

⑤ Corresponding to the state machine, the condition from Temp run to Run is that the power supply voltage is greater than sleep voltage or IGT ON (IGT needs to be enabled in configuration);

Configure the sleep mode:

| Parameter | Description |
| :--- | :--- |
| Enable IGT | After IGT is enabled, the device uses the IGT status as the condition for entering or exiting Sleep. The IGT status is not ticked by default. |
| Wake-up interval | The interval between the device automatically wakes up in Sleep, whose default value is 120 minutes. When the value is 0, device will do not enter sleep mode. |
| Wake-up time | The interval between the time when the device enters the next Sleep, whose default value is 5 minutes. |
| Sleep Delay | Wait time for enter sleep. |

## Restoration of the Default Account and Password for Hardware

Because configuration usually involves the device certificate file, when the device is restored to the factory via hardware, only the username and password are restored to default. As is shown in the following picture, press the Reset button with a screwdriver or other tools for more than 8 seconds, and then loosen it.

![1681197862498-0cabdce9-cd55-40a0-b5d5-a3b0b288df77.png](./img/JTyXJeajJoIDZj8-/1681197862498-0cabdce9-cd55-40a0-b5d5-a3b0b288df77-032849.png)

ps: By double-clicking "Reset", you can restart the device when it goes wrong.

## How to Get the Device Log

Make sure that the computer is connected to the VT through USB to serial port through configuration wire, and open a serial port connection tool such as the serial port debugging software. The software can be downloaded in Mircrosoft Store.

![1681197863045-d7770b2f-bafa-40af-b6b9-09d6014eed90.png](./img/JTyXJeajJoIDZj8-/1681197863045-d7770b2f-bafa-40af-b6b9-09d6014eed90-394928.png)

1. Open the serial port debugging software and select the link serial port. The default baud rate of the serial port is 115200/8/n/1. Click "Open serial port". Note that the Character encoding mode (Character encoding) is ASCII, and the line break mode (Linet break) is \n(LF).

![1681197863567-88c921ae-79ad-415c-a4cc-2608ec0c3194.png](./img/JTyXJeajJoIDZj8-/1681197863567-88c921ae-79ad-415c-a4cc-2608ec0c3194-944967.png)

1. Enter +++ in the content sending serial port or not to activate the CLI mode, as is shown below;

![1681197864193-54a96dbe-a207-4cc9-aced-2add9c67beae.png](./img/JTyXJeajJoIDZj8-/1681197864193-54a96dbe-a207-4cc9-aced-2add9c67beae-907845.png)

1. Enter the Username (press the enter key), click "Send", enter the password (press the enter key), and click send to enter the command line mode.

![1733364725856-9df7f5af-e8a9-457c-aca8-93517feda7ac.png](./img/JTyXJeajJoIDZj8-/1733364725856-9df7f5af-e8a9-457c-aca8-93517feda7ac-355844.png)

![1733364783447-a80c3529-ff1b-49dd-b7e1-75404d328449.png](./img/JTyXJeajJoIDZj8-/1733364783447-a80c3529-ff1b-49dd-b7e1-75404d328449-986390.png)

1. Enable the log function. In the send text box, enter “log console enable” (press the enter key) and click "Send". The following screenshot shows the log information in the receive window.

![1681197865696-2aba261a-9ef1-42b8-a1af-5e3a0f916165.png](./img/JTyXJeajJoIDZj8-/1681197865696-2aba261a-9ef1-42b8-a1af-5e3a0f916165-765968.png)

1. Close log function, write “log console disable” (press the enter key) in the send text box and click "Send". The receive window stops receiving logs.

![1681197866054-677d6376-0c63-4f4b-ad87-82532ebf4a59.png](./img/JTyXJeajJoIDZj8-/1681197866054-677d6376-0c63-4f4b-ad87-82532ebf4a59-297166.png)

1. If you need to link the configuration tool after exiting the serial port, write “exit” (press the enter key) in the send text box, click "Send" (used to exit the CLI mode), and then close the serial port. Or you wait for 180 seconds when the device automatically exits the CLI mode.
