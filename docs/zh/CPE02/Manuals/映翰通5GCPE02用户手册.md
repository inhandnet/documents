### 1.1 概述

CPE02是北京映翰通网络技术股份有限公司推出的新一代5G简单联网路由器，专为便捷联网场景设计，具备1Gbps转发能力和3000Mbps Wi-Fi 6覆盖。设备支持即插即用，无需布线，通过小星云平台实时监控门店网络状态、流量使用和客户端连接，提供实时告警，大幅降低IT管理难度，助力企业轻松实现高效网络部署与管理。

## 第2章 安装与首次使用

### 2.1 安装前准备

| 项目 | 要求 |
|------|------|
| 电源 | 需使用原装电源适配器 |
| 网线 | 标准RJ45网线 |
| PC | 具备以太网接口，浏览器推荐Chrome |
| 环境 | 温度、湿度需符合设备规格要求 |

> **注意**：安装设备时请勿将设备置于强电磁干扰环境下，避免与高功率设备过于接近。安装完成后需确认设备是否易发生晃动，避免因设备跌落造成损坏。

### 2.2 安装指南

#### 2.2.1 电源连接与网络接线

1. 将设备接入电源。
2. 用网线将设备任意LAN口连接至PC。

#### 2.2.2 PC网络配置

1. 设备默认开启DHCP服务，将PC设置为"自动获得IP地址"后，设备将为PC自动分配地址。需确认PC获取到的IP地址处于192.168.2.1网段。
2. 若PC无法自动获取IP地址，需手动将PC的IP地址配置为192.168.2.2~192.168.2.254中的任意地址，默认网关设置为192.168.2.1，子网掩码为255.255.255.0，DNS配置为223.5.5.5或运营商DNS服务器地址。

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1747623633208-5397452e-dc51-41e4-86c4-75e514a8b389-925298.webp" alt="PC手动配置/自动获取IP地址"></p>

<p align="center"><strong>图 2-1 PC手动配置/自动获取IP地址</strong></p>

#### 2.2.3 Web界面登录

1. 在浏览器地址栏中输入设备默认地址 `192.168.2.1`。
2. 输入用户名和密码（参考铭牌信息），进入设备Web管理界面。
3. 若页面提示网页不安全，打开隐藏或高级选项，选择"继续前往"。

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1750155138913-48b4b6f6-9386-4ca7-a2d2-878f8ecc3fe8-270022.webp" alt="登录设备管理页面"></p>

<p align="center"><strong>图 2-2 登录设备管理页面</strong></p>

### 2.3 快速检查

- [ ] 设备已正确上电，指示灯正常亮起
- [ ] PC与设备LAN口已通过网线连接
- [ ] PC已获取到192.168.2.x网段的IP地址
- [ ] 可通过浏览器访问 `192.168.2.1` 并正常显示登录页面
- [ ] 可使用铭牌上的用户名和密码成功登录

## 第3章 常用场景配置

### 场景1：蜂窝联网

**目标**：通过4G/5G蜂窝网络接入互联网。

**前提**：已插入SIM卡并安装天线，设备已上电。

**预计用时**：约5分钟。

**操作步骤**：
1. 将SIM卡正确插入卡槽，并安装蜂窝天线。
2. 设备启动完成后，通常可自动蜂窝拨号，无需额外配置。
3. 若为专网APN用户，进入【Internet】→【Cellular】菜单，配置对应的APN账号密码。
4. 点击"保存"并等待连接建立。

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1750928941501-a12bd9d0-eda4-4462-be46-1eea8120584b-729666.webp" alt="设备蜂窝接口配置"></p>

<p align="center"><strong>图 3-1 设备蜂窝接口配置</strong></p>

**验证方法**：
1. 查看系统状态灯为绿色常亮，网络指示灯为绿色常亮。
2. 通过PC访问任意互联网网站，确认能够正常打开。

**常见问题**：
- 网络连接失败：检查SIM卡是否正确插入、APN参数是否正确。
- 数据发送接收异常：检查信号强度和流量余额。

### 场景2：有线联网

**目标**：通过有线WAN口接入互联网。

**前提**：设备已上电，WAN口已连接上行网络。

**预计用时**：约5分钟。

**操作步骤**：
1. 用网线将CPE02的WAN口连接至上行网络（如光猫、交换机）。
2. 默认情况下，WAN口自动通过DHCP获取IP地址，无需额外配置。
3. 若需配置静态IP或PPPoE拨号，进入【Internet】菜单，选择对应WAN接口进行配置。
4. 点击"保存"并等待连接建立。

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1750218611330-3c60bd6f-519f-459d-8cc6-866e1ec96b77-107845.webp" alt="设备WAN口配置"></p>

<p align="center"><strong>图 3-2 设备WAN口配置</strong></p>

1. 查看网络指示灯为绿色常亮。
2. 通过PC访问任意互联网网站，确认能够正常打开。

**常见问题**：
- 无法获取IP地址：检查上行网络是否正常、网线是否连接牢固。
- PPPoE拨号失败：检查宽带账号密码是否正确。

### 场景5：建立IPsec VPN隧道

**目标**：通过IPsec VPN隧道实现分支机构与总部网络的安全互访。

**前提**：CPE02作为IPsec客户端，对端存在可访问的IPsec服务器；已获取预共享密钥、对端地址、本地子网、对端子网等信息。

**预计用时**：约15分钟。

**操作步骤**：
1. 登录CPE02本地Web界面，进入【VPN】→【IPsec VPN】，点击"添加"。

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1751246771200-66ad1429-82c1-44a1-b742-b41893f3ff57-919408.webp" alt="IPSec VPN编辑/新建入口"></p>

<p align="center"><strong>图 3-8 IPSec VPN编辑/新建入口</strong></p>

2. 填写名称、预共享密钥、上行接口、对端地址、本地子网、对端子网等必填参数。

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1751246821256-0a4cbe17-8c3c-475c-bb65-db6de6070767-240433.webp" alt="配置IPSec VPN"></p>

<p align="center"><strong>图 3-9 配置IPSec VPN</strong></p>

3. 根据实际需求配置IKE策略和IPsec策略，或保留默认值。
4. 点击"保存"，等待隧道建立。

**验证方法**：
1. 进入【状态】→【VPN】菜单，查看IPsec VPN状态是否为"已建立"。
2. 从本地子网中的终端访问对端子网中的设备，确认通信正常。

**常见问题**：
- 隧道无法建立：检查预共享密钥两端是否一致，对端地址是否可达，本地子网和对端子网是否配置正确。
- 隧道建立后无法通信：检查防火墙规则是否放行相关流量，确认静态路由配置正确。

## 第4章 功能说明与参数参考

### 4.1 WPS按钮

CPE02支持WPS功能，可为支持WPS的客户端提供免密连接Wi-Fi的功能。

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1750907202531-4adfd624-3ce7-49d7-ab9f-60796b002306-424374.webp" alt="WPS按键"></p>

<p align="center"><strong>图 4-1 WPS按键</strong></p>

**操作步骤**：

#### 4.3.6 上行链路设置

用户可通过"Internet>上行链路设置"功能配置链路检测设置，并配置不同上行接口之间的协作模式。

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1750937081841-f5f19375-0fb4-4ee3-863a-af59d1b5f266-759951.webp" alt="上行链路设置"></p>

<p align="center"><strong>图 4-28 上行链路设置</strong></p>

1. 链路探测开关：开启后对所有上行接口的连通性进行探测，默认启用。
2. 探测地址：填写指定的探测目标地址。
   - 填写后，所有上行链路只探测此地址，当填写的地址不可达时，会影响设备联网。
   - 未填写时，设备将从上行接口的DNS地址和备选探测地址选择可用的地址进行探测。
3. 默认探测上行接口的DNS地址及备选地址119.29.29.29和223.5.5.5。
4. 基于探测项的链路切换：链路备份模式下，用户可设置探测项触发链路间切换。需同时满足以下条件：
   - 开启链路开关。
   - 启用探测项。
   - 工作在"链路备份"模式且非"不切换"选项。

> **注意**：
> - 对Internet菜单的修改可能导致设备联网中断，需谨慎操作。
> - 当链路探测地址为空时，默认探测上行接口获取的DNS地址。填写探测地址后，所有上行接口只会探测用户填写的地址。
> - 当路由器处于链路备份模式时，用户可自定义探测参数，设备将根据使能的探测项进行链路切换。未使能探测项时，上行链路切换仅根据优先级和链路连通性触发切换。
> - 当设备处于负载均衡模式时，所有可以正常工作的上行链路都会转发业务流量。

|------|------|

### 4.6 VPN

虚拟专用网络（VPN）是一种加密技术，用于在公共互联网上创建安全的、私人的网络连接。它允许用户通过互联网安全地访问私人网络资源，无论他们身在何处。VPN通过加密通信数据，确保通信的机密性和安全性，防止未经授权的访问。这种技术对于连接到公司网络、维护在线隐私以及访问受限制的内容非常有用。VPN具有广泛的应用领域，包括企业、个人和移动设备领域。它是保护隐私和数据安全的关键工具之一。

#### 4.6.1 IPsec VPN

IPsec（Internet Protocol Security）VPN是一种用于加强网络通信安全性的协议套件，旨在通过加密和认证来保护数据的传输。它广泛应用于建立安全的远程访问、站点到站点连接以及虚拟专用网络（VPN）。CPE02通常作为IPsec的客户端接入。

#### 4.8.3 DNS Server

DNS（Domain Name System）服务器是一种关键的网络组件，它负责将人类可读的域名（如www.example.com）转换为计算机能理解的IP地址（如192.168.1.1）。DNS服务器充当了互联网上的地址簿，它帮助计算机和设备找到其他设备的位置，并确保信息能够在网络上正确传递。

当用户在"服务>DNS Server"中没有设置DNS服务器地址时，使用设备上行接口获取到的DNS地址进行域名解析；当用户设置DNS服务器地址后，将使用配置的DNS地址进行域名解析。

#### 4.8.6 Dynamic DNS

动态DNS（Dynamic Domain Name System）用于自动更新域名系统中的名称服务器内容。根据互联网的域名规则，域名通常需要与固定的IP地址关联。动态DNS技术能够为动态IP地址的用户提供固定的名称服务器，这使得外部用户能够通过定期更新的方式连接到动态IP地址的用户的URL。

### 案例1：分支机构通过IPsec VPN接入总部网络

**场景描述**：某企业在不同地区设有分支机构和总部，分支机构部署CPE02作为边缘网关，总部部署ER2000作为中心路由器，两地通过IPsec VPN隧道实现安全互访。

**网络拓扑**：

<p align="center"><img src="./img/jIZJ3DrGUjxbW8hw/1751247369531-de6eb2db-b795-40f8-8020-6283605869ed-274209.webp" alt="IPsec VPN组网拓扑"></p>

<p align="center"><strong>图 5-1 IPsec VPN组网拓扑</strong></p>

**设备角色**：本设备（CPE02）作为IPsec VPN客户端，负责将分支机构的本地子网安全接入总部网络。

**配置步骤**：
1. 在CPE02上完成蜂窝或有线联网，确保设备能够访问互联网。
2. 登录CPE02 Web界面，进入【VPN】→【IPsec VPN】，点击"添加"。
3. 填写名称、预共享密钥、上行接口、对端地址、本地子网、对端子网等必填参数。
4. 根据实际需求配置IKE策略和IPsec策略，或保留默认值。
5. 点击"保存"，等待隧道建立。
6. 在总部ER2000上配置对应的IPsec VPN服务器参数，确保预共享密钥、本地子网、对端子网与CPE02匹配。

## 附录 安全注意事项

1. 请使用原装电源适配器，避免因电源适配器和设备不匹配造成设备损坏。
2. 安装设备时请勿将设备安装在强电磁干扰环境下，避免与高功率设备过于接近。安装完成后请确认设备是否易发生晃动，避免因设备跌落造成设备损坏。
3. 请确认设备使用环境满足规格书所要求的温度和湿度。
4. 请定期检查设备线缆，包括网线、电源适配器连接线，保持线材清洁，如有发生破损请及时更换。
5. 清洁设备时，避免使用化学试剂直接喷洒在设备表面，避免损害外壳或内部组件。需使用柔性的布料清洁产品。
6. 请勿私自拆修或改装设备，这将会导致安全问题并且设备会失去保修资格。

> **警告**：非专业人员请勿打开设备外壳，存在触电风险。

## FAQ 常见问题解答

### 问题1：云平台是否免费？

映翰通一直致力于为中小连锁机构提供优质的网络服务，用户使用云平台服务时，需要逐台购买许可费用，以享受丰富的云端功能。

### 问题2：设备如何添加到云平台？

1. 首先通过 <https://star.inhandcloud.cn/> 网址注册小星云管家登录账号。
2. 使用注册账号登录到云平台，在设备菜单下点击"添加"，按照提示填写设备的序列号和MAC地址，即可完成添加。设备首次添加时，默认赠送1年免费基础版许可，后续用户可按需续费。

### 问题3：设备是否可以不用云平台？

可以。用户可在本地完成绝大多数配置功能，但批量的配置下发、升级、Connector等功能，需要结合云平台使用。若无法通过上述答案解决问题或遇到其他问题，请联系北京映翰通网络技术股份有限公司获取技术支持。可访问 www.inhand.com.cn 获取更多信息。

### 问题4：ER路由器与普通路由器区别？

1. 边缘路由器：ER路由器支持多种联网方式（有线/5G），多千兆网口和Wi-Fi 6芯片设计，支持各类终端设备接入，同时设备支持SD-WAN、带外管理和云管理。
2. 普通路由器：通常依赖固定的宽带连接，如DSL、光纤，通过有线连接到网络。普通路由器无统一的管理平台和防火墙、SD-WAN等高级特性。

### 问题5：4G/5G无法联网？

1. 物理环境：首先检查SIM卡是否插到正确的卡槽位，蜂窝天线是否全部安装。
2. APN设置：确保APN的配置信息与运营商提供的一致。
3. 检查设备连通性：登录到设备本地界面，通过系统自带的ICMP工具，ping 8.8.8.8测试是否连通，若可连通，则检查终端设备（如电脑、手机）与路由器的连通性。
4. 判断SIM卡是否正常：取出SIM卡，安装在手机中查看是否可以正常联网。
5. 重启：尝试将路由器断电，等待几秒钟，然后重新连接电源，重试网络连接。
6. 恢复出厂：将路由器恢复出厂，然后重新尝试。
