# DeviceLive 用户手册

## 1. 产品简介

### 1.1 产品概述与定位

DeviceLive平台专为工业物联网领域设计，搭配映翰通边缘智能硬件，帮助工业企业快速构建智能的边缘网络。平台具备设备管理、远程监控、边缘计算应用管理和终端远程维护四大能力，云端协同助力实现智能边缘部署升级、边缘数据采集预处理和状态可视化监控。


### 1.2 核心功能与应用场景

与上节一致，平台面向工业物联网场景，搭配映翰通边缘智能硬件；能力包括设备管理、远程监控、边缘计算应用管理、终端远程维护；云端协同用于智能边缘部署升级、边缘数据采集预处理和状态可视化监控。

### 1.3 名词解释

| **名词** | **说明** |
| --- | --- |
| **机构/企业** | 在系统中，使用邮箱注册成功后，即创建了一个机构/企业。各个机构/企业之间的数据是完全隔离的。 |
| **组织** | 注册机构/企业成功后，您的机构/企业默认为是一级组织，您可以根据实际业务需求，在一级组织下创建其他组织。一级组织拥有最高数据权限，能查看当前机构的所有数据。下级组织只能看到当前组织及其下级组织的数据，无法看到上级组织的数据。 |
| **内部用户** | 属于当前机构/企业的用户。 |
| **外部用户** | 不属于当前机构/企业的用户，他们无法访问当前机构/企业的数据。但是由于业务需求，可以通过邀请的方式，使其能访问到当前机构/企业的数据，如提供技术支持。 |

## 2. 快速上手指南

### 2.1 前置条件（环境/浏览器/网络）

- 访问 DeviceLive 平台地址：<https://device.inhandcloud.cn/>。
- 注册与登录：文档中推荐使用 Google 浏览器。
- 设备接入前需完成现场网络连通；不同产品系列、固件版本的本地配置界面可能存在差异，以实际设备为准。
- 若设备通过蜂窝上网，物联网卡白名单需包含：`*.inhandcloud.cn`，`*.amazonaws.com.cn`（见 **7. 常见问题（FAQ）** 条目 2）。

### 2.2 三步快速入门流程

以下三步对应原文中的注册与登录、进入 DeviceLive、以及设备联网与上架流程；**界面截图与逐步说明见第 3 章、第 4 章对应小节**，此处不重复贴图以免与后文重复。

**步骤一：注册** — 使用浏览器访问 <https://device.inhandcloud.cn/>，在注册登录页完成账号注册（推荐使用 Google 浏览器）。详见 **3.1 登录 DeviceLive** 中的「注册」小节。

**步骤二：登录并进入 DeviceLive** — 注册完成后登录映翰通云服务控制台，通过 DeviceLive 的「访问」入口进入平台；亦可直接访问上述地址。详见 **3.1 登录 DeviceLive** 中的「登录」及安全设置说明。

**步骤三：设备联网并添加到平台**


设备将在接入DeviceLive后自动进行配置同步，实现云端管理。设备接入平台步骤：

1. 设备本地配置云管理；

2. 将设备添加到平台；

设备侧联网配置、本地云管理开关及在平台「设备」菜单中添加设备的操作，详见 **4.1 设备联网方式说明** 与 **4.2 设备添加与上线流程**（含原文配图）。

### 2.3 常见入门问题排查

以下条目与 **7. 常见问题（FAQ）** 中第 1、2 条一致（原文摘录）。

**1.平台添加设备报错**

在平台添加设备报错，通常为以下原因：

1. 序列号已存在：该设备已被添加到平台，无法重复添加；
2. 设备信息无效：请确认设备的序列号及MAC地址与设备铭牌上的信息一致；

如仍存在问题，请联系技术支持。\

**2.设备无法在平台上线**

设备添加到平台后无法上线，请排查：

1. 设备的网络是否正常；
2. 设备本地的云服务是否已启用；
3. 确认设备当前的固件版本是否支持连接云平台；
4. 设备如果通过蜂窝上网，请检查物联网卡的白名单地址是否包含：*.inhandcloud.cn，*.amazonaws.com.cn ；
5. 设备是否绑定了许可证，如未绑定，可进入“**映翰通云服务 >> 订阅**”进行许可证绑定。

**2.1.2用户角色**

DeviceLive提供四种角色：系统管理员、组织管理员、设备管理员、只读用户，各个角色的功能权限如下：

| **角色** | **功能权限** |
| --- | --- |
| 系统管理员 | 用户拥有最高权限，能查看系统所有数据，并拥有所有功能的操作权限。<br/>:::color3 \*\*注意：\*\*用于注册的账号默认为“系统管理员”。<br/>:::  |
| 组织管理员 | 1. 用户在DeviceLive具有所有的功能权限，但在映翰通云服务中仅具备控制台、用户管理和个人信息页面的功能权限；<br/>2. 用户只能查看到当前组织及其下级组织的数据。 |
| 设备管理员 | 1. 用户拥有总览、设备、云连接、分组、OTA升级、告警、报表、消息页面及映翰通云服务的控制台、个人信息页面的功能权限；<br/>2. 用户只能查看到当前组织及其下级组织的数据。 |
| 只读用户 | 1. 用户拥有总览、设备、云连接、分组、告警、报表、消息页面及映翰通云服务的控制台、个人信息页面的查看权限，无编辑管理权限；<br/>2. 用户可以为当前权限范围内数据生成报表，并下载查看；<br/>3. 用户只能查看到当前组织及其下级组织的数据。 |

**2.1.3分配角色**

管理员可以为其他用户分配角色。每个用户在同一应用下只能有一个角色，分配新角色将覆盖当前角色。

如果为用户分配的角色为系统管理员，则该用户在映翰通云服务的所有应用中都为最高权限。

#### 3.2.2 个人信息与系统设置

### 2.2个人信息

登录DeviceLive后，点击左上角的应用切换图标进入映翰通云服务，到“个人信息”页面，可修改账号登录密码、绑定手机号、添加双因素身份验证。也可以点击右上角账号图标，下拉点击“安全设置”。

![1719466865960-a69adc2a-ab2f-4a43-9c2c-ad31cd64326c.png](./img/lIXbPgQ0h5ykN6EX/1719466865960-a69adc2a-ab2f-4a43-9c2c-ad31cd64326c-858248.png)

**2.2.1修改登录密码**

点击“**修改**”，输入账号当前密码以验证身份，输入新密码并确认，新密码确认无误后提交修改即可修改成功。

:::color3
**注意：** 安全性高的密码可以使账号更安全，建议你定期更换密码。

:::

**2.2.2手机绑定**

绑定手机号后，可用手机号+短信验证码进行登录。或在配置系统告警时，可以用这个手机号来接收告警消息。\

**2.2.3双因素身份验证**

通过二次身份验证来提高账户安全性。你可以同时设置多种身份验证方式，以便于在你丢失设备或无法获取APP验证码时，可以用另一种方式作为备份。

1. 短信验证\  点击“设置”，按照操作提示绑定手机号码，绑定成功后再次登录时，会给绑定的手机号发送验证码以做二次登录验证。当你不需要短信验证时，可以点击“关闭”来关闭短信验证。
2. APP验证
   1. 点击“设置”，按照操作提示，在你的手机上下载安装Google Authenticator 或 Microsoft Authenticator APP。然后打开APP，扫描界面二维码来设置你的账号。
   2. 扫码后，输入APP上的6位验证码来验证身份，验证通过后，APP验证即绑定成功。绑定成功后在登录时，将需要你输入APP上的6位验证码来做二次登录验证。
   3. 当你不需要APP验证时，可以点击“关闭”来关闭APP验证。
3. 备份码验证\  在首次设置双因素身份验证时，会同时生成一个备份码，备份码可用于当你无法获取短信验证码或APP验证码时，可用这个备份码来绕过二次验证。登录时选择“使用备份代码”。

**2.2.4偏好设置**

设置当前账号中列表每页默认显示的条目数量以及系统语言。

### 2.3系统设置

管理员可进入“系统设置”页面，为整个机构设置登录超时时间、强制MFA及系统消息接收邮箱。

![1719466865960-104ad0fa-5182-4f59-859a-9a011409b301.png](./img/lIXbPgQ0h5ykN6EX/1719466865960-104ad0fa-5182-4f59-859a-9a011409b301-777680.png)

**登录空闲超时**

设置成功后，整个机构中的账号空闲时间超过设置时间后，将需要重新登录后才能访问。

**强制MFA**

开启后，机构内的所有用户都需要设置“双因素身份验证”后才能登录。

**系统消息接收**

目前支持许可证即将过期时，系统自动邮件提醒。您可以根据实际管理情况，设置指定的通知邮箱。


#### 3.2.3 多组织数据权限、企业信息、外部用户与注销（原文第 2.4–2.7 节）

### 2.4多组织数据权限

系统支持定义多级组织，用户、设备等资源都属于某一个具体的组织，组织内的用户可以查看当前组织及其下级组织的设备等数据。你可以使用此功能来实现用户账号的数据权限控制、多子客户隔离管理，让数据管理更有条理，保护账号隐私安全。

**2.4.1添加组织**

在“用户管理”页面，点击新增组织，新增组织时要选择组织的上级组织，默认注册账号所在的组织为一级组织。

![1719466866003-bc721201-3126-4437-871b-bcf3f47e6c28.png](./img/lIXbPgQ0h5ykN6EX/1719466866003-bc721201-3126-4437-871b-bcf3f47e6c28-808831.png)

**2.4.2设备与组织**

在“设备”页面，新增和导入设备默认会把设备添加到操作人所在的组织。支持修改设备所在的组织。

选中设备，点击“**移动**”，选择要移入的组织或组织下的分组，选择后，保存修改。此时，设备会移动到新的组织下，只有新组织下的用户才可以在“设备”页面看到此设备。\

**2.4.3分组与组织**

在“分组”页面，新增分组要选择分组所在的组织。只有同组织下的用户才能在“分组”页面看到这些分组，同时，也可以在“设备”页面看到这些分组内的设备。\

**2.4.4云连接与组织**

在“云连接”页面，新增云连接网络要选择网络所在的组织。只有同组织下的用户才能在“云连接”页面看到这些网络，同时可以对网络内的设备、终端、账号成员进行操作。

当用户具备某个网络权限，但不具备网络内的设备权限时，用户还是只能从网络维度对设备进行操作，无法在“设备”页面看到这些设备。\

**2.4.5用户与组织**

在“用户管理”页面，新增账号要选择账号所属组织。一个账号能看到的数据（包括设备、分组、网络、用户）为所属组织及其下级组织内的数据。

:::color3
**注意：** 一级组织内的用户能看到当前机构内的所有数据，包括所有的设备、分组、网络和用户数据。

:::

### 2.5企业信息

在DeviceLive顶部菜单栏，点击“企业管理”，即可跳转到映翰通云服务管理当前机构的企业信息。

![1719466866068-fa10d04b-bf8d-482f-863d-7275f66b6e95.png](./img/lIXbPgQ0h5ykN6EX/1719466866068-fa10d04b-bf8d-482f-863d-7275f66b6e95-240764.png)

企业信息管理包含以下信息：

- 企业基本信息：企业名称、所属地区、系统管理员等；
- 联系人：企业的联系人信息；
- 账单管理信息：出账的日期及账单的收件信息；

- 关联MSP账号：当您需要MSP提供帮助时，您可以在此处填写MSP在映翰通云服务的邮箱账号。关联MSP账号后，MSP可以为您的账号添加设备和许可证等。

### 2.6外部账号

当你需要某个企业外部用户加入组织来提供外部支持时，如果这个外部用户在DeviceLive已注册邮箱账号，此时你可以通过添加“外部用户”的方式来邀请他加入你当前的机构。

1. 进入“**用户管理**”页面，点击用户列表上方的添加；

- Compose模式部署

原生容器通过docker - compose.yml文件进行部署，配置应用程序所需的各个服务、网络和存储卷，部署格式如下：

```plain
version: "3"
services:
  nginx:
    image: nginx
    ports:
      - 80:80
      
networks:
  my_network:
    driver: bridge
    
volumes:
  nginx_data:
```

此 `docker-compose.yml` 文件主要功能为构建一个基于 `nginx` 镜像的服务，将容器 80 端口映射至主机 80 端口以提供对外访问，同时创建名为 `my_network` 的桥接网络用于容器间网络通信，还定义了 `nginx_data` 存储卷，虽当前未被 `nginx` 服务直接挂载使用，但为数据持久化及多容器应用网络架构搭建与扩展提供了基础支撑与可能。

**二、原生应用部署**

原生应用采用 systemd 来管理，它的主要功能是负责启动、停止、管理系统中的各种服务和进程，同时也处理系统的初始化任务，部署格式如下：

```plain
[Unit]
Description=test systemd
After=network.target

[Service]
Type=simple
EnvironmentFile=/userdata/app/run/.env
ExecStart=/userdata/app/run/test.sh $MY_VAR

[Install]
WantedBy=multi-user.target
```

在这个单元文件中，`[Unit]`部分描述了单元的基本信息，如服务的描述和依赖关系（这里表示在网络启动后启动该服务）。`[Service]`部分定义了服务的启动类型（`ExecStart`）、服务环境变量（`Environment`）和启动命令（`ExecStart`）。`[Install]`部分指定了该服务在系统启动时应该被安装到哪个目标（`multi - user.target`表示多用户模式下启动）。

配置步骤如下：

1. 进入“项目”页面，点击”添加“按钮；
2. 选择”原生应用”并点击“添加”

### 5.6 终端管理


传统模式中，当设备出现故障，需要修改程序或调试时，往往需要工程师出差到现场对设备进行维护，会耗费许多人力物力。Device Live平台支持对串口/以太网连接的PLC/HMI设备进行远程管理和维护。当设备出现故障，需要修改程序或监控时，能直接通过PC远程访问到现场的PLC、HMI等设备，从而实现对设备的程序上下载、在线监控程序运行情况等，减少运营维护成本。

您可以在“终端管理”列表或者“网关详情 >> 终端”列表添加终端，以下将以设备详情页面添加终端为例，为您介绍如何通过DeviceLive远程访问您的串口/网口终端。

### 9.1添加终端

### 9.1.1 添加串口终端设备到平台

1. 进入DeviceLive平台的【设备】页面，点击设备名称，进入设备详情的【终端】页面；

2. 点击“添加终端”，并输入网关连接的PLC信息提交；

注意：请确认终端信息与实际保持一致。

![1768470138261-b2a1f1df-6b97-4cfa-a8b8-74823a761e75.png](./img/lIXbPgQ0h5ykN6EX/1768470138261-b2a1f1df-6b97-4cfa-a8b8-74823a761e75-115073.png)

| **字段名称** | **描述** |
| :--- | :--- |
| 终端名称 | 自定义的PLC/HMI等终端设备的名称； |
| 连接类型 | 串口或二层网口； |
| 串口名称 | 网关设备的串口，如COM1； |
| 串口类型 | 串口对应的协议类型； |
| 波特率 | 串口的波特率信息； |
| 数据位 | 串口的数据位信息； |
| 停止位 | 串口的停止位信息； |
| 校验位 | 串口的校验位信息。 |

### 9.1.2 添加网口终端设备到平台

1. 进入DeviceLive平台的【设备】页面，点击设备名称，进入设备详情的【终端】页面；

1. 在“**订阅>>设备**”页面，点击操作列的**应用许可**（**设备已经应用许可**）；

2. 在叠加许可弹窗，选择许可证，并点击应用。

**说明：**

①叠加后，原应用给设备的许可证的时长将延长；

②叠加操作不可逆，许可证叠加后，无法再拆分。

:::

![1719485392362-8029a12f-6c50-447e-8a62-7aac3ceda14d.png](./img/lIXbPgQ0h5ykN6EX/1719485392362-8029a12f-6c50-447e-8a62-7aac3ceda14d-498543.png)

4、批量应用许可

当设备量较大时，平台支持批量快速的为多个设备应用许可证。操作步骤如下：

1. 在“**订阅>>设备**”页面，点击设备列表上方的批量许可；

2. 选择需要应用的**许可证类型及设备**，点击**批量许可**，平台将自动分配许可；

3. 确认许可结果，并点击**应用**，批量许可生效。

为了保证您账号下的许可得到最大化的利用，平台将优先分配已激活的许可，其他类型许可将不区分时长随机分配。

![1719485412015-7e26e1b1-d937-428d-9a07-235ac2bb4948.png](./img/lIXbPgQ0h5ykN6EX/1719485412015-7e26e1b1-d937-428d-9a07-235ac2bb4948-005231.png)

:::color3
\*\*注意：\*\*如果设备已有在用的同类许可，再次应用后，许可时长将会叠加，此操作不可逆，请谨慎操作。

:::

5、单个取消许可

当某个设备业务用途变更，需要切换其他类型的许可证，或由于设备故障报修，需要把许可证应用到其他设备时，可以取消这个设备的许可证，取消后，被取消的许可证可以应用到其他设备，设备原许可范围内的服务将立即终止。操作步骤如下：

1. 在“**订阅>>设备**”页面，点击目标设备操作列的**取消许可**；

2. 二次确认后，许可将从设备取消应用。

![1719485451002-e23d073a-42da-4f53-b1df-38c20b6f14a9.png](./img/lIXbPgQ0h5ykN6EX/1719485451002-e23d073a-42da-4f53-b1df-38c20b6f14a9-980547.png)

6、批量取消许可

平台支持快速取消多个设备的许可证，取消后，被取消的许可证仍可以应用到其他设备，设备原许可范围内的服务将立即终止。

1. 在“**订阅>>设备**”页面，选择需要取消许可的设备；

2. 点击设备列表上方的**取消许可**，二次确认即可。

![1719485466529-19688d2b-27cf-4334-abfb-129a959b0dec.png](./img/lIXbPgQ0h5ykN6EX/1719485466529-19688d2b-27cf-4334-abfb-129a959b0dec-857869.png)

:::info
\*\*说明：\*\*未应用许可证的设备，不支持取消许可。

:::

### **15.4升级许可证**

当设备绑定的低版本许可证功能不足以满足业务需求时，您可以升级这个设备的许可证。操作步骤如下：

1. 在“**订阅>>设备**”页面，点击设备列表上方的**升级许可**；

2. 选择需要升级的**许可证类型**。选择后，平台将自动过滤绑定了此许可类型的设备；

3. 选择需要升级许可的**设备**；

4. 选择**预升级到的许可证类型**，点击提交；

5. 确认升级许可的结果，点击确认后，许可升级生效，设备的许可变更为高级版许可。如取消升级许可，则此次升级不生效。

![1719485525892-fea74603-5a31-4673-ac3c-125f99917e9b.png](./img/lIXbPgQ0h5ykN6EX/1719485525892-fea74603-5a31-4673-ac3c-125f99917e9b-422697.png)\

:::info
\*\*说明：\*\*支持对已激活的同类型许可批量升级。升级后，许可证剩余时长将根据前后两个许可证类型的价格比重进行等比调整。

:::

### **15.5转移许可证**

平台支持用户转移许可证到其他账号，转移后，许可证将从原账号移除；已应用到设备的许可证不支持转移。操作步骤如下：

1. 其他账号需要成为您在映翰通云服务的客户；

(1) 进入“**客户 >> 客户列表**”添加和管理客户；

(2) 客户收到邀请邮件，并**接受邀请**。\ ![1739773536296-047ecd43-14f3-44e9-a7bc-b26387e059ff.png](./img/lIXbPgQ0h5ykN6EX/1739773536296-047ecd43-14f3-44e9-a7bc-b26387e059ff-415808.png)

1. 在“订阅>>许可证”页面，选择需要转移的许可证；
2. 在“转移许可”弹窗，确认需要转移的许可证，并选择客户，点击转移，即可转移成功；

![1722409145484-280675b3-8556-470b-91f7-10ddad5036b2.png](./img/lIXbPgQ0h5ykN6EX/1722409145484-280675b3-8556-470b-91f7-10ddad5036b2-254724.png)

:::info
\*\*\*\***说明：**

①使用中或已过期的许可证不支持转移；

②单次最多可转移1000个许可证。

:::

### **15.6转移设备**

平台支持将设备及其绑定的许可证转移给其他账号，在执行转移操作前，您需要同“转移许可证”一样，让其他账号成为您在映翰通云服务的客户。操作步骤如下：

1. 在“订阅>>设备”页面，选择要转移给客户的设备;

2. 点击列表上方的转移；

3. 在“转移设备”弹窗，确认转移的设备并选择转移到的客户账号，点击确认，即可转移成功。

![1719485674410-1a4b2fce-f253-4937-8edc-17c0e8e6cd2b.png](./img/lIXbPgQ0h5ykN6EX/1719485674410-1a4b2fce-f253-4937-8edc-17c0e8e6cd2b-054240.png)

:::color3
\*\*注意：\*\*转移后，设备及绑定的许可证将会从当前账号移除。

### **15.7统一许可到期日期**

当您需要对齐设备许可服务的到期时间，以便于您更好的管理业务，统一财务对账结算。你可以执行以下操作：

1. 进入“**设备**”或“**许可证**”页面；
2. 选择需要对齐到期时间的许可证；
3. 点击“统一许可到期日期”；
4. 确认统一后的结果信息，点击“确定”，系统将自动将所选的许可证的到期时间统一到同一时间。

![1719487194489-6501dda6-f512-4d99-ba6f-f65a55466be3.png](./img/lIXbPgQ0h5ykN6EX/1719487194489-6501dda6-f512-4d99-ba6f-f65a55466be3-805741.png)

:::color3
**注意：**

①系统会自动把所选的同一类型的许可证的过期日期统一到同一时间。例如：当前为2024-06-01，选择了2个剩余时长为1年、1个剩余时长为60天的基础版许可证，那么平均剩余时长为：（2*365+1*60）/ 3= 264。则这三个许可证的统一后过期日期为：2024-06-01 + 264天 = 2025-02-20。

②“统一许可到期日期”操作不可逆，请谨慎操作。

:::

## 7. 常见问题（FAQ）

**1.平台添加设备报错**

在平台添加设备报错，通常为以下原因：

1. 序列号已存在：该设备已被添加到平台，无法重复添加；
2. 设备信息无效：请确认设备的序列号及MAC地址与设备铭牌上的信息一致；

如仍存在问题，请联系技术支持。\

**2.设备无法在平台上线**

设备添加到平台后无法上线，请排查：

1. 设备的网络是否正常；
2. 设备本地的云服务是否已启用；
3. 确认设备当前的固件版本是否支持连接云平台；
4. 设备如果通过蜂窝上网，请检查物联网卡的白名单地址是否包含：*.inhandcloud.cn，*.amazonaws.com.cn ；
5. 设备是否绑定了许可证，如未绑定，可进入“**映翰通云服务 >> 订阅**”进行许可证绑定。

**3.\*\*\*\*外部用户如何切换机构到您的机构提供支持？**

1.外部用户需要先接受邮件邀请，然后登录到DeviceLive；

1. 点击页面右上角个人信息，点击“切换组织”；

3.在切换组织弹窗中，选择需要协助的组织，并点击“切换”，即可成功切换到对应组织提供技术支持。
