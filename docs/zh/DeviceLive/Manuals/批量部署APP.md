# 批量部署APP

## 步骤一：新增分组

在部署原生应用、容器应用之前，需要创建一个分组，并将相应设备添加到对应分组中，如下图所示：

1. 在左侧的向导栏中选择“分组“，进入分组页面，点击”创建“
2. 依次填写分组名称、需要部署APP的设备型号及版本信息，并点击“保存”
3. 在弹出框中勾选需要部署APP的具体设备
4. 最后点击“保存”，设备被成功添加到分组当中。

## 步骤二：新增环境增量包

在设备上部署的应用对宿主机有环境要求的情况下（例如：为了正常运行应用程序，宿主机必须先安装Python 3.7运行环境以及相关的Python依赖库，如pymysql和paho-mqtt），则可以先找一台测试设备，手动部署好环境，并通过平台提取出对应增量包。步骤如下：

1. 找一台demo设备，部署好设备应用所需的环境
2. 将设备添加到DeviceLive上对应分组中，参考步骤一
3. 在“分组”页面，点击“上传环境增量包”
4. 依次填写增量包名称（可自定义），选择设备（已经手动部署好环境的demo设备），并对本增量包进行描述，方便一眼看出本增量包的作用

## 步骤三：应用部署

### 一、容器应用部署

### 1、Compose模式部署

原生容器通过docker - compose.yml文件进行部署，配置应用程序所需的各个服务、网络和存储卷，部署格式如下：

```plain
version: "3"
services:
  nginx:
    image: nginx
    ports:
      - 80:80
      
networks:
  my_network:
    driver: bridge
    
volumes:
  nginx_data:
```

此 `docker-compose.yml` 文件主要功能为构建一个基于 `nginx` 镜像的服务，将容器 80 端口映射至主机 80 端口以提供对外访问，同时创建名为 `my_network` 的桥接网络用于容器间网络通信，还定义了 `nginx_data` 存储卷，虽当前未被 `nginx` 服务直接挂载使用，但为数据持久化及多容器应用网络架构搭建与扩展提供了基础支撑与可能。

配置步骤如下：

1. 进入“项目”页面，点击”添加“按钮；
2. 在”容器应用“中，选择”Compose“模式
3. 输入docker - compose.yml文件内容

### 二、原生应用部署

原生应用采用 systemd 来管理，它的主要功能是负责启动、停止、管理系统中的各种服务和进程，同时也处理系统的初始化任务，部署格式如下：

```plain
[Unit]
Description=test systemd
After=network.target

[Service]
Type=simple
EnvironmentFile=/userdata/app/run/.env
ExecStart=/userdata/app/run/test.sh $MY_VAR

[Install]
WantedBy=multi-user.target
```

在这个单元文件中，`[Unit]`部分描述了单元的基本信息，如服务的描述和依赖关系（这里表示在网络启动后启动该服务）。`[Service]`部分定义了服务的启动类型（`ExecStart`）、服务环境变量（`Environment`）和启动命令（`ExecStart`）。`[Install]`部分指定了该服务在系统启动时应该被安装到哪个目标（`multi - user.target`表示多用户模式下启动）。

配置步骤如下：

1. 进入“项目”页面，点击”添加“按钮；
2. 选择”原生应用”并点击“添加”
3. 输入”Systemd 配置“，并点击确认

### 三、应用依赖配置

应用依赖包含两个部分：应用依赖文件和环境增量包。

- 应用依赖文件：包括运行应用所需的可执行文件和脚本、相关环境变量等文件；例如在部署原生应用时，在ExecStart部分，所需要执行的文件、脚本等。
- 环境增量包：运行应用所需要的运行环境，选择在“步骤二：新增环境增量包”部分创建好的环境增量包

配置步骤如下：

1. 进入“项目”页面，点击”添加“按钮，新增一个版本号
2. 选择”应用依赖配置“，并选择上传对应依赖文件
3. 最后点击”提交“

## 步骤四：批量部署到分组

在步骤一至步骤三中，我们分别对容器应用、原生应用、应用依赖关系（包括环境增量包）进行了配置，最后配置好的环境批量部署到组内设备即可，配置步骤如下：

1. 点击”项目“，在创建好的多个项目版本中，选择目标版本
2. 点击目标版本后面的”部署“按钮，点击部署到分组
3. 在弹出的框内选择“确定“，当页面弹出”操作成功“提示，代表部署成功
