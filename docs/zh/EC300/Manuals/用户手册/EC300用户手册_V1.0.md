# 第1章 认识设备

## 1.1 概述

EC300是基于Arm架构的边缘计算机，面向工业自动化及轻量级边缘应用场景设计。设备内置Linux系统，提供丰富的工业接口，包括以太网、串口、CAN总线、数字量输入输出、模拟量输入及USB等，并支持通过扩展模块进一步扩充接口能力。EC300可内置映翰通DeviceSupervisor™ Agent服务，实现数据采集、处理及上云，同时支持DeviceLive云管理，适用于智能制造、能源监控、交通运输等需要边缘计算与远程管理的场景。

## 1.5 恢复出厂设置

有两种方法可以恢复出厂设置：

1、通过键入命令，系统会自动重启并恢复出厂设置。

![1719803386710-456311b5-ab25-4c6d-a7d7-b3b6f68d1285.png](D:/user-manual/zh/device-dsa/EC300/img/59Hs2LEPhAS7d-b_/1719803386710-456311b5-ab25-4c6d-a7d7-b3b6f68d1285-785580.png)

2、通过按键恢复出厂设置：

（1）长按恢复出厂设置按钮10-20s，看到警告灯为常亮状态。

（2）当警告灯常亮时，释放恢复出厂设置按钮。

**注意：**

（1）系统复位是Arm计算机的一项重要功能。系统复位后，Arm计算机将恢复到默认状态，所有用户数据和配置将丢失。

（2）重置Arm计算机系统前，请确保关键数据已有效备份并传输到外部存储介质（如可移动磁盘）



# 第2章 安装与首次使用

## 2.1 安装前准备

### 2.1.1 环境要求

EC300边缘计算机的安装环境需满足以下基本要求：

- 稳定的安装平面（DIN轨、墙面或机柜）
- 可用的电源接口
- 用于首次配置的PC机（Linux或Windows系统）
- 以太网网线

### 2.1.2 工具准备

根据安装方式的不同，需准备相应的工具和配件：

- DIN轨安装：DIN轨安装板（出厂已附在设备后面板上，采用6个M3螺钉固定）
- 壁挂式安装：壁挂套件（需单独购买）、M3螺钉
- 电源连接：适配的电源线
- 首次配置：以太网网线、SSH客户端软件

### 2.1.3 安全注意事项

> **警告**：安装SIM卡前，必须先断开设备电源。带电插拔SIM卡可能导致设备损坏或数据丢失。

> **注意**：出于安全考虑，建议在首次登录后禁用默认的用户账户，并创建新的用户帐户。

## 2.3 首次登录

首次配置EC300需要一台PC机，通过以太网网线连接到EC300并登录到命令行界面。

出厂默认的用户名和密钥：

- **Username**: edge
- **Password**: security@edge

EC300设备出厂默认创建了root用户，但登录被禁用。如需使用root用户，需手动修改系统配置，输入`sudo -s`切换至root用户。用户edge在sudo组中，因此可在edge用户下使用`sudo`来执行系统级命令。

> **注意**：当出现`command not found`提示时，输入`sudo -s`切换至root用户，或使用`sudo`命令来操作。

EC300支持通过以太网的SSH（Secure Shell，可以理解为：一种加密的远程登录协议）连接。使用以下默认IP地址连接到EC300：

| 端口 | 默认IP |
| --- | --- |
| ETH 1 | 192.168.3.100 |
| ETH 2 | 192.168.4.100 |

## 场景2：蜂窝拨号上网

**目标**：通过4G/5G蜂窝网络接入互联网。

**前提**：已插入SIM卡并安装天线，设备已上电。

**预计用时**：约10分钟。

**操作步骤**：
1. 登录设备Web管理界面，进入【网络】→【蜂窝】。
2. 在蜂窝配置页面中，确认`Enabled`为开启状态（默认值：开启）。
3. 根据SIM卡类型配置拨号参数：
   - 若使用普通公网SIM卡，通常无需修改`Profiles`及其他参数。
   - 若使用专网SIM卡，在`Profiles`中添加拨号参数集，配置APN（Access Point Name，可以理解为：运营商网络的"门牌号"，用于指定移动数据网络的接入点）、用户名、密钥及认证方式；拨号参数集最多可添加10条记录。随后在`APN Profile`中选择对应参数集的Index（默认值：自动）。
4. `Network Mode`选择蜂窝网络制式（如LTE、WCDMA等）；若不确定，保持默认值`自动`。
5. 确认`Enable Default Route`为开启状态（默认值：开启），以便拨号成功后自动添加蜂窝口的默认路由。
6. （可选）如需使用双卡单拨（可以理解为：设备插入两张SIM卡，主卡拨号失败时自动切换到副卡的功能），开启`Dual SIM Enabled`（默认值：关闭），并配置以下参数：
   - `Main SIM`：选择主SIM卡（默认值：sim1）。
   - `Max Number of Dials`：设置当前SIM卡拨号失败达到指定次数后切换至另一张SIM卡。
   - `PIN Code`：SIM卡的PIN码。
7. （可选）建议开启ICMP探测以检测拨号假连接。配置以下参数：
   - `ICMP Detection Server Probes`：配置1~2个ICMP探测地址；未配置任何地址时，ICMP探测功能关闭。
   - `Detection Interval`：设置ICMP探测周期。
   - `Detection Timeout`：设置ICMP探测超时时间，即等待多久未收到响应则判定探测失败。
   - `Detection Max Retries`：设置最大探测次数，探测失败达到该值时触发重新拨号。取值范围：[1,5]。
   - `Detection Strict`：是否开启严格探测。关闭时，探测程序在每个周期检测蜂窝接口接收报文数量是否变化，有变化则跳过ICMP探测以节省流量；开启时，无论接收报文数量是否变化，均周期性发送ICMP探测报文（默认值：关闭）。
8. （可选）在高级配置中，可根据需要调整以下参数：
   - `Debug Mode enabled`：是否启用拨号调试日志（默认值：关闭）。
   - `Enable Infinitely Redial`：是否开启无限重拨。开启后，无论拨号失败多少次，系统均不会重启；关闭时，拨号失败达到一定次数后会重启系统尝试恢复（默认值：关闭）。
   - `Dial Interval`：拨号失败后进行下一次拨号前的等待时间。
   - `Signal Query Interval`：信号强度检测周期。
9. 点击保存或应用配置，等待设备拨号成功。

**验证方法**：
1. 在蜂窝配置页面查看拨号状态显示为成功。
2. 使用`ping`命令测试外网地址（如配置的ICMP探测地址），确认网络连通。

**常见问题**：
- 拨号状态显示成功但无法ping通目标地址（假连接）。建议开启ICMP探测功能，以便在出现假连接时自动重新拨号恢复。
- 未插入SIM卡或SIM卡欠费时，拨号将失败。若未开启无限重拨，系统可能在多次拨号失败后自动重启。

## 场景3：Wi-Fi配置

**3.1 连接Wi-Fi热点（Station模式）**

**目标**：使EC300作为客户端连接到现有Wi-Fi网络。

**前提**：已获取目标Wi-Fi网络的SSID（Service Set Identifier，可以理解为：Wi-Fi网络的名称标识）及连接密码。

**预计用时**：约5分钟。

**操作步骤**：
1. 登录设备Web管理界面，进入【网络】→【Wi-Fi】。
2. 开启Wi-Fi使能开关。
3. 在Client SSID处输入需要连接的SSID（Service Set Identifier，可以理解为：Wi-Fi网络的名称标识），或通过扫描按钮获取附近可用的SSID。
4. 根据目标Wi-Fi网络的认证方式，选择Auth Method（认证方式），可选：无认证、WPA-PSK、WPA2-PSK、WPA-PSK/WPA2-PSK Mixed。
5. 选择Encrypt Mode（加密方式），可选：CCMP（Wi-Fi加密算法标准）、TKIP（Wi-Fi加密算法标准）、TKIP和CCMP。
6. 在WPA/WPA2 PSK Key处输入Wi-Fi密码。
7. 配置默认路由：
   - Enable Default Route：默认开启，Wi-Fi连接成功后会自动添加一条wlan口的默认路由。
   - Metric：设置Wi-Fi口默认路由的度量值。当蜂窝口、Wi-Fi和以太网口同时配置默认路由时，度量值最小的生效。
8. 保存配置并等待Wi-Fi连接成功。

<p align="center"><img src="./img/59Hs2LEPhAS7d-b_/1719803381849-ba863dde-0586-4ebd-b7fb-8e82840153d2-070728.webp" alt="Station模式配置界面"></p>
<p align="center"><strong>图 3-9 Station模式配置界面</strong></p>

**3.2 开启Wi-Fi热点（AP模式）**

**目标**：使EC300发射Wi-Fi信号供其他设备连接。

**前提**：确认设备支持的无线频段及射频类型，并规划好AP的SSID（Service Set Identifier，可以理解为：Wi-Fi网络的名称标识）、密码及IP地址参数。

**预计用时**：约5分钟。

**操作步骤**：
1. 登录设备Web管理界面，进入【网络】→【Wi-Fi】。
2. 配置AP模式相关参数：
   - SSID Broadcast（SSID广播）：默认开启。开启时，附近设备可扫描到该SSID；关闭时，需手动输入SSID进行连接。
   - Band（无线频段）：选择2.4G（适用于更远覆盖范围）或5.8G（适用于高速数据传输）。
   - Radio Type（射频类型）：选择无线标准，如802.11a/n/ac（支持5G）或802.11b/g/n（支持2.4G）。
   - Channel（信道）：手动选择无线信号使用的信道，或使用默认通道。
   - SSID：设置Wi-Fi的网络名称（SSID）。
   - Auth Method（认证方式）：选择Open（无认证）、WPA-PSK、WPA2-PSK或WPA-PSK/WPA2-PSK Mixed。
   - Encryption Mode（加密方式）：选择AES或TKIP。
   - WEP Key（Wi-Fi密码）：设置Wi-Fi访问密码。
   - Bandwidth（无线带宽）：选择20MHz（适用于复杂环境，抗干扰能力强）或40MHz/80MHz（适用于高速传输，要求较低干扰）。
   - Stations Limit（最大客户端数）：设置允许连接到AP的最大客户端数量。
   - IP Address（IP地址）：设置Wi-Fi AP模式下的本地IP地址，默认`192.168.100.100`。
   - Subnet Mask（子网掩码）：设置子网划分参数，默认`255.255.255.0`。
3. 保存配置，等待AP生效后，其他设备即可搜索并连接该Wi-Fi热点。

区域规则参数说明：
- **入站规则**：针对从该域收到且目标地址为设备本身的报文的控制策略，可选`接受`、`拒绝`或`丢弃`。
- **出站规则**：针对设备本身产生且通过该域发送出去的报文的控制策略，可选`接受`、`拒绝`或`丢弃`。
- **转发策略**：针对从该域收到且需要被转发的报文的控制策略，可选`接受`、`拒绝`或`丢弃`。

4. 进入【网络】→【防火墙】→【端口转发】。

**DNAT规则**

DNAT（Destination NAT，可以理解为：将外部访问的目标地址转换为内网地址的端口转发技术）规则通常用于报文从广域网方向进入、访问局域网设备时使用。

<p align="center"><img src="./img/59Hs2LEPhAS7d-b_/1754363282958-87361cea-3698-450d-a853-6dadbb2a9fe8-040142.webp" alt="DNAT规则配置"></p>

DNAT规则参数说明：
- **DNAT规则名称**：用于标识规则名称，可说明规则用途。
- **状态**：规则添加后可启用或禁用；Status为off时禁用该规则。
- **协议**：支持IP/UDP/TCP等协议类型。
- **源区域**：标识设备从哪个域（即哪个接口）收到报文。
- **源IP地址**：标识报文的源IP地址；不填则匹配所有报文。
- **内部IP地址**：标识需要访问的目标设备IP地址，通常为局域网内设备（如HTTP服务器）。
- **外部IP地址**：当一个域有多个IP地址时，用于标识源区域对应接口的IP地址；接口有多个IP地址时需选择一个。

> **示例**：当从`wan`域收到源地址为10.5.30.198、目的地址为192.168.3.100的报文时，将目的地址修改为`192.168.6.6`。

**SNAT规则**

SNAT（Source NAT，可以理解为：将内网设备发出的报文源地址转换为公网地址的技术）规则通常用于报文从局域网进入、需要将报文源IP地址修改为WAN口IP地址时使用。

<p align="center"><strong>图 3-16 SNAT规则配置</strong></p>

SNAT规则参数说明：
- **SNAT规则名称**：用于标识规则名称，可说明规则用途。
- **状态**：规则添加后可启用或禁用；Status为off时禁用该规则。
- **协议**：支持IP/UDP/TCP等协议类型。
- **源区域**：报文来自哪个域（即哪个接口），通常为局域网域。
- **源IP地址**：匹配报文的源IP地址；可不填写，即匹配所有源IP地址。
- **目标区域**：报文从哪个域发出去，通常为广域网域。
- **目标IP**：报文的目的IP地址；可不填写，即匹配所有报文。
- **SNAT IP地址**：将报文源IP修改后的IP地址。

> **示例**：将从`lan`域（eth2）进入、路由到`wan`域（eth1）发送的报文源IP地址，修改为eth1接口的IP地址（即10.5.30.193）。

通信规则参数说明：
- **通信规则名称**：用于标识规则，可描述规则作用。

**5.1 配置OpenVPN服务器**

**目标**：在EC300上搭建OpenVPN服务器，供远程客户端安全接入。

<p align="center"><img src="./img/59Hs2LEPhAS7d-b_/1750733717375-12c081d8-bf83-4314-bc06-98abab122f76-453049.webp" alt="OpenVPN服务器手动配置页面"></p>
<p align="center"><strong>图 3-23 OpenVPN服务器手动配置</strong></p>

4. 配置基础参数：
   - **服务端名称**：唯一标识服务器端配置的字符串，只能包含英文字母和数字，并以字母开头，长度不超过64个字符。
   - **状态**：开启或关闭，用于控制OpenVPN服务器的启动或停止。
   - **接口类型**：目前只支持TUN模式。
   - **网络类型**：支持`p2p`、`net30`和`subnet`。`p2p`和`net30`用于建立点对点的连接，`subnet`用于建立虚拟局域网。
   - **虚拟网络**：`subnet`模式下，需要配置虚拟网络的网段信息；如果是`p2p`或`net30`，配置的则是本端IP地址和远端IP地址。
   - **网络掩码**：`subnet`模式下，需要配置虚拟网络的子网掩码信息。
   - **CA证书**：导入CA证书（Certificate Authority，可以理解为：用于验证通信双方身份可信度的根证书）文件。
   - **公钥证书**：导入服务器的公钥证书文件。
   - **私钥证书**：导入服务器的私钥证书文件。
   - **DH参数**：导入服务器的DH参数（Diffie-Hellman，用于密钥交换的数学参数）文件。
5. 展开高级设置，配置以下参数：
   - **服务器地址**：OpenVPN监听的IP地址，默认监听`0.0.0.0`，即监听所有IP地址。
   - **协议类型**：支持`udp`和`tcp`，默认`udp`。
   - **服务器端口**：服务器监听的端口号，默认`1194`。
   - **加密算法**：配置用于保证数据传输机密性的加密算法。
   - **哈希算法**：配置用于保证数据传输完整性的哈希算法。
   - **密钥保持**：启用密钥保持后，即使OpenVPN因网络异常出现短暂断开也能快速恢复，无需重新握手。
   - **隧道保持**：启用隧道保持后，可以防止隧道因短暂网络波动而关闭。
   - **LZO压缩**：是否启用LZO压缩（一种数据压缩算法）。
   - **客户端最大数量**：配置同时连接OpenVPN Server的客户端数量。
   - **用户**：指定运行OpenVPN进程的用户。
   - **用户组**：指定运行OpenVPN进程的用户组。
   - **日志级别**：取值范围`1~11`，数值越大，日志信息越详细；可以在日志页面，`ALL`类型日志中查看OpenVPN日志。
   - **连接检测时间间隔**：发送心跳包检测VPN（Virtual Private Network，可以理解为：在公共网络上建立的加密专用通信通道）连接是否存活的周期。
   - **连接检测超时时间**：如果在超时时间内未收到对方响应，判定连接已断开并尝试重启。
   - **允许客户端到客户端的通信**：是否允许客户端和客户端相互通信。
   - **推送到客户端**：可以推送路由和DNS等信息到客户端。
6. 点击保存，完成OpenVPN服务器配置。

## 场景7：Linux命令行网络管理

**7.1 关闭IEOS并使用Linux原生网络管理**

**目标**：关闭IEOS系统程序，改用Linux原生命令行进行网络配置。

**前提**：已通过SSH登录到EC300命令行界面。

**预计用时**：约5分钟。

**操作步骤**：
1. 停止IEOS守护进程：
   ```shell
   systemctl stop ieos_daemon
   ```
2. 禁止IEOS开机自启动（可选，若需长期关闭）：
   ```shell
   systemctl disable ieos_daemon
   ```

**说明**：关闭IEOS后，拨号、Wi-Fi等无线联网功能需通过Linux原生命令实现，且无法通过DeviceLive平台对设备进行远程管理。

**7.2 配置静态IP地址**

**目标**：通过修改网络接口配置文件，为以太网接口设置静态IP地址。

**前提**：IEOS已关闭，已登录EC300命令行界面。

**预计用时**：约5分钟。

**操作步骤**：
1. 使用vim编辑器打开目标接口的配置文件（以eth2为例）：
   ```shell
   vim /etc/network/interfaces.d/eth2
   ```

## 4.2 基于IEOS的web管理

IEOS（InHand Edge Operating System，可以理解为：映翰通自研的一套运行在Linux系统上的网络管理和系统管理程序）提供Web界面，用于对EC300设备进行网络管理和系统管理。通过IEOS Web界面，用户可进行以太网口IP地址、蜂窝拨号、DHCP Client/Server、静态路由、防火墙等网络配置，也可对系统时间、时区、固件升级和系统重启等进行操作。

IEOS采用状态和配置分离的设计方案，分为网络管理、系统管理和状态三大功能板块。网络管理菜单和系统管理菜单下只能进行网络和系统的相关配置，状态信息需统一到状态页面查看。

> **注意**：使用IEOS程序管理网络配置和系统配置时，若同时使用Linux原生命令，二者可能会相互影响，导致异常的运行状态。建议IEOS支持的配置均通过IEOS Web实现管理；IEOS不支持的配置（如VPN），可结合Linux原生命令实现配置目标。

设备出厂默认启用IEOS。若用户需要基于Linux原生命令行进行网络管理和系统管理，需先关闭IEOS。

#### 4.2.2.3 Wi-Fi

**功能说明**

本设备支持两种Wi-Fi工作模式：

- **Station模式（STA模式）**：设备作为客户端连接其他Wi-Fi热点，适用于通过现有无线网络接入互联网的场景。
- **Access Point模式（AP模式）**：设备自身发射Wi-Fi信号供其他终端设备连接，适用于为周边设备提供无线接入服务的场景。

用户可根据实际网络环境选择其中一种模式进行配置，两种模式不可同时启用。

**入口**

进入【网络】→【Wi-Fi】菜单，选择对应的工作模式标签页进行配置。

**操作说明**

**Station模式配置**

1. 进入【网络】→【Wi-Fi】→【Station】页面。
2. 将"Enable Wi-Fi"开关置为开启状态。
3. 在"Client SSID"字段输入待连接的Wi-Fi网络名称（SSID），或点击扫描按钮获取附近可用的SSID列表后选择。
4. 根据目标网络的认证要求，在"Auth Method"下拉框中选择认证方式：Open（无认证）、WPA-PSK、WPA2-PSK或WPA-PSK/WPA2-PSK Mixed。
5. 在"Encrypt Mode"下拉框中选择加密方式：CCMP、TKIP或TKIP和CCMP。
6. 在"WPA/WPA2 PSK Key"字段输入对应网络的连接密码。
7. （可选）配置"Enable Default Route"选项。该选项默认开启，开启后当Wi-Fi连接成功时，系统将自动添加一条wlan口的默认路由。
8. （可选）如需调整路由优先级，在"Metric"字段输入度量值。当蜂窝口、Wi-Fi口和以太网口均配置默认路由时，度量值最小的路由优先生效。
9. 点击"保存"按钮使配置生效。

**Access Point模式配置**

1. 进入【网络】→【Wi-Fi】→【Access Point】页面。
2. 在"SSID"字段设置Wi-Fi网络的名称（SSID）。
3. 在"SSID Broadcast"选项中选择是否广播SSID。默认开启，开启时附近设备可扫描到该SSID；关闭时需手动输入SSID进行连接。
4. 在"Band"下拉框中选择无线频段：2.4G或5.8G。
5. 在"Radio Type"下拉框中选择射频类型：
   - 802.11a/n/ac（适用于5G频段）
   - 802.11b/g/n（适用于2.4G频段）
6. 在"Channel"下拉框中选择无线信道，可手动指定或使用默认通道。
7. 在"Auth Method"下拉框中选择认证方式：Open、WPA-PSK、WPA2-PSK或WPA-PSK/WPA2-PSK Mixed。
8. 在"Encryption Mode"下拉框中选择加密方式：AES或TKIP。
9. 在"WEP Key"字段设置Wi-Fi访问密码。

13. 在"Subnet Mask"字段配置子网掩码，默认值为`255.255.255.0`。
14. 点击"保存"按钮使配置生效。

## 4.3 基于Linux命令行管理

### 4.3.1 功能说明

#### 4.3.5.6 使用localtime文件

#### 4.4.3 Micro SD卡

EC300支持micro SD存储卡，不支持热插拔。SD卡分区需格式化为FAT32格式，格式化方法如下：

- SD卡容量 <= 32GB：将SD卡插入读卡器，再将读卡器插入Windows系统电脑进行格式化。大部分Windows系统仅支持将容量小于32GB的SD卡格式化为FAT32文件系统，格式化时会自动创建一个分区；也可在Linux系统下格式化为FAT32文件系统，方法与SD卡容量 > 32GB时相同。
- SD卡容量 > 32GB：需在Linux系统下格式化为FAT32格式。将SD卡插入读卡器，再将读卡器插入Linux系统电脑或安装Linux虚拟机的电脑。

格式化步骤：

1. 执行`fdisk -l`查看SD卡对应的设备节点，示例中为`/dev/sda`。
2. 执行`fdisk /dev/sda`进行分区（`p`查看当前分区，`d`删除已有分区，`n`新建分区，`w`保存更改），创建分区`/dev/sda1`。
3. 执行`sudo mkfs.vfat -F 32 /dev/sda1`，将分区格式化为FAT32格式。
4. 格式化完成后，将SD卡从读卡器取下，在设备断电状态下插入设备。

挂载操作参考Linux mount手册。

## 4.5 安全功能

EC300 集成多种安全机制，用于保障设备运行及数据访问的安全性。

### 4.5.1 功能说明

EC300 的安全机制主要包括以下三方面：

- **sudo 机制**：用于权限管理。根用户（root）被禁用，普通用户通过 sudo 获取执行特权命令的权限，从而降低因误操作导致系统损害的风险。
- **防火墙**：基于 Netfilter/iptables 的包过滤防火墙工具，可对流入、流出及流经设备的数据包进行精细化控制。
- **TPM 2.0**：可信平台模块（Trusted Platform Module），一种硬件安全模块，用于提供系统的安全与加密功能。EC300 已集成标准的 TPM 2.0 协议栈及相关工具。

### 4.5.2 入口

sudo 机制、防火墙及 TPM 2.0 均通过命令行界面进行配置与管理。用户需通过 SSH 或本地终端登录设备后执行相关命令。

### 4.5.3 操作说明

#### 4.5.3.1 sudo机制

sudo 允许经授权的用户以根用户或其他用户身份执行命令。使用该机制时，系统会提示输入当前用户的密码，无需知晓根用户密码即可获取相应特权。日常操作中，用户大部分时间以普通用户身份工作，仅在执行需要特权的命令时通过 sudo 提升权限。

#### 4.5.3.2 防火墙

iptables 作为 Linux 系统自带的包过滤防火墙工具，支持对数据包进行过滤、NAT 转换及流量控制等操作。用户可通过命令行配置规则，实现对进出设备网络流量的精细化管理。

#### 4.5.3.3 TPM2.0

TPM 2.0 是一种硬件安全模块，内置加密协处理器，支持存储加密密钥、数字证书及其他安全数据，并提供多种加密算法与安全协议。EC300 已集成标准的 TPM 2.0 协议栈和 TPM 2.0 工具，用户可通过命令行调用相关工具进行密钥管理、数据加密等操作。

### 4.5.4 参数说明

| 安全机制 | 说明 |
|---------|------|
| sudo 机制 | 基于最小权限原则，授权普通用户执行特权命令，降低误操作风险 |
| 防火墙（iptables） | 基于包过滤的防火墙，支持流入、流出及转发数据包的精细化控制 |
| TPM 2.0 | 硬件安全模块，支持加密密钥存储、数字证书管理及多种加密算法 |

### 4.6.1 IO编程指南

设备配备8个IO接口：DI1~DI4为4路数字输入管脚，DO1~DO4为4路数字输出管脚。

通过设备描述信息文件可获取各IO对应的设备节点路径。对IO接口进行编程时，直接操作对应设备节点下的value文件（路径格式为`/sys/class/gpio/gpioxxx/value`）。

**操作示例**

DO1输出高电平：

```shell
echo 1 > /sys/class/gpio/gpio323/value
```

查看DI1电平状态：

```shell
cat /sys/class/gpio/gpio454/value
```

**完整Shell脚本示例**

```shell
#!/bin/bash

gpio323="/sys/class/gpio/gpio323/value"
gpio453="/sys/class/gpio/gpio453/value"
gpio465="/sys/class/gpio/gpio465/value"
gpio461="/sys/class/gpio/gpio461/value"
# DO1输出高电平
if [ -f "$gpio323" ]; then
   echo 1 > /sys/class/gpio/gpio323/value
else
   echo "no file exit "$gpio323
fi

# DO1输出低电平
if [ -f "$gpio323" ]; then
   echo 0 > $gpio323
else
   echo "no file exit "$gpio323
fi

gpio454="/sys/class/gpio/gpio454/value"
gpio455="/sys/class/gpio/gpio455/value"
gpio456="/sys/class/gpio/gpio456/value"
gpio457="/sys/class/gpio/gpio457/value"
# 查看DI1电平
if [ -f "$gpio454" ]; then
   cat $gpio454
else
   echo "no file exit "$gpio454
fi
```

### 4.6.2 LED编程指南

设备提供USER1、USER2、USER3、USER4四个用户LED，可用于状态提示。LED位置以设备面板丝印为准。

通过设备描述信息文件可获取各LED对应的设备节点路径：

| LED名称 | 设备节点路径 |
|---------|-------------|
| user1 | /sys/class/leds/user1 |
| user2 | /sys/class/leds/user2 |
| user3 | /sys/class/leds/user3 |
| user4 | /sys/class/leds/user4 |

各LED设备节点目录下包含以下控制文件：

| 控制文件 | 功能说明 |
|---------|---------|
| brightness | 控制LED亮灭。写入1为常亮，写入0为常灭 |
| trigger | LED触发器。写入timer表示定时器触发，写入none表示取消触发器 |
| delay_on | LED点亮持续时间，单位为ms |
| delay_off | LED熄灭持续时间，单位为ms |

> **注意**：当trigger配置为timer时，brightness中的值将不再生效，自动变为0。

将文件路径中的user1替换为user2、user3或user4，即可控制对应LED。

**操作示例**

USER1常亮：

```shell
echo 1 > /sys/class/leds/user1/brightness
```

USER1闪烁（亮1秒、灭1秒）：

```shell
# 启动定时器
echo timer > /sys/class/leds/user1/trigger

# 亮1000ms
echo 1000 > /sys/class/leds/user1/delay_on

# 灭1000ms
echo 1000 > /sys/class/leds/user1/delay_off
```

**完整Shell脚本示例**

```shell
#!/bin/bash

USER1_BRIGTHNESS="/sys/class/leds/user1/brightness"
USER1_TRIGGER="/sys/class/leds/user1/trigger"
USER1_DELAY_ON="/sys/class/leds/user1/delay_on"
USER1_DELAY_OFF="/sys/class/leds/user1/delay_off"

# 点亮USER1灯
if [ -f "$USER1_BRIGTHNESS" ]; then
   echo 1 > $USER1_BRIGTHNESS
else
   echo "no file exit "$USER1_BRIGTHNESS
fi

if [ -f "$USER1_TRIGGER" ]; then
   echo timer > $USER1_TRIGGER
else
   echo "no file exit "$USER1_TRIGGER
fi

# 设置USER1灯亮1000ms
if [ -f "$USER1_DELAY_ON" ]; then
   echo 1000 > $USER1_DELAY_ON
else
   echo "no file exit "$USER1_DELAY_ON
fi

# 设置USER1灯灭1000ms
if [ -f "$USER1_DELAY_OFF" ]; then
   echo 1000 > $USER1_DELAY_OFF
else
   echo "no file exit "$USER1_DELAY_OFF
fi

# 关闭USER1闪烁
if [ -f "$USER1_TRIGGER" ]; then
   echo none > $USER1_TRIGGER
else
   echo "no file exit "$USER1_TRIGGER
fi
```

**交叉编译工具链**：gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu.tar.gz

**环境变量配置步骤**

2. 编辑`~/.bashrc`文件，在文件末尾添加：`PATH=$PATH:/opt/gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu/bin`
3. 执行`source ~/.bashrc`使环境变量在当前终端生效；新打开的终端自动生效。

**编译示例**

以Hello World程序为例，创建目录和文件：

```shell
mkdir ~/example
touch ~/example/hello.c
touch ~/example/Makefile
```

`~/example/hello.c`文件内容：

```c
#include <stdio.h>

int main(void) 
{
    printf("hello, world!\n");
    return 0;
}
```

`~/example/Makefile`文件内容：

```makefile
# 定义目标文件名和源文件名
TARGET  := hellworld
DIRS    := $(shell find . -maxdepth 3 -type d) 
SRCS    := $(foreach dir,$(DIRS),$(wildcard $(dir)/*.c))
OBJS    := $(SRCS:.c=.o)


CC=aarch64-linux-gnu-gcc

# 定义编译器和编译选项
CFLAGS  := -Wall -Wextra -g -Wno-unused-parameter

# 定义默认目标
all: $(TARGET)

# 定义目标文件的依赖关系和编译命令
$(TARGET): $(OBJS)
       $(CC) $(CFLAGS) $(LIBS) $^ -o $@ 

# 定义将源文件编译成目标文件的命令
%.o: %.c
       $(CC) $(CFLAGS) $(LIBS) -c $< -o $@

# 定义清除临时文件的命令
clean:
       rm -f $(TARGET) $(OBJS)

# 声明伪目标 ".PHONY"
.PHONY: all clean
```

在`~/example`目录下执行`make`，生成目标文件`helloworld`。

### 4.6.4 Watchdog编程

设备配备外部硬件看门狗，若60秒内未执行喂狗操作，设备将自动重启。

可通过命令`systemctl stop watchdog`禁用系统喂狗服务，在用户程序中自行实现喂狗逻辑。

**Watchdog程序示例**

```c
#include <fcntl.h>
#include <unistd.h>
#include <stdlib.h>
#include <stdio.h>
#include <string.h>
#include <signal.h>
#include <sys/types.h>
#include <sys/stat.h>

static int fd = 0;

/* 信号处理函数 */
void signal_handler(int signal)
{
    /* 关闭watchdog，关闭后如果不喂狗设备60秒后会重启 */
    write(fd, "V", 1);
    close(fd);
    exit(0);
}

int main(int argc, char **argv)
{
    /* 通过open来打开watchdog */
    fd = open("/dev/watchdog0", O_WRONLY);

    /* 注册Kill信号 */
    signal(SIGTERM, signal_handler);
    
    while (1) {
        /* 通过write来喂狗 */
        write(fd, "\0", 1); 

        /* 每10秒执行一次喂狗 */
        sleep(10);
    }
}
```

### 2 IO 编程指南

EC300 提供 8 个 IO 接口：DI1~DI4 为 4 个输入管脚，DO1~DO4 为 4 个输出管脚。可通过查询 `/tmp/ieos/etc/system_info.json` 获取各 IO 对应的设备节点。

对 IO 接口进行编程时，直接操作 sysfs（Linux 系统中用于暴露内核对象信息的虚拟文件系统）中对应设备节点下的 `value` 文件即可。例如：

- DO1 输出高电平：

```shell
echo 1 > /sys/class/gpio/gpio323/value
```

- 读取 DI1 电平：

```shell
cat /sys/class/gpio/gpio454/value
```

完整 Shell 脚本示例：

```shell
#!/bin/bash

gpio323="/sys/class/gpio/gpio323/value"
gpio453="/sys/class/gpio/gpio453/value"
gpio465="/sys/class/gpio/gpio465/value"
gpio461="/sys/class/gpio/gpio461/value"

if [ -f "$gpio323" ]; then
   echo 1 > /sys/class/gpio/gpio323/value
else
   echo "no file exit "$gpio323
fi

if [ -f "$gpio323" ]; then
   echo 0 > $gpio323
else
   echo "no file exit "$gpio323
fi

gpio454="/sys/class/gpio/gpio454/value"
gpio455="/sys/class/gpio/gpio455/value"
gpio456="/sys/class/gpio/gpio456/value"
gpio457="/sys/class/gpio/gpio457/value"

if [ -f "$gpio454" ]; then
   cat $gpio454
else
   echo "no file exit "$gpio454
fi
```

### 3 LED 编程指南

EC300 提供 USER1、USER2、USER3、USER4 四个用户可编程 LED，用于状态提示。各 LED 对应的 sysfs 设备节点如下（以 USER1 为例）：

- `/sys/class/leds/user1/brightness`：控制 LED 亮灭，写入 1 为常亮，写入 0 为常灭。
- `/sys/class/leds/user1/trigger`：LED 触发器，写入 `timer` 启用定时器触发，写入 `none` 取消触发器。
- `/sys/class/leds/user1/delay_on`：LED 点亮持续时间，单位为 ms。
- `/sys/class/leds/user1/delay_off`：LED 熄灭持续时间，单位为 ms。

若配置 `trigger` 为 `timer`，则 `brightness` 中的值将不再生效，会自动变为 0。将路径中的 `user1` 替换为 `user2`、`user3` 或 `user4`，即可控制对应 LED。

示例：

- USER1 常亮：

```shell
echo 1 > /sys/class/leds/user1/brightness
```

- USER1 闪烁（亮 1 秒，灭 1 秒）：

```shell
echo timer > /sys/class/leds/user1/trigger
echo 1000 > /sys/class/leds/user1/delay_on
echo 1000 > /sys/class/leds/user1/delay_off
```

完整 Shell 脚本示例：

```shell
#!/bin/bash

USER1_BRIGTHNESS="/sys/class/leds/user1/brightness"
USER1_TRIGGER="/sys/class/leds/user1/trigger"
USER1_DELAY_ON="/sys/class/leds/user1/delay_on"
USER1_DELAY_OFF="/sys/class/leds/user1/delay_off"

if [ -f "$USER1_BRIGTHNESS" ]; then
   echo 1 > $USER1_BRIGTHNESS
else
   echo "no file exit "$USER1_BRIGTHNESS
fi

if [ -f "$USER1_TRIGGER" ]; then
   echo timer > $USER1_TRIGGER
else
   echo "no file exit "$USER1_TRIGGER
fi

if [ -f "$USER1_DELAY_ON" ]; then
   echo 1000 > $USER1_DELAY_ON
else
   echo "no file exit "$USER1_DELAY_ON
fi

if [ -f "$USER1_DELAY_OFF" ]; then
   echo 1000 > $USER1_DELAY_OFF
else
   echo "no file exit "$USER1_DELAY_OFF
fi

if [ -f "$USER1_TRIGGER" ]; then
   echo none > $USER1_TRIGGER
else
   echo "no file exit "$USER1_TRIGGER
fi
```

### 4 交叉编译指南

用户自己的 C/C++ 程序可以通过在开发机器上使用交叉编译工具链进行交叉编译（可以理解为：在一种架构的计算机上编译出另一种架构可执行程序的过程），然后将目标文件上传到 EC300 设备上执行。

交叉编译工具压缩包：

```
gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu.tar.gz
```

#### 配置交叉编译工具链环境变量

1. 在开发机上将 `gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu.tar.gz` 解压到 `/opt` 路径下（也可以解压到其他路径，步骤 2 中设置 PATH 环境变量时做相应调整）。
2. 编辑 `~/.bashrc` 文件，在文件最后添加一行：

```shell
PATH=$PATH:/opt/gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu/bin
```

3. 执行 `source ~/.bashrc` 使环境变量在当前终端生效；新打开的终端会自动生效。

#### 编译示例

以经典的 hello world 程序为例，创建如下目录和文件：

```shell
mkdir ~/example
touch ~/example/hello.c
touch ~/example/Makefile
```

`~/example/hello.c` 文件内容：

```c
#include <stdio.h>

int main(void)
{
    printf("hello, world!\n");
    return 0;
}
```

`~/example/Makefile` 文件内容：

```makefile
TARGET  := hellworld
DIRS    := $(shell find . -maxdepth 3 -type d)
SRCS    := $(foreach dir,$(DIRS),$(wildcard $(dir)/*.c))
OBJS    := $(SRCS:.c=.o)

CC=aarch64-linux-gnu-gcc

CFLAGS  := -Wall -Wextra -g -Wno-unused-parameter

all: $(TARGET)

$(TARGET): $(OBJS)
	$(CC) $(CFLAGS) $(LIBS) $^ -o $@

%.o: %.c
	$(CC) $(CFLAGS) $(LIBS) -c $< -o $@

clean:
	rm -f $(TARGET) $(OBJS)

.PHONY: all clean
```

在 `~/example` 目录下执行 `make`，生成目标文件 `helloworld`。

### 5 Watchdog 编程

EC300 配备一个外部硬件 Watchdog（看门狗，可以理解为：用于监测系统运行状态并在异常时自动重启的硬件机制）。若 60 秒没有喂狗，设备将自动重启。

可通过以下命令禁用系统的喂狗服务，以便在用户程序中自行实现喂狗：

```shell
systemctl stop watchdog
```

Watchdog 编程示例（C 语言）：

```c
#include <fcntl.h>
#include <unistd.h>
#include <stdlib.h>
#include <stdio.h>
#include <string.h>
#include <signal.h>
#include <sys/types.h>
#include <sys/stat.h>

static int fd = 0;

/* 信号处理函数 */
void signal_handler(int signal)
{
    /* 关闭watchdog，关闭后如果不喂狗设备60秒后会重启 */
    write(fd, "V", 1);
    close(fd);
    exit(0);
}

int main(int argc, char **argv)
{
    /* 通过open来打开watchdog */
    fd = open("/dev/watchdog0", O_WRONLY);

    /* 注册Kill信号 */
    signal(SIGTERM, signal_handler);

    while (1) {
        /* 通过write来喂狗 */
        write(fd, "\0", 1);

        /* 每10秒执行一次喂狗 */
        sleep(10);
    }
}
```
