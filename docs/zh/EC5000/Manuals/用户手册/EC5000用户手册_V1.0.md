## 第1章 认识设备

### 1.1 概述

EC5000系列边缘AI计算机搭载NVIDIA Jetson Orin NX或Orin Nano模组，面向边缘AI运算应用场景设计。系统提供2个千兆以太网接口、1个HDMI视频输出接口、6个USB 3.2接口、2个RS-232/422/485可切换串行接口、2个GMSL 2.0视频接口、4路隔离式数字量输入与4路隔离式数字量输出、1路CAN FD接口、2个SIM卡槽，并支持4G/5G蜂窝网络、Wi-Fi/蓝牙无线通信以及TPM 2.0安全模块。该设备适用于工业视觉检测、智能交通、边缘推理等需本地AI算力的场景。

#### 1.3.11 Recovery轻触开关按键

EC5000可拆卸挡板下有Recovery轻触开关按键。在系统正常运行时按住Recovery 10秒，等待系统状态灯由闪烁变为常亮后松开，设备将进入系统重置状态（恢复为系统出厂状态）。在系统上电前或系统运行中按住Recovery按键，并重启系统（按Reset重启、reboot重启或掉电重启），则系统进入烧录模式。

#### 1.3.12 Reset轻触开关按键

EC5000可拆卸挡板下有Reset轻触开关按键。在系统正常运行时按住Reset 3秒内松开，则系统重启并使能硬件看门狗；按住Reset 3秒后松开，则系统重启并禁用硬件看门狗。在系统烧录前需禁用硬件看门狗，防止硬件看门狗重启系统导致烧写失败。

### 1.5 恢复出厂设置

EC5000可通过以下方式恢复出厂设置：

1. **Recovery按键重置**：在系统正常运行时按住Recovery按键10秒，等待系统状态灯由闪烁变为常亮后松开，设备将进入系统重置状态（恢复为系统出厂状态）。也可在上电后系统启动期间，状态灯快速闪烁3秒时按住Recovery键，持续按住约5秒，直到状态灯长亮后5秒内松开，设备进入系统重置状态。
2. **命令行重置**：打开Terminal，执行命令 `sudo update reset` 进行系统重置。

> **注意**：系统重置会将系统的配置状态和文件系统状态恢复到出厂状态，用户安装的软件也会被清除。此功能默认启用，可通过命令 `sudo update resetkey <enable/disable>` 进行控制。

## 第3章 常用场景配置

### 场景1：首次启动与系统登录

**目标**：完成设备上电启动，并通过HDMI或SSH方式登录系统。

**前提**：设备已完成物理安装，电源、接地、天线（如使用蜂窝/Wi-Fi/GNSS）已正确连接。

**预计用时**：约10分钟。

**操作步骤**：

1. 确认电源电压在9~36 VDC范围内，给设备上电。
2. 观察前面板指示灯：电源指示灯红色常亮，系统状态灯绿色以1Hz频率闪烁，表示系统运行正常。
3. **HDMI方式登录**：
   1. 通过HDMI 2.0接口连接显示器，将键盘鼠标接入USB 3.2接口。
   2. 等待系统启动完成后，在登录界面选择System User对应账户，输入密码后登录。
4. **SSH方式登录**：
   1. 将PC与设备以太网接口连接，配置PC与设备默认IP处于同一网段。
   2. 打开SSH终端工具，输入设备IP地址，按提示输入默认账户和密码完成登录。

**验证方法**：

1. 登录成功后，桌面或终端可正常操作。
2. 在Terminal执行命令 `sudo ecversion`，可查询到系统版本信息。

**常见问题**：

- 无法看到登录界面：检查HDMI连接是否牢固，显示器是否切换至正确输入源。
- SSH连接超时：检查PC与设备是否在同一网段，确认设备IP地址正确。

### 场景2：配置以太网连接

**目标**：通过IEOS Web管理界面配置以太网接口，实现设备与网络的连接。

**前提**：已通过网线将设备与PC或上级网络连接，且可访问IEOS Web管理界面。

**预计用时**：约5分钟。

**操作步骤**：

1. 登录IEOS Web管理界面，进入【网络】->【以太网】。
2. 如需配置静态IP地址：
   1. 选择对应以太网接口，点击"编辑"按钮。
   2. 将IPv4 Method设置为Static，填写Address、Netmask、Gateway及DNS。
   3. 点击"保存"使配置生效。
3. 如需配置DHCP Client：
   1. 选择对应以太网接口，将IPv4 Method设置为DHCP。
   2. 点击"保存"使配置生效。
4. 如需在接口上启动DHCP Server功能：
   1. 进入对应接口配置页面，启用DHCP Server。
   2. 配置Starting Address、Max Address Number及Lease period。
   3. 点击"保存"。

**验证方法**：

1. 在IEOS【状态】->【设备信息】页面查看网络接口IP地址是否获取正确。
2. 通过Terminal执行 `ping <网关地址>`，确认网络层连通性。

**常见问题**：

- 配置保存后未生效：部分配置需重启网络接口或重启系统生效。
- 无法访问Web界面：确认PC与设备eth2接口处于同一网段，且使用HTTPS协议及9100端口。

### 场景3：配置蜂窝网络联网

**目标**：通过4G/5G蜂窝网络接入互联网。

**前提**：已插入SIM卡并安装蜂窝天线，设备已上电，IEOS Web管理界面可正常访问。

**预计用时**：约10分钟。

**操作步骤**：

1. 将SIM卡插入SIM1/SIM2卡槽，安装蜂窝天线。
2. 登录IEOS Web管理界面，进入【网络】->【蜂窝】。
3. 在基本配置中确认Enabled开关为开启状态。
4. 如使用专网卡，需在Profiles中添加拨号参数集（APN、用户名、密码、认证方式），并在APN Profile中选择对应参数集Index。
5. 配置Network Mode（通常选择"自动"即可）。
6. 确认Enable Default Route已开启，以便拨号成功后添加默认路由。
7. （建议）开启ICMP探测功能以检测拨号假连接：
   1. 配置ICMP Detection Server Probes（至少1个可达地址）。
   2. 设置Detection Interval、Detection Timeout及Detection Max Retries。
   3. 保存配置。
8. 如需使用双卡单拨功能，开启Dual SIM Enabled，配置Main SIM及Max Number of Dials。
9. 点击"保存"，等待拨号成功。

**验证方法**：

1. 在IEOS【状态】->【蜂窝拨号状态信息】页面查看IMEI、IMSI、信号强度及获取的IP地址。
2. 通过Terminal执行 `ping 8.8.8.8`，确认蜂窝网络连通性。

**常见问题**：

- 拨号失败无信号：检查SIM卡是否正确插入、天线是否连接、APN参数是否与运营商一致。
- 拨号成功但无法上网：可能是拨号假连接，开启ICMP探测功能后可自动恢复。
- 双卡切换异常：确认两张SIM卡均可用，且Max Number of Dials设置合理。

### 场景4：配置Wi-Fi Station联网

**目标**：通过Wi-Fi接入现有无线网络。

**前提**：设备配备Wi-Fi模组，且可访问IEOS Web管理界面。

**预计用时**：约5分钟。

**操作步骤**：

1. 登录IEOS Web管理界面，进入【网络】->【Wi-Fi Station】。
2. 开启Enable Wi-Fi开关。
3. 在Client SSID中输入需要连接的无线网络名称，或通过"扫描"按钮获取附近可用SSID。
4. 选择Auth Method（如WPA2-PSK）及Encrypt Mode（如CCMP）。
5. 输入WPA/WPA2 PSK Key。
6. 确认Enable Default Route按需开启，并设置合理的Metric值。
7. 点击"保存"，等待Wi-Fi连接建立。

**验证方法**：

1. 在IEOS【状态】->【Wi-Fi Station状态信息】页面查看获取的IP地址、网关及DNS。
2. 通过Terminal执行 `ping <网关地址>` 或访问互联网，确认连通性。

**常见问题**：

- 扫描不到目标SSID：检查Wi-Fi天线是否安装，确认目标网络在设备信号覆盖范围内。
- 连接失败：核对SSID、密码及认证加密方式是否与目标网络一致。

### 场景5：配置IEOS Web管理

**目标**：通过IEOS Web界面完成设备基础系统管理配置（云管理、时区、固件升级）。

**前提**：可正常访问IEOS Web管理界面。

**预计用时**：约15分钟。

**操作步骤**：

1. **配置云管理（DeviceLive）**：
   1. 进入【系统管理】->【基础配置】->【云管理】。
   2. 开启Enabled开关，选择Cloud Server（国内平台或海外平台）。
   3. 点击"保存"。
2. **配置时区与NTP**：
   1. 进入【系统管理】->【基础配置】->【时区和NTP client】。
   2. 选择正确的时区，配置NTP Server地址（最多10个）。
   3. 点击"保存"。
3. **固件升级**：
   1. 进入【系统管理】->【固件升级】。
   2. 上传升级镜像文件，点击"升级"。
   3. 如需升级后自动重启，开启自动重启选项；否则手动重启系统使升级生效。
4. **配置备份与恢复**：
   1. 进入【系统管理】->【基础配置】->【配置导入导出】。
   2. 点击"导出"备份当前配置；点击"导入"恢复已保存的配置。
   3. 点击"重置"可将系统恢复出厂（该操作会清除用户安装的软件）。

**验证方法**：

1. 在【状态】->【设备信息】页面确认固件版本、系统时间正确。
2. 若启用云管理，在DeviceLive平台查看设备是否在线。

**常见问题**：

- 固件升级失败：确认镜像文件与设备型号匹配，升级过程中勿断电。
- NTP时间不同步：检查设备网络连通性，确认NTP服务器可达。

### 场景6：配置串口与外设

**目标**：完成串口工作模式切换及数字量输入输出控制。

**前提**：已通过SSH或HDMI登录设备系统。

**预计用时**：约10分钟。

**操作步骤**：

1. **串口模式切换**：
   1. 打开Terminal。
   2. 执行命令查询或切换串口模式，例如将COM1切换为RS-485模式：
      ```
      sudo uart_mode COM1 485
      ```
   3. COM1对应设备节点 `/dev/ttyCOM1`，COM2对应 `/dev/ttyCOM2`。
2. **数字量输入读取**：
   1. DI0~DI3对应GPIO路径 `/sys/class/gpio/gpio308` ~ `/sys/class/gpio/gpio311`。
   2. 通过读取对应GPIO value文件获取输入状态。
3. **数字量输出控制**：
   1. DO0~DO3对应GPIO路径 `/sys/class/gpio/gpio312` ~ `/sys/class/gpio/gpio315`。
   2. 通过向对应GPIO value文件写入1或0控制输出状态。
4. **CAN FD配置**：
   
   1. 打开Terminal，执行命令启动CAN接口：
      ```
      sudo ip link set can0 up type can bitrate 1000000 dbitrate 5000000 restart-ms 1000 fd on
      ```

**验证方法**：

1. 串口模式切换后，通过 `ls -l /dev/ttyCOM1` 确认设备节点存在。
2. 数字量输入输出可通过读取/写入GPIO值验证。
3. CAN FD可通过 `ip link show can0` 查看接口状态为UP。

**常见问题**：

- 串口通信异常：确认串口模式（RS-232/422/485）与对端设备一致，检查RS-485拨码开关状态。
- CAN FD无法启动：确认波特率参数正确，检查CAN总线终端电阻。

### 4.2 基于IEOS的Web管理

EC5000基于Ubuntu22.04系统，因此可以使用Linux原生的命令进行网络管理和系统管理；为了便于用户配置，映翰通开发了一套IEOS系统程序，提供web界面，用户可以方便的通过web进行网络管理和系统管理，但需要注意的是，当启用IEOS功能后，IEOS会接管网络管理和系统管理，此时通过Linux原生命令进行网络管理和系统管理可能会失效；设备出厂默认启用IEOS，如果用户需要基于Linux原生的命令行进行网络管理和系统管理，需要先关闭IEOS。

IEOS是映翰通自研的一套运行在Linux系统上的网络管理和系统管理程序，IEOS提供web界面，用户可以通过web进行以太网口ip地址，蜂窝拨号，Wi-Fi Station，DHCP Client/Server，静态路由，防火墙等网络配置；也可以对系统时间，时区，固件升级和系统重启等进行操作；另外IEOS还支持对接映翰通的设备管理平台DeviceLive，用户可以通过DeviceLive平台对EC5000设备进行远程监控和管理。

IEOS采用状态和配置分离的设计方案，分为网络管理，系统管理和状态三大功能板块。网络管理菜单和系统管理菜单下只能进行网络和系统的相关配置，状态信息需要统一到状态页面查看。

重要说明：使用IEOS程序管理网络配置和系统配置的时候，如果同时使用Linux原生命令，二者可能会相互影响，导致异常的运行状态。建议IEOS支持的配置都通过IEOS web来实现管理，IEOS不支持的配置，比如VPN，可以结合Linux原生命令一起实现配置目标。

无线蜂窝网络比较复杂，有些时候会出现拨号假连接的情况，即拨号状态是成功的，但却ping不通目标地址；发生这些状况的时候，重新拨号一次就可以恢复正常。IEOS蜂窝拨号支持ICMP探测以发现假连接。建议使用蜂窝联网的客户开启ICMP探测，这样出现假连接的时候也可以快速恢复。

ICMP探测参数：

**ICMP Detection Server Probes**：ICMP探测地址；可以配置2个探测地址，只要有1个地址探测成功，则说明蜂窝没有出现假连接。2个地址都没有配置的时候，ICMP探测功能处于关闭状态。

**Detection Interval**：间隔多久进行一次ICMP探测。

**Detection Timeout**：ICMP探测超时时间，等待多久没有收到探测响应报文，则认为此次探测失败

**Detection Max Retries**：最大探测次数；当探测失败达到这个值的时候，触发重新拨号。取值范围[1,5]

**Detection Strict**：是否开启严格探测。关闭严格探测时，探测程序在每个探测周期会检测蜂窝接口接收的报文是否发生变化，如果有变化，说明蜂窝网络是通的，则不会发送ICMP报文进行探测，这样可以节省一些流量；如果开启探测，则无论蜂窝接口接收报文的数量是否发生变化，都会周期性的发送ICMP探测报文。默认是关闭的。

**Debug Mode enabled**：是否启用debug功能，开启后，日志中会增加一些拨号相关的调试信息，默认关闭。

**Enable Infinitely Redial**：开启无限重拨。某些情况下，拨号会处于异常状态，通过重启系统可以恢复正常；默认无限重拨是关闭的，当拨号失败一定次数后，会重启系统尝试恢复。由于拨号默认是开启的，有些客户在不插sim卡的情况下，拨号失败，系统会重启，针对这种情况，可以开启无限重拨；这样无论拨号失败多少次，系统都不会重启。

- MAC Address filtering

<p align="center"><img src="./img/_qlKrdiOZbCvx7xR/1754360617973-5f9d910e-84af-4dc7-93eb-23305dd28cc9-841311.webp" alt="防火墙MAC地址过滤"></p>

<p align="center"><strong>图 4-16 防火墙MAC地址过滤</strong></p>

MAC地址过滤功能用于控制设备下挂设备产生的报文是否可以被设备转发，但不会影响下挂设备对设备本身的访问。MAC地址列表中最多可以添加500条规则。

该功能包含三种模式：Unlimited, Blacklist和 Whitelist。当模式为Unlimited时，相当于该功能是关闭的，不会对MAC地址列表中的设备产生影响；当模式为Blacklist时，MAC地址列表中的设备产生的报文将被丢弃，其他MAC地址的设备产生的报文可以被正常转发；当模式为Whitelist是，只有MAC地址列表中的设备产生的报文可以被转发，其他MAC地址的设备产生的报文会被丢弃。

MAC地址列表中每一个记录都可以启用/禁用，当某个记录处于禁用状态时，该MAC地址对应的设备相当于不存在于Blacklist或 whitelist中。

**服务端名称**：唯一标识服务器端配置的字符串，只能包含英文字母和数字，并以字母开头，长度不超过64个字符。

**状态**：开启或关闭，用于控制OpenVPN服务器的启动或停止.

**接口类型**：目前只支持tun模式。

**网络类型**：支持p2p，net30和subnet。p2p和 net30用于建立点对点的连接，subnet用于建立虚拟局域网。

**虚拟网络**：subnet模式下，需要配置虚拟网络的网段信息；如果是p2p或 net30配置的则是本端IP地址和远端IP地址。

高级设置中可以配置一些改动频率比较低的参数。

**服务器地址**：OpenVPN监听的IP地址，默认监听0.0.0.0，即监听所有IP地址

**加密算法**：配置用于保证数据传输机密性的加密算法。

**哈希算法**：配置用于保证数据传输完整性的哈希算法。

**密钥保持**：启用密钥保持后，即使OpenVPN因网络异常出现短暂断开也能快速恢复，无需重新握手。

**隧道保持**：启用隧道保持后，可以防止隧道因短暂网络波动而关闭。

**LZO压缩**：是否启用LZO压缩。

**客户端最大数量**：配置同时连接OpenVPN Server的客户端数量。

**用户**：指定运行 OpenVPN 进程的用户

**用户组**：指定运行 OpenVPN 进程的用户组

**日志级别**：取值范围 1~11，数值越大，日志信息越详细；可以在日志页面，ALL类型日志中查看OpenVPN日志。

**连接检测时间间隔**：发送心跳包检测VPN连接是否存活的周期

**连接检测超时时间**：如果在超时时间内未收到对方响应，判定连接已断开并尝试重启

**允许客户端到客户端的通信**：是否允许客户端和客户端相互通信

**推送到客户端**：可以推送路由和DNS等信息到客户端。

### 4.3 Ubuntu图形界面管理

使用Linux命令行进行网络配置和系统配置的时候，首先需要关闭IEOS程序。IEOS是通过systemctl来管理的，

关闭IEOS的方式如下：

```shell
systemctl stop ieos_daemon
```

这种关闭只针对本次启动生效，系统重启后，IEOS程序依然会启动，禁止IEOS程序开机启动的方式如下：

```shell
systemctl disable ieos_daemon
```

重要说明：关闭IEOS之后，拨号，Wi-Fi等无线联网功能需要用户基于Linux原生命令来实现，而且也无法实现对接DeviceLive平台对设备进行远程管理。

## 第5章 典型应用

### 案例1：工业边缘AI视觉检测

**场景描述**：在工厂产线部署EC5000作为边缘AI推理节点，通过GMSL接口连接工业摄像头，利用NVIDIA Jetson Orin的AI算力对产线产品进行实时视觉检测和质量判断。

**设备角色**：本设备作为边缘AI推理网关，负责采集摄像头视频流、执行AI模型推理、输出检测结果，并通过蜂窝网络或以太网将结果上报至云端平台。

**配置步骤**：

1. 将GMSL摄像头连接至EC5000的GMSL接口（注意断电操作，不支持热插拔）。
2. 给设备上电，通过HDMI或SSH登录系统。
3. 执行初始化脚本 `gmsl_init.sh`，选择对应的摄像头型号。
4. 验证视频流：执行 `gmsl_play.sh 0` 播放摄像头画面。
5. 部署AI推理程序，调用NVIDIA Jetson的GPU资源进行模型推理。
6. 配置网络（以太网或蜂窝），确保设备可与云端平台通信。
7. （可选）配置IEOS云管理，接入DeviceLive平台进行远程监控。

### 案例2：工业数据采集与远程管理

**场景描述**：在工业现场部署EC5000，通过串口（RS-485/232）连接PLC或传感器，通过数字量输入输出采集开关量信号，利用CAN FD接入车载或工业控制总线，并通过4G/5G蜂窝网络将数据回传至远程监控中心。

**设备角色**：本设备作为边缘数据采集网关，负责汇集现场串口、DI/DO、CAN总线数据，进行边缘预处理，并通过蜂窝网络或VPN隧道与远程平台交互。

**配置步骤**：

1. 根据现场设备接口类型，配置EC5000串口工作模式（RS-232/422/485）。
2. 连接数字量输入/输出线缆，确认隔离式输入输出参数匹配。
3. 配置CAN FD接口波特率，接入现场CAN总线网络。
4. 插入SIM卡并安装蜂窝天线，配置蜂窝拨号参数。
5. 登录IEOS Web管理界面，启用云管理（DeviceLive）功能。
6. （可选）配置OpenVPN客户端，建立与数据中心的安全隧道。

5. 系统运行允许的电流范围为15A至5.6A，需确保供电回路具备足够容量。
6. 接地线需使用16AWG绿黄双色线，并可靠连接至系统接地螺丝。
7. 当多个USB满负载工作时，需根据总负载情况选择120W（或以上）电源适配器。
8. 系统重置功能会将用户安装的软件清除，操作前需备份重要数据。
9. GMSL摄像头不支持热插拔，连接或断开时需给设备断电。

> **警告**：非专业人员请勿打开设备外壳，存在触电风险。

## FAQ 常见问题解答

### 问题1：EC5000与普通工控机区别？

1. EC5000搭载NVIDIA Jetson Orin NX/Orin Nano模组，具备40~100 TOPS AI算力，支持边缘AI推理。
2. 集成工业级通信接口（RS-232/422/485、CAN FD、DI/DO、GMSL），适应工业现场环境。
3. 支持4G/5G蜂窝网络、Wi-Fi、蓝牙及GNSS无线通信，便于移动或偏远场景部署。
4. 内置IEOS网络管理系统和DeviceLive云管理平台支持，降低运维复杂度。
5. 工业级宽温设计（-20~60℃），支持DIN导轨和壁挂安装。

### 问题2：无法通过IEOS Web管理网络？

1. 确认IEOS程序正在运行：通过SSH执行 `systemctl status ieos_daemon`。
2. 确认访问地址和端口正确：`https://<设备IP>:9100`。
3. 确认PC与设备处于同一网段，且网络连通。
4. 若需使用Linux原生命令管理网络，需先关闭IEOS：`systemctl stop ieos_daemon`。
5. 注意：IEOS与Linux原生命令同时操作网络可能导致配置冲突。

### 问题3：如何恢复出厂设置？

1. 物理按键恢复：按住Recovery按键10秒，待系统状态灯由闪烁变为常亮后松开。
2. 命令恢复：在Terminal执行 `sudo update reset`。
3. 注意：恢复出厂会清除用户安装的软件和数据，操作前请备份。

### 问题4：蜂窝网络无法联网？

1. 物理环境：检查SIM卡是否插到正确的卡槽位，蜂窝天线是否全部安装。
2. APN设置：确保APN的配置信息与运营商提供的一致（专网卡通常需要配置）。
3. 检查设备连通性：登录到设备本地界面，通过系统自带的ICMP工具或Terminal ping 8.8.8.8测试是否连通。
4. 判断SIM卡是否正常：取出SIM卡，安装在手机中查看是否可以正常联网。
5. 重启：尝试将设备断电，等待几秒钟，然后重新连接电源，重试网络连接。
6. 恢复出厂：将设备恢复出厂，然后重新尝试。

### 问题5：GMSL摄像头无法识别？

1. 确认GMSL摄像头不支持热插拔，连接时必须断电。
2. 确认摄像头型号与初始化脚本支持的型号匹配。
3. 执行 `gmsl_init.sh` 选择正确型号后，使用 `gmsl_play.sh 0` 验证视频流。
4. GMSL_CAM0对应/dev/video0，GMSL_CAM1对应/dev/video1。
