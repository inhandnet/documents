## 第1章 认识设备

### 1.1 概述

EC942系列边缘计算机是映翰通公司面向工业物联网领域推出的一款轻量级AI加速边缘计算机。该产品搭载ARM Cortex-A55@2.0GHz四核处理器，标配4GB RAM和16GB eMMC，提供强劲的计算平台。产品采用发行版Linux系统，向用户提供灵活多样的二次开发环境。支持安全特性如Secure Boot和TPM2.0，保障软件及数据安全。可内置映翰通DeviceSupervisor Agent（简称DSA）服务，实现数据采集、处理及上云，并支持DeviceLive云管理。

本手册适用于基于Arm架构的边缘计算机EC940系列，并涵盖适用于所有支持型号的说明。

| Error | 闪烁 | 系统发生错误（恢复出厂尚未完成） |
| SIM1 | 常亮 | 选中SIM卡1进行拨号 |
|  | 常灭 | 选择SIM卡2拨号或关闭拨号 |
| SIM2 | 常亮 | 选中SIM卡2进行拨号 |
|  | 常灭 | 选择SIM卡1拨号或关闭拨号 |

| User2 | 默认熄灭 | 用户可编程控制 |

**蜂窝网信号强度指示灯说明**

| LED | 无信号 | 信号弱（RSSI < -90） | 信号中等（-90 <= RSSI < -70） | 信号强（RSSI >= -70） |
|-----|--------|----------------------|------------------------------|----------------------|
| L1 | 灭 | 亮 | 亮 | 亮 |
| L2 | 灭 | 灭 | 亮 | 亮 |
| L3 | 灭 | 灭 | 灭 | 亮 |

**恢复出厂设置过程指示灯状态**

| LED | 状态 |
|-----|------|
| WARN | 闪烁 |
| ERROR | 闪烁 |
| STATUS | 熄灭 |

执行恢复出厂设置后，系统会进行一次重启，重启完成后恢复出厂尚未完成，此时WARN灯和ERROR闪烁，STATUS熄灭。在此状态下不能对设备断电，否则可能导致文件丢失而影响系统功能。该状态持续约30秒，恢复出厂完成后WARN和ERROR熄灭，STATUS恢复闪烁。

### 1.5 恢复出厂设置

EC942支持两种恢复出厂设置方式：

1. **命令行方式**

   登录设备命令行界面，执行恢复出厂命令，系统会自动重启并恢复出厂设置。

<p align="center"><img src="./img/XzTk7OzJw-ghCpdU/1719887727771-0be25abc-bb1b-44f9-9eba-151a5b537019-751406.webp" alt="恢复出厂命令"></p>

<p align="center"><strong>图 1-3 恢复出厂命令（factory_reset）</strong></p>

2. **按键方式**

   - 长按恢复出厂设置按键10~20秒，看到WARN灯长亮。
   - WARN灯长亮后，松开恢复出厂设置按键。
   - 松开按键后，ERROR灯闪烁几次，系统开始重启并执行恢复出厂设置。
   - 系统重启后，WARN灯和ERROR灯闪烁，STATUS熄灭；约30秒后，WARN和ERROR灯停止闪烁，同时STATUS开始闪烁，表示恢复出厂设置完成。

#### 2.2.7 首次访问设备

**方式一：SSH命令行连接**

1. 将网线一端插入EC942的任一网口，另一端插入电脑网口。
2. 将电脑IP地址设置为设备接口同网段地址（ETH1：192.168.3.0/24，ETH2：192.168.4.0/24）。

## 第3章 常用场景配置

### 场景1：蜂窝联网

**目标**：通过4G/5G蜂窝网络接入互联网。

**前提**：已插入SIM卡并安装天线，设备已上电，可通过Web界面或命令行访问设备。

**预计用时**：约5分钟。

**操作步骤**：

1. 将SIM卡插入对应卡槽，安装蜂窝天线到ANT1~ANT4接口。
2. 登录IEOS Web管理界面，进入【网络】→【蜂窝】菜单。
3. 确认"启用蜂窝网络"开关处于开启状态。
4. 如使用专网卡，配置拨号参数集（APN、用户名、密码、认证方式）。
5. 根据需求选择网络制式（3G/4G/5G或自动）。
6. 确认"是否添加默认路由"已启用（默认开启）。
7. 如需双卡备份，开启"启用双SIM卡"功能，配置主卡和最大拨号次数。
8. 建议开启ICMP探测功能，配置探测服务器地址以检测拨号假连接。
9. 点击"保存"并等待拨号建立。

**验证方法**：

1. 查看4G/5G指示灯，确认拨号成功后常亮。
2. 在Web界面的【状态】→【蜂窝拨号状态信息】页面查看获取的IP地址、DNS等信息。
3. 使用网络诊断工具ping外网地址（如8.8.8.8），确认网络连通。

**常见问题**：

- 网络连接失败：检查SIM卡是否正确插入、APN参数是否正确、天线是否安装到位。
- 拨号假连接（状态成功但无法上网）：检查ICMP探测是否开启，或尝试手动重新拨号。
- 信号弱：调整设备位置或天线方向，参考L1/L2/L3信号强度指示灯判断信号质量。

### 场景2：Wi-Fi Station接入

**目标**：将EC942作为Wi-Fi客户端接入上游无线网络。

**前提**：设备支持Wi-Fi功能，已安装Wi-Fi天线，已知目标Wi-Fi网络的SSID和密码。

**预计用时**：约5分钟。

**操作步骤**：

1. 将Wi-Fi天线安装到WiFi1/WiFi2接口。
2. 登录IEOS Web管理界面，进入【网络】→【Wi-Fi Station】菜单。
3. 开启"启用Wi-Fi"开关。
4. 在"客户端SSID"中输入目标Wi-Fi网络的SSID，或通过"扫描"按钮获取附近可用网络。
5. 选择正确的认证方式（无认证、WPA-PSK、WPA2-PSK、WPA-PSK/WPA2-PSK Mixed）。
6. 选择加密方式（CCMP、TKIP或两者）。
7. 输入"WPA/WPA2 PSK密钥"。
8. 确认"是否添加默认路由"设置（默认开启）。
9. 点击"保存"并等待Wi-Fi连接建立。

**验证方法**：

1. 在Web界面的【状态】→【Wi-Fi Station状态信息】页面查看获取的IP地址、网关及DNS。
2. 使用网络诊断工具ping外网地址，确认网络连通。

**常见问题**：

- 无法连接Wi-Fi：检查SSID和密码是否正确，确认Wi-Fi信号强度是否足够。
- 无法上网：检查"是否添加默认路由"是否启用，或检查路由度量值设置是否合理。

### 场景3：以太网配置

**目标**：配置EC942以太网接口的IP地址及网络参数。

**前提**：设备已通过网线连接到管理电脑，可访问Web界面或命令行。

**预计用时**：约5分钟。

**操作步骤**：

1. 登录IEOS Web管理界面，进入【网络】→【以太网】菜单。
2. 选择需要配置的接口（eth1或eth2）。
3. 如需配置静态IP：
   - 将协议类型设置为"静态地址"。
   - 填写IP地址、子网掩码、网关。
   - 点击"保存"。
4. 如需配置DHCP Client：
   - 将协议类型设置为"DHCP客户端"。
   - 点击"保存"，等待自动获取IP地址。
5. 如需启动DHCP Server（为下挂设备分配地址）：
   - 在对应接口启用DHCP Server功能。
   - 配置起始分配地址、最大地址分配数量、租期。

**验证方法**：

3. 从下挂设备ping设备网关地址，确认网络连通。

**常见问题**：

- 无法获取IP地址：检查网线连接、确认DHCP服务器是否正常工作。
- 下挂设备无法上网：检查DHCP Server地址池是否与接口网段匹配，检查网关和DNS配置。

### 场景4：OpenVPN远程接入

**目标**：通过OpenVPN建立安全的远程接入隧道。

**前提**：已具备OpenVPN服务器环境或已知OpenVPN服务器信息，设备已接入互联网。

**预计用时**：约10分钟。

**操作步骤**：

1. 登录IEOS Web管理界面，进入【网络】→【OpenVPN】菜单。
2. 如需配置OpenVPN服务器：
   - 点击"添加服务器"，选择手动配置或导入配置。
   - 手动配置时，填写服务端名称、虚拟网络、网络掩码。
   - 导入CA证书、公钥证书、私钥证书及DH参数。
   - 配置高级参数（协议类型、端口、加密算法等）。
   - 点击"保存"。
3. 如需配置OpenVPN客户端：
   - 点击"添加客户端"，选择手动配置或导入配置。
   - 手动配置时，填写客户端名称、OpenVPN服务器地址（格式：IP 端口 协议）。
   - 导入CA证书、公钥证书、私钥证书。
   - 配置高级参数。
   - 点击"保存"。
4. 在状态栏开启该OpenVPN配置。

**验证方法**：

1. 在Web界面的【状态】→【OpenVPN状态信息】页面查看连接状态。
   - 服务器状态：查看已连接的客户端信息。
   - 客户端状态：查看获取的虚IP地址及隧道连接时长。
2. 通过VPN隧道ping对端虚拟IP地址，确认隧道连通。

**常见问题**：

- 无法建立连接：检查服务器地址和端口是否正确，确认证书是否匹配，检查防火墙是否放行了OpenVPN端口。
- 隧道建立后无法通信：检查虚拟网络网段配置，确认是否开启了"允许客户端到客户端的通信"。

### 4.2 基于IEOS的Web管理

EC942基于Debian 10系统，可使用Linux原生命令进行网络管理和系统管理。为便于配置，映翰通开发了IEOS系统程序，提供Web界面进行网络和系统管理。当启用IEOS功能后，IEOS会接管网络管理和系统管理，此时通过Linux原生命令进行网络管理和系统管理可能会失效。设备出厂默认启用IEOS。

> **注意**：使用IEOS程序管理网络配置和系统配置时，如果同时使用Linux原生命令，二者可能会相互影响，导致异常运行状态。建议IEOS支持的配置都通过IEOS Web实现管理，IEOS不支持的配置可结合Linux原生命令实现。

#### 4.2.1 登录Web

考虑到用户的程序可能会需要使用HTTP/HTTPS标准端口号80/443，IEOS使用9100端口号作为HTTPS连接的端口，不支持通过HTTP访问；当用户使用HTTP访问Web时，会自动跳转到HTTPS。以通过eth2的默认地址192.168.4.100为例进行说明。用户在浏览器中输入192.168.4.100:9100后跳转到登录页面。

> **注意**：IEOS程序启用时，会预留端口号范围9100~9200供内部通信使用。启用IEOS后，用户程序应避开使用这些端口号，否则可能产生冲突导致功能异常。

<p align="center"><img src="./img/XzTk7OzJw-ghCpdU/1754364060972-570699aa-683b-4b47-8c6b-f308be1b56fc-519136.webp" alt="MAC地址过滤"></p>

MAC地址过滤用于控制EC942下挂设备产生的报文是否可被转发，但不影响下挂设备对EC942本身的访问。MAC地址列表最多可添加500条规则。

该功能包含三种模式：

- **无限制**：功能关闭，不对MAC地址列表中的设备产生影响

- **白名单**：仅列表中的设备产生的报文可被转发，其他设备被丢弃

MAC地址列表中每个记录均可单独启用/禁用。

自动重启选项默认关闭，升级固件后需手动重启系统才能生效；开启自动重启选项后，升级成功会自动重启系统。

**系统升级**

准备一个U盘(micro SD卡)。如果该U盘(SD卡)有多个分区，请使用第一个分区。建议不做多个分区。U盘(SD卡)分区需要格式为FAT32格式。本文档以升级EC942-V2.0.0.img为例进行说明。

- 在U盘（SD卡）根目录下创建一个空的ec900\_img目录，将inhand发布的EC942-V2.0.0.img文件和md5.txt文件放到ec900\_img目录下。

- 确认md5.txt文件中仅有EC942-V2.0.0.img一行的md5哈希值。EC942不支持多个img镜像ota升级。

- 在电脑上正常退出U盘（SD卡）。注意不要直接拔u盘，要在桌面选择“退出”或“弹出”操作。

- 将U盘（SD卡）插入目标EC942设备。目标设备会自动校验EC942-V2.0.0.img文件，并进行OTA升级。升级过程中WARN和ERROR灯会有相应的显示。等WARN和ERROR恢复正常后，升级操作即完成。因为img文件较大，升级时间较长，请耐心等候。

- 升级结束后，EC942会把升级中的关键信息写入ec900\_img目录下的日志文件。请查看相关文件。

#### 4.4.3 Micro SD卡

EC942支持micro SD存储卡，不支持热插拔。插入SD卡并上电后，系统会自动挂载所有分区到/mnt/路径下，挂载文件夹命名格式为`sd_<node>_<num>`。

SD卡分区需格式化为FAT32格式：

- SD卡容量 ≤ 32GB：可在Windows系统中格式化，或使用Linux系统的`mkfs.vfat -F 32`命令。
- SD卡容量 > 32GB：需在Linux系统中格式化。

Linux下格式化步骤：

1. 执行`fdisk -l`查看SD卡对应的设备节点（如/dev/sda）。
2. 执行`fdisk /dev/sda`进行分区（p查看当前分区，d删除已有分区，n新建分区，w保存更改）。
3. 执行`sudo mkfs.vfat -F 32 /dev/sda1`将分区格式化为FAT32格式。
4. 格式化后，在设备断电情况下插入SD卡。

### 4.5 安全功能

#### 4.5.1 sudo机制

EC942中root用户被禁止使用。Sudo是一个允许授权用户以root用户或其他用户身份执行命令的程序。使用sudo比直接使用root会话更安全：

- 无需知道root密码（sudo会提示当前用户密码），可授予普通用户特权
- 通过sudo运行需要特权的命令，其余时间作为无特权用户工作，降低错误操作造成的损害
- 某些系统级命令不能直接对用户使用

> **注意**：在非root权限下使用管道或重定向（如`>`、`<`、`>>`、`<<`等）时可能得到"permission denied"提示，此时需使用`sudo su -c`并包含完整命令的单引号。

#### 4.5.2 防火墙

Netfilter/iptables是Linux系统自带的基于包过滤的防火墙工具，功能强大，可对流入、流出及流经设备的数据包进行精细控制。IEOS Web界面中的防火墙功能基于iptables实现。

#### 4.5.3 TPM2.0

TPM（Trusted Platform Module，可信平台模块）是一种硬件安全模块，旨在提供计算机系统的安全和加密功能。它包含加密协处理器，用于存储加密密钥、数字证书和其他安全数据，支持多种加密算法和安全协议。EC942已集成标准的TPM2.0协议栈和TPM2.0工具供用户使用。

### 4.6 编程指南

EC942提供了一个JSON格式的设备信息描述文件，操作IO、LED、串口等外设时，可通过查询该文件获取外设的设备节点信息。

设备描述信息文件路径：`/tmp/ieos/etc/system_info.json`

#### 4.6.1 IO编程

根据设备描述信息文件，IO设备节点如下：

- DI0~DI3：`/sys/class/gpio/gpio487` ~ `/sys/class/gpio/gpio490`
- DO0~DO3：`/sys/class/gpio/gpio491` ~ `/sys/class/gpio/gpio494`

操作IO接口时，直接读写设备节点下的value值即可。

设置DO0输出高电平：

```shell
echo 1 > /sys/class/gpio/gpio491/value
```

查看DI0电平：

```shell
cat /sys/class/gpio/gpio487/value
```

完整shell脚本示例：

```shell
#!/bin/bash

gpio491="/sys/class/gpio/gpio491/value"
gpio492="/sys/class/gpio/gpio492/value"

if [ -f "$gpio491" ]; then
    echo 1 > $gpio491
else
    echo "no file exit "$gpio491
fi

if [ -f "$gpio492" ]; then
    echo 0 > $gpio492
else
    echo "no file exit "$gpio492
fi

gpio487="/sys/class/gpio/gpio487/value"

if [ -f "$gpio487" ]; then
    cat $gpio487
else
    echo "no file exit "$gpio487
fi
```

#### 4.6.2 LED编程

EC942提供USER1和USER2两个用户可编程LED。

根据设备描述信息文件，LED设备节点为：

- USER1：`/sys/class/leds/user1`
- USER2：`/sys/class/leds/user2`

LED控制文件说明：

| 文件 | 功能 |
|------|------|
| brightness | 控制LED亮灭，写入1为常亮，0为常灭 |
| trigger | LED触发器，写入timer表示定时器触发，写入none表示取消触发器 |
| delay_on | LED亮的时间，单位为ms |
| delay_off | LED灭的时间，单位为ms |

> **注意**：若配置了trigger为定时触发，brightness中的值将不再生效，自动变为0。

设置USER1常亮：

```shell
echo 1 > /sys/class/leds/user1/brightness
```

设置USER1闪烁（1秒亮、1秒灭）：

```shell
echo timer > /sys/class/leds/user1/trigger
echo 1000 > /sys/class/leds/user1/delay_on
echo 1000 > /sys/class/leds/user1/delay_off
```

关闭USER1闪烁：

```shell
echo none > /sys/class/leds/user1/trigger
```

#### 4.6.3 交叉编译

用户C/C++程序可通过在开发机上使用交叉编译工具链进行交叉编译，然后将目标文件上传到EC942设备执行。

交叉编译工具：`gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu`

**配置环境变量**

1. 在开发机上将交叉编译工具链解压到/opt路径下（也可解压到其他路径）。
2. 编辑`~/.bashrc`文件，在末尾添加：

```shell
PATH=$PATH:/opt/gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu/bin
```

3. 执行`source ~/.bashrc`使环境变量在当前终端生效；新打开的终端自动生效。

**编译示例**

以hello world程序为例，创建目录和文件：

```shell
mkdir ~/example
touch ~/example/hello.c
touch ~/example/Makefile
```

hello.c内容：

```c
#include <stdio.h>

int main(void)
{
    printf("hello, world!\n");
    return 0;
}
```

Makefile内容：

```makefile
TARGET  := helloworld
DIRS    := $(shell find . -maxdepth 3 -type d)
SRCS    := $(foreach dir,$(DIRS),$(wildcard $(dir)/*.c))
OBJS    := $(SRCS:.c=.o)

CC=aarch64-linux-gnu-gcc

CFLAGS  := -Wall -Wextra -g -Wno-unused-parameter

all: $(TARGET)

$(TARGET): $(OBJS)
	$(CC) $(CFLAGS) $(LIBS) $^ -o $@

%.o: %.c
	$(CC) $(CFLAGS) $(LIBS) -c $< -o $@

clean:
	rm -f $(TARGET) $(OBJS)

.PHONY: all clean
```

在~/example目录下执行`make`，生成目标文件`helloworld`。

## 第5章 典型应用

### 案例1：工业远程监控与数据采集

**场景描述**

某工厂需对分散在多个车间的PLC和传感器进行统一监控。EC942部署于各车间现场，通过串口或以太网连接PLC，利用内置的DeviceSupervisor Agent采集设备数据，经蜂窝网络上传至DeviceLive云平台，实现远程实时监控和运维管理。

**设备角色**

EC942作为边缘网关，承担以下角色：
- 通过串口/以太网采集现场PLC和传感器数据
- 运行DeviceSupervisor Agent进行数据预处理
- 通过蜂窝网络将数据上传至DeviceLive云平台
- 作为本地网络路由器，为现场维护人员提供Web管理入口

**配置步骤**

1. **硬件安装**
   1. 将EC942安装到车间机柜DIN导轨上。
   2. 连接12~48VDC电源，确认PWR指示灯常亮。
   3. 安装蜂窝天线到ANT1~ANT4接口。
   4. 插入SIM卡到SIM1卡槽。
   5. 使用网线或串口线连接PLC到EC942。

2. **蜂窝网络配置**
   1. 登录IEOS Web界面，进入【网络】→【蜂窝】。
   2. 确认"启用蜂窝网络"已开启。
   3. 如使用专网卡，配置拨号参数集（APN、用户名、密码）。
   4. 开启ICMP探测功能，配置探测服务器地址。
   5. 点击"保存"，等待拨号成功（4G/5G指示灯常亮）。

3. **云管理平台接入**
   1. 进入【系统】→【基础配置】→【云管理】。
   2. 开启"启动云服务"。
   3. 选择对应的云服务器地址（国内/海外）。
   4. 点击"保存"。
   5. 登录DeviceLive平台，确认设备状态为"在线"。

4. **串口数据采集配置**
   1. 确认PLC与EC942串口（COM1/COM2）物理连接正确。
   2. 使用`stty`命令配置串口参数（波特率、数据位、停止位）与PLC匹配。
   3. 部署DeviceSupervisor Agent采集程序，配置采集协议和寄存器地址。
   4. 验证数据采集是否正常。

**验证方法**

1. 在DeviceLive平台查看设备在线状态和数据上报情况。
2. 在IEOS【状态】→【蜂窝拨号状态信息】页面确认蜂窝网络正常。
3. 在EC942本地使用`cat /dev/ttyS3`验证串口数据接收。
4. 检查DeviceSupervisor Agent日志，确认数据采集和上传无异常。

1. 设备应在规定的温度和湿度范围内使用，工作温度范围为-20~70℃，环境湿度为5~95%（无凝霜）。
2. 请勿在易燃易爆环境中使用本设备。
3. 电源连接前需确认电压符合设备规格要求（9~48VDC）。
4. SIM卡和Micro SD卡不支持热插拔，插拔操作前需断开设备电源。
5. USB存储设备断开连接前，需执行`sync`命令同步数据，防止数据丢失。
6. 恢复出厂设置过程中（WARN和ERROR闪烁期间）严禁断电，否则可能导致文件丢失。
7. 固件升级过程中请勿断电或拔出U盘/SD卡，否则可能导致系统损坏。

> **警告**：非专业人员请勿打开设备外壳，存在触电风险。
