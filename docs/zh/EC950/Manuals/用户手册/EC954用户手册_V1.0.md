## 第1章 认识设备

### 1.1 概述

EC954是一款基于ARM架构的高性能多接口边缘计算机，属于EC950系列产品。该设备搭载ARM Cortex-A55@2.0GHz四核处理器，最高可扩展26 TOPS AI算力，并支持TensorRT/cuDNN/VisionWorks/OpenCV AI框架。设备采用Debian 11发行版Linux系统，提供灵活多样的二次开发环境。EC954支持Secure Boot和TPM2.0安全特性，可内置映翰通DeviceSupervisor™ Agent（DSA）服务，实现数据采集、处理及上云，并支持DeviceLive云管理平台远程管理。

执行恢复出厂设置后，系统会进行一次重启。重启完成后，恢复出厂尚未完成，此时WARN灯和ERR灯闪烁，STAT灯熄灭。该状态下不能对设备断电，否则可能导致文件丢失。该状态持续约30秒，恢复出厂完成后，WARN和ERR灯熄灭，STAT灯恢复闪烁。

### 1.5 恢复出厂设置

EC954支持以下两种方式恢复出厂设置：

1. **命令行方式**
   登录设备后，执行以下命令：
   ```
   sudo factory_reset
   ```
   系统会自动重启并恢复出厂设置。

2. **按键方式**
   1. 长按恢复出厂设置按键10-20秒，看到WARN灯长亮。
   2. WARN灯长亮后，松开恢复出厂设置按键。
   3. 松开按键后，ERR灯闪烁几次，系统开始重启并执行恢复出厂设置。
   4. 系统重启后，WARN灯和ERR灯闪烁，STAT灯熄灭；约30秒后，WARN灯和ERR灯停止闪烁，STAT灯开始闪烁，表示恢复出厂设置完成。

#### 4.4.3 Micro SD卡

Micro SD卡分区需格式化为FAT32格式。

- SD卡容量 ≤ 32GB：可在Windows系统下直接格式化为FAT32；也可在Linux系统下格式化。
- SD卡容量 > 32GB：需在Linux下格式化为FAT32。

Linux下格式化步骤：
1. 执行 `fdisk -l` 查看SD卡对应的设备节点（示例中为 `/dev/sda`）。
2. 执行 `fdisk /dev/sda` 进行分区（`p` 查看当前分区，`d` 删除已有分区，`n` 创建新分区，`w` 保存更改），创建分区如 `/dev/sda1`。
3. 执行 `sudo mkfs.vfat -F 32 /dev/sda1` 将分区格式化为FAT32。
4. 格式化完成后，在设备断电的情况下将SD卡插入设备。

EC954支持micro SD存储卡但不支持热插拔。设备会自动挂载所有分区到 `/mnt/` 路径下，挂载文件夹命名格式为 `sd_<node>_<num>`。

### 4.5 安全功能

#### 4.5.1 sudo机制

在EC954中，为获得更好的安全性，root用户被禁止使用。Sudo是一个让系统管理员允许被授权用户作为root用户或其他用户执行命令的程序。最基础的准则是给予尽可能少的特权完成工作。

使用sudo比使用root身份更安全：
- 无需知道root密码（sudo会提示当前用户的密码），可授予普通用户特权
- 通过sudo运行需要特权的命令，其余时间作为无特权用户工作，降低错误操作可能造成的损害
- 有些系统级命令不能直接对用户使用

#### 4.5.3 TPM2.0

#### 4.6.1 IO编程

设备上共8个IO接口：DI0～DI3为4个输入管脚，DO0～DO3为4个输出管脚。

根据设备描述信息文件可获取IO设备节点：
- DI0～DI3：`/sys/class/gpio/gpio487`～`/sys/class/gpio/gpio490`
- DO0～DO3：`/sys/class/gpio/gpio491`～`/sys/class/gpio/gpio494`

对IO接口编程时，直接操作后台设备节点下面的value值即可（`/sys/class/gpio/gpioxxx/value`）。

DO0输出高电平：
```shell
echo 1 > /sys/class/gpio/gpio491/value
```

查看DI0电平：
```shell
cat /sys/class/gpio/gpio487/value
```

完整shell脚本示例：
```shell
#!/bin/bash

gpio491="/sys/class/gpio/gpio491/value"
gpio492="/sys/class/gpio/gpio492/value"
gpio493="/sys/class/gpio/gpio493/value"
gpio494="/sys/class/gpio/gpio494/value"

if [ -f "$gpio491" ]; then
   echo 1 > /sys/class/gpio/gpio491/value
else
   echo "no file exit "$gpio491
fi

if [ -f "$gpio492" ]; then
   echo 0 > $gpio492
else
   echo "no file exit "$gpio492
fi

gpio487="/sys/class/gpio/gpio487/value"
gpio488="/sys/class/gpio/gpio488/value"
gpio489="/sys/class/gpio/gpio489/value"
gpio490="/sys/class/gpio/gpio490/value"

if [ -f "$gpio487" ]; then
   cat $gpio487
else
   echo "no file exit "$gpio487
fi
```

#### 4.6.2 LED编程

设备上用户可使用USER1和USER2两个灯进行状态提示。

根据设备描述信息文件可获取LED设备节点：
- USER1：`/sys/class/leds/user1`
- USER2：`/sys/class/leds/user2`

控制文件说明：
- `brightness`：控制LED亮或灭。写入1为常亮，写入0为常灭
- `trigger`：LED触发器。写入 `timer` 表示定时器触发，写入 `none` 表示取消触发器
- `delay_on`：LED亮的时间，单位为ms
- `delay_off`：LED灭的时间，单位为ms

若配置trigger为定时触发，brightness中的值将不再生效，会自动变为0。

USER1常亮：
```shell
echo 1 > /sys/class/leds/user1/brightness
```

USER1闪烁（1秒周期）：
```shell
echo timer > /sys/class/leds/user1/trigger
echo 1000 > /sys/class/leds/user1/delay_on
echo 1000 > /sys/class/leds/user1/delay_off
```

#### 4.6.3 交叉编译

用户自己的C/C++程序可通过在开发机器上使用交叉编译工具链进行交叉编译，然后将目标文件上传到EC954设备上执行。

交叉编译工具压缩包：`gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu.tar.gz`

配置环境变量方法：
1. 在开发机上将交叉编译工具链解压到 `/opt` 路径下（也可解压到其他路径，步骤2中做相应调整）。
2. 编辑 `~/.bashrc` 文件，在文件最后添加一行：
   ```
   PATH=$PATH:/opt/gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu/bin
   ```
3. 执行 `source ~/.bashrc` 使环境变量在当前终端生效；新打开的终端自动生效。

以经典的hello world程序为例：

`~/example/hello.c`：
```c
#include <stdio.h>

int main(void)
{
    printf("hello, world!\n");
    return 0;
}
```

`~/example/Makefile`：
```makefile
TARGET  := hellworld
DIRS    := $(shell find . -maxdepth 3 -type d)
SRCS    := $(foreach dir,$(DIRS),$(wildcard $(dir)/*.c))
OBJS    := $(SRCS:.c=.o)

CC=aarch64-linux-gnu-gcc
CFLAGS  := -Wall -Wextra -g -Wno-unused-parameter

all: $(TARGET)

$(TARGET): $(OBJS)
	$(CC) $(CFLAGS) $(LIBS) $^ -o $@

%.o: %.c
	$(CC) $(CFLAGS) $(LIBS) -c $< -o $@

clean:
	rm -f $(TARGET) $(OBJS)

.PHONY: all clean
```

在 `~/example` 目录下执行 `make`，生成目标文件。

#### 4.6.5 编译内核模块

**在设备上直接编译**

1. 使用 `sudo -s` 切换到root用户
2. 安装编译环境：
   ```
   apt-get update
   apt-get install -y build-essential
   ln -s /usr/bin/python3 /usr/bin/python
   ```
3. 创建 `hello.c`：
   ```c
   #include <linux/module.h>
   #include <linux/kernel.h>
   
   int init_module(void) {
       printk(KERN_INFO "Hello, world - this is the kernel speaking\n");
       return 0;
   }
   
   void cleanup_module(void) {
       printk(KERN_INFO "Goodbye, world - leaving the kernel\n");
   }
   
   MODULE_LICENSE("GPL");
   ```
4. 创建 `Makefile`：
   ```makefile
   obj-m += hello.o
   
   all:
    make -C /lib/modules/$(shell uname -r)/build M=$(PWD) modules
   
   clean:
    make -C /lib/modules/$(shell uname -r)/build M=$(PWD) clean
   ```
5. 执行 `make` 编译模块，生成 `hello.ko`
6. 执行 `insmod hello.ko` 安装模块

**交叉编译**

1. 安装编译环境：
   - 已测试系统：Ubuntu 22.04
   - 解压交叉编译工具链：`tar -xf gcc-arm-10.3-2021.07-x86_64-aarch64-none-linux-gnu.tar.gz -C /opt/`
   - 创建目录：`mkdir linux-headers`
   - 解压内核编译环境：`tar -xf linux-headers.tar -C linux-headers`
2. 创建 `hello.c`（同上）
3. 创建 `Makefile`：
   ```makefile
   obj-m += hello.o
   
   all:
    make -C linux-headers M=$(PWD) CROSS_COMPILE=/opt/gcc-arm-10.3-2021.07-x86_64-aarch64-none-linux-gnu/bin/aarch64-none-linux-gnu- ARCH=arm64 modules
   
   clean:
    make -C linux-headers M=$(PWD) CROSS_COMPILE=/opt/gcc-arm-10.3-2021.07-x86_64-aarch64-none-linux-gnu/bin/aarch64-none-linux-gnu- ARCH=arm64 clean
   ```
4. 执行 `make` 编译模块，生成 `hello.ko`，将此文件传输到设备中安装

### 4.7 系统升级

#### 4.7.1 离线系统升级（U盘/SD卡）

准备一个U盘或Micro SD卡。若该存储设备有多个分区，请使用第一个分区。建议不做多个分区。U盘/SD卡分区需格式化为FAT32格式。以下以升级 `EC954-V2.0.0.img` 为例说明。

1. 在U盘（SD卡）根目录下创建空的 `ec900_img` 目录，将固件镜像文件和 `md5.txt` 文件放入该目录。
2. 确认 `md5.txt` 文件中仅有该镜像文件一行的MD5哈希值。EC954不支持多个img镜像同时OTA升级。
3. 在电脑上正常退出U盘（SD卡），不要直接拔插。
4. 将U盘（SD卡）插入目标EC954设备。设备自动校验镜像文件并进行OTA升级。升级过程中WARN和ERR灯会有相应显示。WARN和ERR灯恢复正常后，升级完成。
5. 升级结束后，EC954会把升级中的关键信息写入 `ec900_img` 目录下的日志文件。

## 附录B 安全注意事项

1. 设备应在规定的温度和湿度范围内使用（工作温度-20℃～70℃，环境湿度5%～95%无凝露）。
2. 请勿在易燃易爆环境中使用本设备。
3. 电源连接前请确认电压符合设备规格要求（12～48VDC）。
4. SIM卡和Micro SD卡不支持热插拔，插拔前必须断电。
5. 恢复出厂设置过程中，当WARN和ERR灯闪烁、STAT灯熄灭时，不能对设备断电，否则可能导致文件丢失。
6. 在断开USB大容量存储设备前，需执行 `sync` 命令同步数据，防止数据丢失。
7. 出于安全考虑，建议创建新的用户账户后禁用默认的edge账户。
8. 非专业人员请勿随意修改系统核心配置文件。

> **警告**：非专业人员请勿打开设备外壳，存在触电风险。

## FAQ

### 问题1：EC954与普通工控机有何区别？

1. EC954是一款边缘计算机，搭载ARM Cortex-A55四核处理器，支持多种工业接口（串口、CAN、DI/DO），并内置蜂窝网络、Wi-Fi、GNSS等无线通信能力。
2. EC954预装Debian 11 Linux系统，支持二次开发，同时集成IEOS网络管理系统，提供Web图形化管理界面。
3. EC954支持DeviceLive云管理平台远程管理，具备Secure Boot和TPM2.0安全特性，部分型号内置AI算力模组。
4. 普通工控机通常依赖固定网络连接，无内置无线通信能力，也无统一的远程云管理平台。

### 问题2：4G/5G无法联网？

1. 物理环境：检查SIM卡是否插到正确卡槽，蜂窝天线是否全部安装。
2. APN设置：确保APN配置信息与运营商提供的一致。
3. 检查设备连通性：登录设备本地界面，通过系统自带ICMP工具ping 8.8.8.8测试是否连通。若可连通，检查电脑与路由器的连通性。
4. 判断SIM卡是否正常：取出SIM卡安装在手机中查看是否可以正常联网。
5. 重启：尝试将设备断电，等待数秒后重新连接电源，重试网络连接。
6. 恢复出厂：将设备恢复出厂设置后重新尝试。

### 问题3：如何关闭IEOS使用Linux原生命令管理网络？

1. 登录设备命令行。
2. 执行 `sudo systemctl stop ieos_daemon` 停止IEOS（仅本次生效）。
3. 执行 `sudo systemctl disable ieos_daemon` 禁止IEOS开机启动。
4. 此后可使用Linux原生命令（如修改 `/etc/network/interfaces.d/eth1`）管理网络。
5. 关闭IEOS后，拨号、Wi-Fi等无线联网功能需自行基于Linux命令实现，且无法对接DeviceLive平台。

### 问题4：USB设备无法自动挂载？

1. 确认USB设备格式是否被系统支持（推荐FAT32/exFAT/ext4）。
2. 检查 `/mnt/` 目录下是否已生成 `usb_<node>_<num>` 文件夹。
3. 若未自动挂载，可尝试手动挂载：
   ```
   sudo mount /dev/sda1 /mnt/usb_manual
   ```
4. 断开USB设备前务必执行 `sync` 命令，并从 `/media/*` 目录退出。

### 问题5：串口通信异常？

1. 确认串口模式是否正确（P1～P4需使用 `ih_uart_ctl` 设置为对应模式：RS-232/RS-422/RS-485）。
2. 检查波特率、数据位、停止位、校验位等参数是否与对端设备一致。
3. 使用 `stty` 命令查看当前串口参数：
   ```
   sudo stty -F /dev/ttyCOM1
   ```
4. 确认接线是否正确，参考 [4.4.1 串口](#441-串口) 中的引脚定义表。
