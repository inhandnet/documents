## 第1章 认识设备

### 1.1 概述

ER615（Edge Router 615）是北京映翰通面向商业联网领域推出的新一代5G高性价比边缘路由器产品。该设备融合4G/5G无线网络与多种宽带服务，为各行业提供高速且安全的网络接入。5千兆网口+AX3000 WiFi射频芯片的设计，使ER615在满足门店联网基础需求外，还能适用更多复杂的应用场景。

## 第2章 安装与首次使用

### 2.1 安装前准备

**环境要求**

| 项目 | 要求 |
|------|------|
| 电源 | 请使用原装电源适配器，避免因电源适配器和设备不匹配造成设备损坏 |
| 安装位置 | 请勿将设备安装在强电磁干扰环境下，避免与高功率设备过于接近 |
| 温湿度 | 请确认设备使用环境满足规格书所要求的温度和湿度 |
| 天线 | 安装4G/5G天线及Wi-Fi天线（如使用蜂窝网络） |
| SIM卡 | 准备运营商提供的有效SIM卡（如使用蜂窝网络） |

> **警告**：安装设备时请确认设备是否易发生晃动，避免因设备跌落造成设备损坏。

### 2.2 安装指南

#### 2.2.1 硬件安装

1. 在设备未接入电源时安装4G/5G天线、Wi-Fi天线并插入SIM卡。
2. 将设备接入电源，并用网线将设备任意LAN口连接至PC。

#### 2.2.2 网络配置

将PC的IP地址设置为与边缘路由器处于同一地址段：

1. 设备默认开启DHCP服务，将PC设置为"自动获得IP地址"后，边缘路由器将为PC自动分配一个地址。
2. 验证PC是否获取到IP地址，且该地址是否处于192.168.2.0网段。
3. 若PC无法自动获取IP地址，需手动将PC的IP地址配置为192.168.2.2~192.168.2.254中的任意地址，默认网关设置为192.168.2.1，子网掩码为255.255.255.0，DNS配置为8.8.8.8或运营商DNS服务器地址。

## 第3章 常用场景配置

### 场景1：蜂窝联网

**目标**：通过4G/5G蜂窝网络接入互联网。

**前提**：已插入SIM卡并安装天线，设备已上电。

**预计用时**：约5分钟。

**操作步骤**：

1. 在设备未接入电源时安装蜂窝天线并插入SIM卡（按照设备插卡处丝印放置SIM卡）。
2. 将设备接入电源，等待设备启动完成。
3. ER615路由器将自动创建拨号连接，并在系统启动后连接到蜂窝网络。
4. 若需配置APN参数，进入【Internet】→【蜂窝连接】菜单，点击"编辑"按钮。
5. 配置APN参数（APN参数需从运营商获取），点击"保存"。

<p align="center"><img src="./img/OrYTAIbcHcjsKauQ/1764817587997-46fc9553-a237-4bdb-a7cd-20526cd641a8-271984.webp" alt="蜂窝编辑入口"></p>

<p align="center"><strong>图 3-1 蜂窝编辑入口</strong></p>

<p align="center"><img src="./img/OrYTAIbcHcjsKauQ/1765264061015-acb838f2-b3ab-4c4e-9863-2832293d4516-161590.webp" alt="配置拨号参数"></p>

<p align="center"><strong>图 3-2 配置拨号参数</strong></p>

**主要配置参数说明**：

| 参数 | 说明 |
|------|------|
| 状态 | 蜂窝功能的使能开关，默认开启 |
| 工作模式 | 默认"仅SIM1"，如需使用SIM2或双卡模式需手动修改 |
| 拨号参数集 | 默认"自动"，选择"手动"后按需填写APN信息 |
| IP类型 | 支持IPv4或IPv6 |
| 制式 | 默认"自动"，SIM卡在5G、4G间选择最优信号 |
| 5G模式 | 可选"SA/NSA、SA、NSA"，默认"SA/NSA" |
| IMS | 运营商4G/5G网络语音和多媒体服务系统，建议禁用 |

**验证方法**：

1. 检查联网状态灯，确认蜂窝网连接成功（绿色常亮）。
2. 登录Web界面，查看【状态】→【链路监控】确认蜂窝接口状态正常。
3. 使用系统工具Ping外部地址（如8.8.8.8），确认网络连通。

**常见问题**：

- 网络连接失败：检查SIM卡是否正确插入、APN参数是否正确。
- 数据发送接收异常：检查信号强度和流量余额。

> **注意**：插拔SIM卡时，必须拔掉电源，以免造成数据丢失或设备损坏。部分专网情况下，需手动关闭【Internet】菜单下的"链路探测"功能。

### 场景2：有线接入

**目标**：通过有线宽带网络接入互联网。

**前提**：已准备网线上行设备（如光猫或上级路由器），设备已上电。

**预计用时**：约5分钟。

**操作步骤**：

1. 使用网线将ER615的WAN口与上行设备连接。
2. 默认情况下，WAN1接口启用DHCP服务，可自动获取IP地址联网。
3. 若需配置静态IP或PPPoE拨号，进入【Internet】→【有线连接】菜单，点击"编辑"按钮。
4. 选择连接方式：
   - DHCP：自动获取IP地址（默认）
   - 静态IP：手动配置IP地址、子网掩码、网关和DNS
   - PPPoE：配置宽带账号和密码
5. 点击"保存"并等待连接建立。

1. 在云平台【设备】列表中查看设备是否显示为"在线"状态。
2. 点击设备名称，查看设备详细信息页面是否正常显示。

### 场景4：Wi-Fi配置

**目标**：配置无线网络，为终端设备提供无线接入服务。

**前提**：设备已上电，Wi-Fi功能已启用。

**预计用时**：约10分钟。

**操作步骤**：

1. 进入【Wi-Fi】→【SSIDs】菜单。
2. 点击"添加"或选择已有SSID点击"编辑"。
3. 配置以下参数：
   - SSID名称：设置无线网络名称（每台设备的SSID名称出厂唯一，可在设备背面铭牌处查看）
   - 频段：选择2.4GHz或5.8GHz
   - 安全：选择WPA2-PSK、WPA3-PSK等认证方式
   - 密码：设置无线密码（支持字母、数字、特殊字符，8-63位长度）
   - 网络：选择关联的本地子网
   - 用户隔离：勾选后，连接此SSID的终端不能相互访问
4. 点击"保存"完成配置。

### 场景5：VPN配置

**目标**：建立安全的VPN连接，实现远程访问或站点互联。

**前提**：设备已连接互联网，已获取对端VPN设备的配置信息。

**预计用时**：约15分钟。

**操作步骤**：

ER615支持IPsec VPN和L2TP VPN两种类型。以下以IPsec VPN为例：

1. 进入【VPN】→【IPsec VPN】菜单，点击"添加"按钮。
2. 配置以下参数：
   - 名称：设置IPSec VPN名称标识
   - IKE版本：选择IKEv1或IKEv2
   - 预共享密钥：配置与对端相同的认证密钥
   - 上行接口：选择建立VPN使用的上行接口
   - 对端地址：填写对端设备的公网地址
   - 本地标识/对端标识：用于IKE协商过程中的身份认证
   - 本地子网：填写需要通过隧道通信的本端网段
   - 对端子网：填写需要通过隧道通信的对端网段
3. 配置IKE策略和IPsec策略（加密方式、认证方式、DH组等）。
4. 点击"保存"完成配置。

#### 4.7.1 防火墙

**入站/出站规则**

用户可以通过“安全 > 防火墙 > 出站规则/入站规则”功能来实现基于接口的流量进出控制。例如，当用户受到来自特定源IP地址的大量攻击行为时，可以使用防火墙入站规则来限制来自该IP地址的流量。

此外，IT人员还可以通过防火墙出站规则来限制某些用户访问外部网络的权限。入站规则和出站规则在可配置的内容上相同，唯一的区别是默认规则。以下是添加入站规则的示例：

<p align="center"><img src="./img/OrYTAIbcHcjsKauQ/1765424948414-0fcb1e65-443b-4521-9057-fee79f7d02f8-408575.webp" alt="添加入站规则"></p>

<p align="center"><strong>图 4-39 添加入站规则</strong></p>

1. **名称**： 用于在本地界面中识别和管理不同的防火墙策略。名称仅作为标识，不影响实际流量匹配。  
2. **状态**：用于开启或禁用当前规则。

- 开启：规则生效并参与流量匹配。
- 关闭：规则不参与匹配，相当于未配置本规则。

3. **接口**：指定规则作用的上行接口。取值为： All WANs，WAN1、Cellular等。

- **入站规则**：匹配从外部网络进入路由器的流量，对应流量进入的 WAN 或上行接口。
- **出站规则**：匹配从内部网络访问外部网络的流量，对应流量发送出去的 WAN 或上行接口。

1. **协议**：匹配流量的协议类型，支持Any、TCP、UDP、ICMP、自定义。
2. **源**：匹配流量的源地址，支持自定义，默认Any。
3. **目的**：匹配流量的目的地址，支持自定义，默认Any。
4. **动作**：入站规则/出站规则的对匹配流量的动作，支持允许和拒绝。
5. **入站规则**：外部网络访问路由器的流量管理规则，默认拒绝所有。
6. **出站规则**：通过路由器访问外部的流量管理规则，默认全部放行。
7. **优先级**：系统支持通过拖动“优先级”栏图标来调整入站或出站规则的匹配顺序。  
   规则按自上而下的顺序依次匹配：

- 第一条匹配成功后即执行对应动作，不再继续向下匹配。
- 将更严格的规则放在更高位置可避免误放行或误拦截。

**端口转发**

端口转发也被称为端口映射或端口重定向，用于将网络数据包从一个网络端口（或地址）重定向到另一个网络端口或地址。用户可在“安全>防火墙>端口转发”配置端口转发规则，当外部流量访问路由器的特定端口时，设备会将数据转发至内部客户端的相应端口上，实现从外部访问路由器内部的服务。  


例：当用户需要从外部访问内部192.168.2.10客户端的1024端口服务时，将此客户端的端口映射到WAN1接口下的1024端口，外部网络只需在浏览器输入 “<https://WAN1地址:1024”即可访问目标设备数据，其中WAN1地址为WAN1接口的IP地址。>

<p align="center"><img src="./img/OrYTAIbcHcjsKauQ/1765425340269-db1596e6-f354-426a-acf6-8de10ba48e3d-653426.webp" alt="添加端口转发规则"></p>

<p align="center"><strong>图 4-40 添加端口转发规则</strong></p>

1. **名称**： 用于标识该端口转发规则的名称，便于管理与区分。名称仅用于本地识别，对实际转发行为无影响。  
2. **状态**： 端口转发规则的启用或禁用开关。启用后规则立即生效；关闭后该规则不再执行任何端口映射。  
3. **接口**： 指定端口转发规则生效的 WAN 接口。当设备具有多个上行接口（如 WAN1、WAN2、蜂窝网络）时，需要选择流量进入的对应接口。  
4. **协议**： 选择需要转发的协议类型，通常包括 TCP、UDP 或 TCP/UDP 双协议。设备将按所选协议匹配并转发端口流量。  
5. **公网端口**：外部访问设备时使用的端口号。来自互联网的流量将通过该端口进入并触发端口转发规则。  
6. **本地地址**： 端口映射目标的内部主机 IP 地址（LAN 侧）。外部流量将被转发至该内网主机。  
7. **本地端口**： 内部主机用于接收转发流量的端口号。如果公网端口和本地端口不同，则设备会在转发时自动进行端口转换。  

**网络地址转换**

NAT（Network Address Translation，网络地址转换）是一项将私有网络地址转换为公网地址的技术，使局域网内多个设备可以通过单一公网 IP 访问互联网。  
设备通过 NAT 功能实现以下目的：

1. **地址复用**：多个 LAN 设备共享一个或多个公网地址访问外网。
2. **安全增强**：隐藏内网结构，减少暴露面。
3. **灵活管理**：支持多种转换方式，适配不同网络部署场景。

常见 NAT 类型包括：

- **源地址转换（SNAT）**：将内网源 IP 替换为设备的公网 IP。
- **目的地址转换（DNAT）**：将访问公网 IP 的流量重定向到内网某台主机（常用于端口转发）。

用户可在“安全>防火墙>网络地址转换”中配置源IP地址转换或目的IP地址转换规则。

<p align="center"><img src="./img/OrYTAIbcHcjsKauQ/1765430261404-db46c47a-4048-420c-9bbc-a1b23a167d14-167205.webp" alt="添加网络地址转换规则"></p>

<p align="center"><strong>图 4-41 添加网络地址转换规则</strong></p>

1. **名称**：标识该地址转换规则.
2. **状态**：启用或停用该条规则。
3. **类型**：选择地址转换的类型
4. SNAT：源IP地址转换，以可将IP数据包中源IP地址字段的值转换为设定值
5. DNAT：目的IP地址转换，以可将IP数据包中目的IP地址字段的值转换为设定值。
6. **协议**：选择需要转换的IP数据包类型、
7. TCP：地址转换规则在TCP报文中生效。
8. UDP：地址转换规则在UDP报文中生效。
9. TCP&UDP：地址转换规则在TCP和UDP报文中生效。
10. **源**：数据包的源IP地址。
11. **目的**：数据包的目的IP地址。
12. **转换后地址**：根据所选择的地址转换类型，设置需要转换的值。

#### 4.7.2 策略路由

策略路由（Policy-Based Routing，PBR）是一种基于策略的网络路由机制，它允许管理员根据源地址、目的地址、服务类型等条件，将特定流量导向指定出口或接口，而不依赖于默认路由表的路由选择。

通过策略路由，可以实现以下应用场景：

- 不同业务或应用走不同的链路（如办公流量走专线，互联网流量走宽带）。
- 实现流量负载均衡或链路备份。
- 精确控制网络访问和安全策略。

策略路由通常在企业网络、分支机构网络或多WAN场景中使用，以提高网络灵活性和管理精度。

<p align="center"><img src="./img/OrYTAIbcHcjsKauQ/1765434309241-b125b60b-2e69-4b45-9979-3f4dc151523a-085408.webp" alt="配置策略路由"></p>

<p align="center"><strong>图 4-44 配置策略路由</strong></p>

- **名称**：为策略路由规则设置唯一标识，用于本地管理和识别。  
- **状态**：用于启用或禁用当前策略路由规则。
  - 开启：策略路由规则生效，匹配的流量将按规则转发。
  - 关闭：策略路由规则失效，流量按默认路由表转发。
- **源**： 指定匹配策略的流量源地址或子网。可设置单个 IP 地址、IP 段或 VLAN。流量从匹配源发出时，策略路由规则生效。
- **目的**：指定匹配策略的目标地址或子网。流量的目的地址与配置匹配时，才会应用策略路由。支持单个 IP、IP 段或特定服务地址。  
- **强制转发**：控制匹配流量是否强制按策略路由转发，忽略系统默认路由。
  - 开启：匹配流量严格按照策略出口转发，不参考系统路由表。
  - 关闭：匹配流量在策略出口不可达时，可回退到系统默认路由。

### 4.8 认证

802.1X 是一种基于端口的网络访问控制协议，用于在局域网（LAN）或无线局域网（WLAN）中对接入设备进行身份验证。通过启用 802.1X，网络能够确保只有经过认证的设备或用户才能访问网络资源，提高网络安全性。   该功能通常应用于企业、校园或分支机构网络中，可结合 RADIUS 服务器进行集中认证和管理。  

<p align="center"><img src="./img/OrYTAIbcHcjsKauQ/1765433015651-719e9621-580b-41e2-b3f0-11074d0a2390-714098.webp" alt="认证"></p>

<p align="center"><strong>图 4-45 认证</strong></p>

- **802.1X 功能开关**：用于启用或关闭 802.1X 功能。
  - 开启：启用 802.1X 认证功能，所有配置的认证端口将启用身份验证。
  - 关闭：关闭 802.1X 功能，端口不进行身份验证。

使用建议：在需要控制网络访问的场景下保持开启，否则关闭。

- **服务器**：指定进行 802.1X 认证的 RADIUS 服务器IP或域名。
- **认证端口**：用于启用 802.1X 身份验证的物理或逻辑端口。端口会在设备接入时触发认证请求，未通过认证的设备将被隔离。
- **计费端口**：指定 802.1X 认证通过后计费或统计流量的端口。仅在启用计费功能的网络中使用。

使用建议：如果不涉及计费或流量统计，可留空或禁用。

- **密钥**：用于 802.1X 与 RADIUS 服务器之间的安全通信。

**示例**

启用 802.1X 功能开关。

配置 RADIUS 服务器地址 192.168.1.100，端口 1812，密钥为 `MySecretKey123`。

将办公区端口 1~24 设置为认证端口。

启用计费端口 1~24，用于流量统计。

保存配置并应用，网络设备接入时将触发 802.1X 认证。 

**注意事项**

配置不当可能导致端口无法访问网络，请确保测试环境先验证认证配置。

802.1X 功能通常结合 VLAN 策略使用，可实现认证成功后动态分配网络资源。

RADIUS 服务器必须可达，否则认证端口的设备将被阻断访问。

#### 4.9.6 Dynamic DNS

动态 DNS（Dynamic DNS，简称 DDNS）用于在公网 IP 地址动态变化的场景下，将设备的当前 IP 地址自动更新到指定的域名服务商，使域名始终指向设备的最新公网地址。当运营商提供的是动态公网 IP（如宽带、蜂窝网络）时，DDNS 允许用户通过固定域名远程访问设备，而无需关心 IP 是否发生变化。



常见应用场景包括：

- 远程管理路由器或 VPN 网关
- 搭建远程监控或服务器
- 在动态公网环境中保持稳定的远程访问入口

用户可在“服务> Dynamic DNS”功能下手动配置动态DNS服务器地址。

<p align="center"><img src="./img/OrYTAIbcHcjsKauQ/1765439428339-d8e82890-f595-4a12-bbdd-609d80607b98-174490.webp" alt="配置动态DNS服务"></p>

<p align="center"><strong>图 4-52 配置动态DNS服务</strong></p>

1. **服务提供商**：由动态DNS运营商提供，您可以选择dyndns，3322，oray，no-ip  或自定义（需要URL）。
2. **域名**： 用户在 DDNS 服务商处注册并获得的域名。
3. **用户名**： 登录 DDNS 服务商的账号。
4. **密码**： 对应服务商账号的密码。
5. **更新间隔（分钟）**： 设置设备向 DDNS 服务商更新 IP 的时间间隔，例如 5 分钟、10 分钟等  
6. **上行接口**：上行链路获取公网IP提供DDNS服务的接口

- 当前的SD-WAN网络支持中心辐射型拓扑，设备角色分为中心设备和分支设备。所有分支设备都通过中心设备创建隧道，分支设备之间的流量通过中心设备。

  

  中心设备：

  1. 中心设备需要公网IP地址来确保SD-WAN网络的运行。用户还可以通过IP映射来解决公网IP不足的问题。
  2. 在具有公网IP地址的中心设备上行接口与分支设备的所有上行接口之间创建隧道。
  3. 在防火墙规则中，中心设备的上游设备需要允许两个端口号，并将它们映射到ER系列路由器的上游接口。端口范围为1-65535。
  4. 支持的设备型号：ER815、ER2000
  5. 最多可以添加5个设备。

  分支设备：

  1. 分支设备对公共 IP 地址没有特定要求。
  2. 可以添加多个分支设备，最终数量由中央设备的性能决定。
  3. 支持的设备型号：ER615、ER815。

  

  **添加设备**

  在“添加网络”页面上，单击中心设备或分支设备的“添加”按钮，具体取决于要添加到网络的设备类型。选择设备后，提供设备的公网映射信息。如果需要修改设备的网络配置，可以点击本地网络的“编辑”按钮进行远程配置。

## 第5章 典型应用

### 案例1：连锁门店网络组网

**场景描述**

某连锁零售企业需要为分布在各地的门店铺设网络，实现门店与总部之间的数据传输和远程管理。门店数量多、分布广，传统专线成本高昂，且需要专业的IT人员进行现场维护。

**网络拓扑**

```
                    总部数据中心
                         |
                   [核心交换机]
                         |
                  [ER815 中心设备]
                         |
         +---------------+---------------+
         |               |               |
    [门店1]          [门店2]         [门店3]
   [ER615]          [ER615]         [ER615]
   分支设备         分支设备        分支设备
```

**设备角色**

- 总部ER815：作为SD-WAN中心设备，负责与各门店建立VPN隧道，集中管理所有分支设备的网络接入
- 门店ER615：作为SD-WAN分支设备，通过蜂窝网络或本地宽带接入互联网，与总部建立安全隧道

**配置步骤**

1. **总部中心设备配置**
   - 确保ER815拥有公网IP地址
   - 在防火墙上开放SD-WAN所需的端口（1-65535范围内指定端口）
   - 将端口映射到ER815的上行接口
   - 登录小星云管家平台，创建SD-WAN网络
   - 添加中心设备，选择总部ER815

2. **门店分支设备配置**
   - 在门店安装ER615，插入SIM卡并安装天线
   - 确认设备通过蜂窝网络正常联网
   - 将设备添加到小星云管家平台
   - 在SD-WAN网络中添加分支设备，选择门店ER615

3. **网络验证**
   - 在小星云管家平台查看SD-WAN拓扑，确认所有隧道状态为"已连接"
   - 从门店Ping总部服务器地址，确认网络连通
   - 测试业务系统访问是否正常

## 附录 安全注意事项

1. 请使用原装电源适配器，避免因电源适配器和设备不匹配造成设备损坏。

2. 安装设备时请勿将设备安装在强电磁干扰环境下，避免与高功率设备过于接近。安装完成后请确认设备是否易发生晃动，避免因设备跌落造成设备损坏。

3. 请确认设备使用环境满足规格书所要求的温度和湿度。

4. 请定期检查设备线缆，包括网线、电源适配器连接线，保持线材清洁，如有发生破损请及时更换。

5. 清洁设备时，避免使用化学试剂直接喷洒在设备表面，避免损害外壳或内部组件。需要使用柔性的布料清洁产品。

6. 请勿私自拆修或改装设备，这将会导致安全问题并且设备会失去保修资格。

> **警告**：非专业人员请勿打开设备外壳，存在触电风险。

## FAQ 常见问题解答

### 问题1：ER路由器与普通路由器区别？

1. 边缘路由器：ER615支持多种联网方式（有线/蜂窝），多千兆网口和WiFi6芯片设计，支持各类终端设备接入，同时设备支持SD-WAN、带外管理和云管理。

2. 普通路由器：通常依赖固定的宽带连接，如DSL、光纤，通过有线连接到网络。普通路由器无统一的管理平台和防火墙、SD-WAN等高级特性。

### 问题2：4G/5G无法联网？

1. 物理环境：首先检查SIM卡是否插到正确的卡槽位，蜂窝天线是否全部安装。

2. APN设置：确保APN的配置信息与运营商提供的一致。

3. 检查设备连通性：登录到设备本地界面，通过系统自带的Ping工具，ping 8.8.8.8测试是否连通，若可连通，则检查设备（如电脑、手机）与路由器的连通性。

4. 判断SIM卡是否正常：取出SIM卡，安装在手机中查看是否可以正常联网。

5. 重启：尝试将路由器断电，等待几秒钟，然后重新连接电源，重试网络连接。

6. 恢复出厂：将路由器恢复出厂，然后重新尝试。

### 问题3：云平台是否免费？

映翰通一直致力于为中小连锁机构提供优质的网络服务，用户使用云平台服务时，需要逐台购买许可费用，以享受丰富的云端功能。

### 问题4：设备如何添加到云平台？

1. 首先通过https://star.inhandcloud.cn网址注册小星云管家登录账号。

2. 使用注册账号登录到云平台，在设备菜单下点击"添加"，按照提示填写设备的序列号和MAC地址（参照设备背面铭牌），即可完成添加。设备首次添加时，默认赠送1年免费基础版许可，后续用户可按需续费。

### 问题5：设备是否可以不用云平台？

1. 可以。用户可以在本地完成绝大多数配置功能，但批量的配置下发、升级、SD-WAN、Connector等功能，需要结合云平台使用。
