## 第1章 认识设备

### 1.1 概述

ES620（Enterprise Switch 620）是北京映翰通网络技术股份有限公司推出的全千兆PoE二层网管交换机。该设备支持零配置上线，组网后可自动接入云平台，实现即插即用部署。通过小星云管家服务，可实现设备的全生命周期管理，适用于商业联网场景的高速网络接入需求。

### 1.2 指示灯说明

| 指示灯 | 状态 | 含义 |
|--------|------|------|
| PWR | 灭 | 设备未上电 |
| | 绿色长亮 | 供电中 |
| SYS | 绿色闪烁 | 系统工作中 |
| Link | 灭 | 端口无连接 |
| | 绿色长亮 | 协商速率1000Mbps，已建立连接 |
| | 黄色长亮 | 协商速率10/100Mbps，已建立连接 |
| | 绿色闪烁 | 协商速率1000Mbps，正在交互数据 |
| | 黄色闪烁 | 协商速率10/100Mbps，正在交互数据 |
| PoE | 灭 | 该端口无PD设备接入 |
| | 绿色长亮 | 该端口正在供电 |

### 1.3 恢复出厂设置

<p align="center"><img src="./img/image4.webp" alt="恢复出厂设置按键"></p>

<p align="center"><strong>图 1-2 恢复出厂设置按键</strong></p>

1. 长按RESET按键的同时为设备上电，待SYS指示灯长亮后，在10秒内松开RESET按钮。
2. 待SYS指示灯熄灭后，长按RESET按钮，待SYS指示灯长亮。
3. 松开RESET按钮，此时SYS指示灯长亮3秒后快闪，设备开始执行恢复程序。

| 用户名和密码 | adm/123456 |

## 第2章 安装与首次使用

### 2.1 安装指南

**电源连接**

1. 连接电源线，等待系统启动完成。

**首次访问Web管理界面**

1. 使用网线将PC连接至交换机任一端口。
2. 将PC设置为静态IP地址，IP地址设置为192.168.20.2~192.168.20.254中的任一地址，默认网关为192.168.20.1，子网掩码为255.255.255.0。

<p align="center"><img src="./img/image5.webp" alt="PC配置静态IP地址"></p>

<p align="center"><strong>图 2-1 PC配置静态IP地址</strong></p>

3. 在浏览器地址栏中输入192.168.20.1，输入用户名和密码（默认adm/123456）后进入设备Web管理页面。若网页提示不安全，点击"隐藏"或"高级"，选择"继续前往"。

<p align="center"><img src="./img/image6.webp" alt="通过Web页面登录"></p>

<p align="center"><strong>图 2-2 通过Web页面登录</strong></p>

> **注意**：当交换机连接至启用DHCP Server的三层设备时，由于交换机默认开启DHCP Client功能，此时交换机将会获得一个自动分配的IP地址。若PC通过"自动获得IP地址"方式联网，连接至交换机的PC也会从该三层设备获得IP地址，且该地址与交换机获取的地址处于同一网段。获取到IP地址后，交换机自身的192.168.20.1地址将无法访问。需从三层设备的客户端详情页查看交换机获取到的地址，并在PC上访问该地址以进入设备管理页面。

### 2.2 快速检查

| 检查项 | 检查内容 | 正常状态 |
|--------|----------|----------|
| 电源 | 设备是否已上电 | PWR指示灯绿色长亮 |
| 系统 | 系统是否正常启动 | SYS指示灯绿色闪烁 |
| 网络连接 | PC与交换机连接是否正常 | Link指示灯绿色或黄色长亮 |
| Web访问 | 能否正常登录管理页面 | 可打开192.168.20.1并登录 |

## 第3章 常用场景配置

### 场景1：接入上层网络

**目标**：通过网线将交换机连接至上层网络设备，实现网络接入。

**前提**：交换机已上电，上层设备已启用DHCP Server功能。

**预计用时**：约5分钟。

**操作步骤**：

1. 使用网线将交换机任一网口连接至上层网络设备（如路由器或三层交换机）。
2. 交换机默认启用DHCP Client功能，将自动从上层设备获取IP地址。
3. 登录上层设备的管理页面，在客户端列表中查看交换机获取到的IP地址。

<p align="center"><img src="./img/image7.webp" alt="设置交换机联网方式"></p>

<p align="center"><strong>图 3-1 设置交换机联网方式</strong></p>

**验证方法**：

1. 检查交换机Link指示灯状态，确认端口已建立连接。
2. 通过获取到的IP地址访问交换机Web管理页面，确认可正常登录。

### 场景2：连接至小星云管家

**目标**：将设备添加至小星云管家云平台，实现远程监控与管理。

**前提**：设备已接入互联网，已获取有效的IP地址；已准备设备序列号和MAC地址。

**预计用时**：约10分钟。

**操作步骤**：

1. 在浏览器中输入网址 <https://star.inhandcloud.cn>，进入小星云管家门户页面，点击"立即创建"以注册平台账户。

<p align="center"><img src="./img/image8.webp" alt="创建小星云管家账号"></p>

<p align="center"><strong>图 3-2 创建小星云管家账号</strong></p>

2. 使用注册时的用户名和密码登录映翰通平台，并选择【小星云管家】服务。

<p align="center"><img src="./img/image9.webp" alt="选择SaaS平台"></p>

<p align="center"><strong>图 3-3 选择SaaS平台</strong></p>

3. 登录小星云管家服务后，进入【设备】菜单，点击"添加"按钮，填写设备名称、序列号、MAC地址，点击"完成"。

<p align="center"><img src="./img/image10.webp" alt="将设备添加至平台"></p>

<p align="center"><strong>图 3-4 将设备添加至平台</strong></p>

> **说明**：设备第一次接入云平台时，云平台会自动下发一个有效期一年的基础版许可证。用户可以在"企业管理→订阅→许可证"中续费或将许可证升级为分支专业版。将设备添加到平台后，在网络正常的情况下设备即可上线，无需其他配置。

**验证方法**：

1. 登录小星云管家平台，在设备列表中查看该设备是否显示为在线状态。
2. 在"企业管理→订阅→许可证"中查看设备许可证是否已下发。

**常见问题**：

- 设备无法上线：检查设备网络连接是否正常，确认序列号和MAC地址填写无误。
- 许可证未下发：检查设备是否已完成首次接入并处于在线状态。

## 第4章 功能说明与参数参考

### 4.1 设备监控

将设备添加到平台后，可从平台管理和监控网络，同时支持在设备本地界面上远程查看实时状态信息。

#### 4.1.1 设备总览

##### 概览

在【设备】列表中，点击【设备名称】可访问设备的详细信息页面。点击左侧菜单中的【仪表盘】，进入设备的信息仪表盘界面。在该界面中可查看设备基本信息、接口状态、与平台的连接/断开记录、许可证状态等。

<p align="center"><strong>图 4-1 从云端查看设备信息</strong></p>

小星云管家提供云端下载设备运行日志和诊断日志的入口。在"设备→设备详情→诊断工具"页面可下载诊断日志以及运行日志，用于检查设备状态及故障排查。

##### 设备信息

在【仪表盘】界面顶部可查看设备基本信息，包括设备名称、设备型号、序列号、MAC地址、在线时长、设备管理地址等。

<p align="center"><img src="./img/image15.webp" alt="查看设备信息"></p>

<p align="center"><strong>图 4-5 查看设备信息</strong></p>

- **名称**：标识设备的名称，默认为"ES620"，支持修改。
- **MAC地址**：标识设备的MAC地址。
- **型号**：设备具体的型号，用于判断设备的硬件特性。
- **在线时长**：设备此次上电以来的运行时间。
- **序列号**：唯一标识设备的编码，用于索引设备或将设备添加到云平台。
- **设备管理地址**：在设备未通过DHCP获取到地址时，默认为192.168.20.1，通过该地址可访问设备Web页面；当设备通过DHCP获取到地址后，192.168.20.1地址失效，可通过获取到的地址访问设备Web页面。
- **固件版本**：设备正在运行的软件版本号。
- **许可证状态**：应用到设备的许可证信息，分为小星云管家基础版和小星云管家分支专业版。
- **系统时间**：设备所在的时区以及当前时刻。

##### 日志

通过"状态→日志"功能可查看设备系统运行的日志信息。设备出现故障时，技术人员可根据系统日志进行问题排查。

<p align="center"><img src="./img/image18.webp" alt="查看设备运行日志"></p>

支持对日志进行以下操作：

- **清除日志**：清除设备此次上电以来产生的运行日志，保留设备的诊断日志。
- **下载日志**：下载设备的运行日志。
- **诊断日志**：下载设备的诊断日志，包含系统运行日志、设备信息、设备配置等。
