## 第1章 认识设备

### 1.1 概述

InGateway502（简称IG502）是映翰通公司面向工业物联网领域推出的高性价比边缘计算网关。该设备尺寸小巧，接口丰富，具备便捷的全球蜂窝接入能力。IG502支持用户使用Python进行二次开发，可内置映翰通DeviceSupervisor™ Agent服务，支持上百种数据采集协议，实现设备数据采集、处理及上云，同时支持映翰通DeviceLive云管理，帮助企业加快数字化进程。IG502适用于工业自动化、智能电网、环境监测、智慧交通等多种工业物联网应用场景。

### 1.5 恢复出厂设置

IG502支持通过硬件方式恢复出厂设置，步骤如下：

1. 设备上电后10s内长按RESET键不松开；
2. 当ERR灯变红后，松开RESET键；
3. 几秒之后当ERR灯熄灭，再次按住RESET键不松开；
4. 当看到ERR灯闪烁时松开RESET键；等待ERR灯熄灭，表明恢复出厂设置成功。

#### 2.1.1 环境要求

| 项目 | 规格 |
|------|------|
| 输入电压 | 12-48 VDC（双引脚端子，V+、V-） |

- 导轨或壁挂安装套件

#### 2.2.4 电源连接

IG502支持12-48V DC供电。将适配器端子插入到IG502的DC端口中，然后连接电源适配器。当PWR电源指示灯长亮则代表设备已经正常上电。

<p align="center"><img src="./img/sJjMR0G-wgzWVDLu/1724982551003-e153c577-e3f1-4d5d-88c1-8abc2ecb0cfc-923948.png" alt="电源/串口端子"></p>

<p align="center"><strong>图 2-13 电源/串口端子</strong></p>

电源、串口采用7PIN端子，接口引脚说明如下：

| 引脚 | 名称 | 定义 |
|------|------|------|
| 1 | V+ | 电源正极 |
| 2 | V- | 电源负极 |
| 3 | TXD或1A | 串口RS232发送或第一路RS485+ |
| 4 | RXD或1B | 串口RS232接受或第一路RS485- |
| 5 | GND | 串口RS232信号地 |
| 6 | 2A | 第二路RS485+ |
| 7 | 2B | 第二路RS485- |

> **注意**：当接口组合为RS232+RS485时，RS232串口对应的设备节点为/dev/ttyO1，RS485串口对应的设备节点为/dev/ttyO3。当接口组合为RS485+RS485时，RS485-1串口对应的设备节点为/dev/ttyO1，RS485-2串口对应的设备节点为/dev/ttyO3。

##### 2.2.5.3 USB接口

IG502提供一个USB 2.0 Host接口，主要用于扩展存储设备。IG502支持USB存储设备热插拔。如果一个USB存储设备有多个分区，IG502可以自动挂载前9个分区，其余的分区需要手动挂载。IG502会把所有USB存储设备分区挂载到/mnt/usb路径下，挂载文件夹的命名格式为sda_<num>。其中<num>可以是1~9的数字。

<p align="center"><img src="./img/sJjMR0G-wgzWVDLu/1729237027099-35b56d98-fb3e-4ba3-bf03-79dd107f185e-653137.png" alt="USB接口"></p>

<p align="center"><strong>图 2-16 USB接口</strong></p>

> **注意**：在断开USB大容量存储设备之前，需输入sync同步命令，以防止数据丢失。断开存储设备时，需从/mnt/usb/sda_<num>目录下退出。如果留在/mnt/usb/sda_<num>中，则自动卸载过程将会失败。如果发生这种情况，需键入umount /mnt/usb/sda_<num>以手动卸载设备。

##### 2.2.5.4 Micro SD卡槽

IG502有一个Micro SD卡槽，SD卡不支持热插拔，需要在断电的情况下操作。

<p align="center"><img src="./img/sJjMR0G-wgzWVDLu/1729243709704-6dcdeff6-64e4-4aa7-9272-1c011c1065e4-070517.png" alt="Micro SD卡槽"></p>

<p align="center"><strong>图 2-17 Micro SD卡槽</strong></p>

如果一个Micro SD存储设备有多个分区，IG502可以自动挂载前9个分区，其余的分区需要手动挂载。IG502会把所有Micro SD存储设备分区挂载到/mnt/sd路径下，挂载文件夹的命名格式为mmcblk0p<num>。其中<num>可以是1~9的数字。

Web的概览页面只显示第一个分区的容量和使用情况，为了方便在Web页面查看SD卡使用情况，建议只创建一个分区，并将该分区格式化为FAT32文件系统。

- SD卡容量 <= 32GB：SD卡插入读卡器，然后将读卡器插入使用Windows系统的电脑。大部分Windows系统只支持将容量小于32GB的SD卡格式为FAT32系统，格式化的时候会自动创建一个分区；也可以在Linux系统下格式化为FAT32文件系统。
- SD卡容量 > 32GB：SD卡容量大于32GB时，需要在Linux下格式化FAT32格式。
  1. 执行fdisk -l查看SD卡对应的设备节点，示例中是/dev/sda
  2. 执行fdisk /dev/sda进行分区（p查看当前分区，d删除已有分区，n新创建分区，w保存更改），创建分区/dev/sda1
  3. 执行sudo mkfs.vfat -F 32 /dev/sda1将分区格式化为FAT32格式。

## 第3章 常用场景配置

### 场景1：蜂窝联网

**目标**：通过4G/5G蜂窝网络接入互联网。

**前提**：已插入SIM卡并安装天线，设备已上电。

**预计用时**：约5分钟。

**操作步骤**：

1. 进入【网络】→【网络接口】→【蜂窝网】菜单。
2. 点击"启用蜂窝网"按钮。
3. 配置拨号参数集（APN、用户名、密码、认证方式等，专网卡需配置，普通SIM卡通常使用默认即可）。
4. 选择网络类型（自动/2G/3G/4G/5G）。
5. 如需双卡冗余，启用"启用双SIM卡"功能，配置主卡选择策略、最大拨号次数、最小连接时间等参数。
6. 点击"保存"并等待连接建立。

**验证方法**：

1. 检查NET指示灯状态，确认网络连接正常（NET常亮表示拨号成功）。
2. 在概览页面查看蜂窝网连接状态，确认已获取IP地址。

**常见问题**：

- 网络连接失败：检查SIM卡是否正确插入、APN参数是否正确。
- 数据发送接收异常：检查信号强度和流量余额。

### 场景2：以太网联网

**目标**：通过以太网接口接入局域网或互联网。

**前提**：网线已连接至上级网络设备或路由器。

**预计用时**：约3分钟。

**操作步骤**：

1. 进入【网络】→【网络接口】→【以太网】菜单。
2. 选择要配置的以太网接口。
3. 选择网络类型：
   - 动态地址（DHCP）：自动获取IP地址和子网掩码
   - 静态IP：手动配置IP地址、子网掩码、网关、DNS
4. 如需配置从IP地址，在"从IP支持"中添加（最多10个）。
5. 点击"提交"应用配置。

**验证方法**：

1. 在以太网配置页面查看接口状态，确认IP地址已获取或配置成功。
2. 在【系统管理】→【工具】中使用Ping工具测试外网连通性。

**常见问题**：

- 无法获取IP地址：检查上级网络设备DHCP服务是否正常，或确认静态IP配置是否正确。
- 无法访问外网：检查网关和DNS配置是否正确。

### 场景3：WLAN配置

**目标**：配置WLAN为接入点（AP）或客户端（Client）模式。

**前提**：设备型号支持WLAN功能（具体支持情况参见产品规格书）。

**预计用时**：约5分钟。

**操作步骤**：

1. 进入【网络】→【网络接口】→【WLAN】菜单。
2. 启用Wi-Fi。
3. 选择接口类型：
   - 接入点（AP）：配置SSID、认证方式、加密方式、WPA/WPA2 PSK密钥、信道、最大客户端数等
   - 客户端（Client）：配置要连接的SSID名称、认证方式、加密方式、WPA/WPA2 PSK密钥、网络类型（静态IP/DHCP）
4. 点击"保存"应用配置。

**验证方法**：

1. AP模式：使用无线设备扫描SSID，确认可以搜索到并连接。
2. Client模式：在WLAN配置页面查看连接状态，确认已关联到目标AP。

**常见问题**：

- 无线设备无法扫描到SSID：检查SSID广播是否开启，无线频段和信道配置是否正确。
- 客户端无法连接AP：检查认证方式和密钥是否与目标AP一致。

### 场景4：VPN配置

**目标**：建立IPsec或OpenVPN隧道，实现安全远程通信。

**前提**：两端设备网络可达，已获取对端IP地址或域名。

**预计用时**：约10分钟。

**操作步骤（以IPsec为例）**：

1. 进入【高级功能】→【VPN】→【IPsec】菜单。
2. 配置IKEv1/IKEv2策略：选择加密算法、哈希算法、Diffie-Hellman密钥交换组、生命周期。
3. 配置IPsec策略：选择封装方式（AH/ESP）、加密算法、认证方式、IPsec模式（隧道模式/传输模式）。
4. 配置IPsec隧道：
   - 设置对端地址、接口名称、IKE版本、策略
   - 选择认证方式（共享密钥/数字证书）
   - 配置本地子网地址和对端子网地址
5. 点击"保存"并等待隧道建立。

**验证方法**：

1. 在IPsec配置页面查看隧道状态，确认SA已建立。
2. 在【系统管理】→【工具】中使用Ping工具测试对端子网连通性。

**常见问题**：

- 隧道无法建立：检查两端加密算法、哈希算法、认证密钥是否一致。
- 数据传输不通：检查本地子网和对端子网配置是否正确，ACL或防火墙是否放行了相关流量。

### 场景5：Python边缘计算

**目标**：启用Python二次开发环境，部署和运行自定义App。

**前提**：设备固件版本支持Python边缘计算功能。

**预计用时**：约10分钟。

**操作步骤**：

1. 进入【边缘计算】→【Python边缘计算】菜单。
2. 启用Python边缘计算引擎。
3. 安装/升级Python SDK。
4. 启用调试模式。
5. 在App列表中添加App，上传App包并配置启动参数。
6. 启动App并查看运行日志。

**验证方法**：

1. 在Python边缘计算页面查看App状态，确认App状态为"运行中"。
2. 点击"查看"按钮查看App运行日志，确认无报错信息。

**常见问题**：

- App无法启动：检查App包格式是否正确，启动参数是否配置正确。
- 日志无输出：确认调试模式已启用，检查App代码逻辑。

### 场景6：设备云平台上云

**目标**：将IG502连接到映翰通设备云平台，实现远程管理和监控。

**前提**：已在云平台注册账户。

**预计用时**：约5分钟。

**操作步骤**：

1. 进入【系统管理】→【设备云平台】菜单。
2. 选择要连接的平台（InHand Connect Service / Device Manager / iSCADA Cloud / Device Live Manager）。
3. 启用连接功能。
4. 选择或自定义服务器地址。
5. 输入云平台注册账户名称。
6. 如需安全通道，在高级设置中启用"启用安全通道"。
7. 点击"保存"并等待设备上线。

**验证方法**：

1. 在设备云平台页面查看连接状态，确认设备已在线。
2. 登录云平台Web界面，确认可以在设备列表中看到该设备。

**常见问题**：

- 设备无法上线：检查服务器地址和注册账户是否正确，确认设备网络可以访问云平台服务器。
- 云平台无法管理设备：检查心跳间隔和流量上报间隔配置是否合理。

  - 网络类型：用户选择设备所使用的移动网络类型，可选择GSM、CDMA。
  - APN（Access Point Name）：APN用来标识WCDMA/LTE网络的业务种类，WCDMA/LTE系统根据用户连接WCDMA/LTE网络的APN提供相应的服务。由运营商提供（CDMA2000系列不设置此项）。
  - 拨号号码：拨号使用的拨号串。拨号串由运营商提供，请向运营商获取。
    - 当3G/LTE数据卡支持WCDMA或LTE标准时，缺省拨号串为*99***1#。
    - 当3G数据卡支持CDMA2000标准时，缺省拨号串为#777。
  - 认证方式
    - 自动：自动选择一种认证方式
    - PAP：密码认证协议，通过两次握手提供一种简单明文认证方式。
    - CHAP：挑战握手认证协议，通过三次握手确认摘要信息从而进行安全认证。
    - MS-CHAP：微软公司的CHAP标准。
    - MS-CHAPv2：MS-CHAP升级版本，它要求双向验证。
  - 用户名：指定接入外部PDN网络用户的用户名。由运营商提供。缺省用户名为gprs。
  - 密码：指定接入外部PDN网络用户的密码。由运营商提供。缺省用户名为gprs。

**短信配置详情如下：**

**1、短信设置蜂窝拨号参数**

短信格式：

```
code=xxx cellular slot=sim1 apn=hello username=xxx password=xxx network=4g dialnumber=*99*1 auth=pap
cellular slot=sim1 apn=hello username=xxx password=xxx network=4g dialnumber=*99*1 auth=pap
```

蜂窝拨号参数配置说明如下：

- code：页面设置的短信密码，若设置了密码，必须以code开头，必选项。没有设置密码时按第二种格式下发指令。
- cellular：表示设置cellular服务相关的参数，cellular关键字的位置需要在短信内容的开始部分；cellular后面的内容是cellular的配置项。
- slot：标志设置哪张卡的拨号参数，可设置的值为：sim1、sim2；
- 必填参数apn：配置apn信息；
- 非必填参数username：配置用户名；
- 非必填参数password：配置密码；
- 非必填参数auth：认证方式；可设置的值为：none、auto、pap、chap；严格区分大小写；
- 非必填参数network：配置网络制式；可设置的值为：2g、3g、4g、auto；严格区分大小写；
- 非必填参数dialnumber：拨号号码。

**注意事项：**

1. 非必填参数需要至少填写1项；除了cellular必须在短信内容的开始部分，其他参数的顺序没有要求；每个参数之间用空格分隔；
2. 参数名称和部分参数值严格区分大小写；
3. 如果短信内容中出现不识别的配置项，例如icmp_interval=10，则配置会失败；
4. 短信内容的长度不能超过140个字符；
5. 由于默认参数集不允许修改，当sim1/sim2选用默认参数集的时候，通过短信配置拨号参数会失败。

**2、短信开启双卡**

短信格式：

```
code=xxx cellular enable_dualsim=0
cellular enable_dualsim=1
```

- enable_dualsim：是否开启双卡，0表示关闭，1表示开启

**3、短信执行切卡**

短信格式：

```
code=xxx cellular switch_to=sim1
cellular switch_to=sim2
```

- switch_to：切换至目标卡，sim1表示切换至sim1卡槽，sim2表示切换至sim2卡槽

**4、短信重启系统**

短信格式：

```
code=xxx system command=reboot
system command=reboot
```

- system：表示执行系统相关的设置；system关键字的位置需要在短信内容的开始部分；
- command：表示需要执行的命令，可取值：reboot；必填参数。

**5、短信查询拨号信息**

短信格式：

```
code=xxx status object=cellular
status object=cellular
```

- status：表示执行状态查询；status关键字的位置需要在短信内容的开始部分；
- object：表示需要查询哪个服务的状态，可取值：cellular；必填参数。

发送查询短信后，会收到如下内容的短信：

```
cellular_status signal=none network=4g hostname=EdgeGateway uptime=150 ip=10.48.104.247
```

回复内容参数说明如下：

1. cellular_status：蜂窝的状态
2. signal：蜂窝的信号质量；可取值：none、poor、moderate、good、great
3. network：网络制式；可取值：2g、3g、4g、unknown
4. hostname：设备的主机名
5. ip：拨号接口的ip地址
6. uptime：设备的启动时长，单位s

**6、短信设备数字量输出DO**

短信格式：

```
code=xxx io do0=1 do1=1 do2=0 do3=0
io do0=1 do1=1 do2=0 do3=0
```

DO设置参数说明：

1. io：表示设置io服务相关的参数，io关键字的位置需要在短信内容的开始部分
2. do0~do3：配置do0~do3的输出，可以配置多个do，配置项之间没有顺序要求，比如可以这样配置：io do0=1 do1=1 do3=0 do2=0；也可以只配置一个do，比如io do0=1；至少需要配置1个do；do的取值只能为0或1

**7、短信接收DI的变化**

使用该功能，需要在io页面配置接收短信的手机号，同时在拨号页面将该手机号加入白名单中。

<p align="center"><strong>图 4-10 短信白名单配置</strong></p>

###### 4.6.1.2.1 Radius

RADIUS协议采用了客户机/服务器（C/S）工作模式。RADIUS协议实体由三个部分组成：用户端、RADIUS客户端（NAS）、RADIUS服务器。其认证流程如下图所示：

<p align="center"><img src="./img/WnWQlWcKwJH_STyE/1720522735071-08ef4552-d966-406f-82fd-284cbf1d4cdf-127996.png" alt="RADIUS认证流程"></p>

<p align="center"><strong>图 4-81 RADIUS认证流程</strong></p>

认证步骤如下：

1. 当用户接入网络时，用户发起连接请求，向RADIUS客户端发送用户名和密码。
2. RADIUS客户端向RADIUS服务器发送包含用户名和密码信息的认证请求报文。（NAS和RADIUS服务器采用预共享密钥的方式，用户密码使用该共享密钥通过MD5加密算法进行加密隐藏，增加了密码的安全性）
3. RADIUS服务器对用户身份的合法性进行检验：
   - 如果用户身份合法，RADIUS服务器向RADIUS客户端返回认证接受报文，允许用户进行下一步动作。由于RADIUS协议合并了认证和授权的过程，因此认证接受报文中也包含了用户的授权信息。
   - 如果用户身份不合法，RADIUS服务器向RADIUS客户端返回认证拒绝报文，拒绝用户访问接入网络。
4. RADIUS客户端通知用户认证是否成功。

###### 4.6.1.2.3 LDAP

轻量级目录存取协议LDAP（Lightweight Directory Access Protocol）是基于TCP/IP的目录访问协议，主要用于存储不常变数据，如电子邮件地址、联系人列表等。LDAP定义多种操作，通过绑定和查询操作可实现用户认证和授权功能。LDAP基于Client/Server结构，目录信息存于服务器上，且不支持计费。LDAP认证流程如下图所示：

<p align="center"><img src="./img/WnWQlWcKwJH_STyE/1720522736373-c9d7844b-d435-4d92-87dd-f41e2c378551-136276.png" alt="LDAP认证流程"></p>

<p align="center"><strong>图 4-85 LDAP认证流程</strong></p>

LDAP认证说明如下：

1. 当用户需要访问LDAP服务器时，用户输入用户名和密码，向设备发起认证请求
2. 设备获取到用户的用户名和密码，以管理员DN和管理员密码为参数向LDAP服务器发送管理员绑定请求报文以获得查询权限。
3. LDAP服务器收到管理员绑定请求报文后，验证管理员DN和管理员密码是否正确。如果管理员DN和管理员密码正确，则绑定成功，LDAP服务器向设备发送管理员绑定响应报文。
4. 设备收到绑定响应报文后，以用户输入的用户名为参数构造过滤条件，向LDAP服务器发送用户DN查询请求报文。例如：构造过滤条件为CN=User2。
5. LDAP服务器收到用户DN查询请求报文后，根据报文中的查询起点、查询范围、以及过滤条件，对用户DN进行查找。如果查询成功，则向设备发送查询成功的响应报文。查询得到的用户DN可以是一个或多个。
6. 设备根据查询得到的用户DN和用户输入的密码为参数，向LDAP服务器发送用户绑定请求报文。
7. LDAP服务器收到用户绑定请求报文后，检查用户输入的密码是否正确。
   - 如果用户输入的密码正确，则向设备发送绑定成功的绑定响应报文。

8. 认证成功后，设备通知用户登录成功。

#### 4.6.3 链路备份

为了保持网络的稳定性，在设备组成的网络环境中，通常都使用一些备份连接，以提高网络的健壮性、稳定性，这里的备份连接也称为备份链路或者冗余链路。

##### 4.6.3.1 SLA

InHand SLA基本原理：1.对象跟踪：对指定对象的可达性进行跟踪。2.SLA probe：对象跟踪功能可使用InHand SLA向对象发出不同类型的探测。3.使用静态路由和跟踪选项。SLA配置步骤如下：

1. 定义一个或多个SLA操作（探测）。
2. 定义一个或多个跟踪(Track)对象，以跟踪SLA操作的状态。
3. 定义与跟踪对象相关联的措施。

SLA参数说明如下：

- 标识：SLA的索引或ID，用户自定义也可自动生成，最多可添加10条SLA
- 类型：探测的类型，默认为icmp-echo，通过简单发送一个ICMP-ECHO（Type8）数据包到目标主机，如果ICMP-ECHO-Reply（ICMPtype0）数据包接收到，说明主机是存活状态

#### 4.6.4 VPN

VPN是在公共网络（如互联网）基础上构建的"虚拟"私人专用通信网络。所谓"虚拟"，是指它并非像传统的专用网络那样拥有独立的物理链路和基础设施，而是通过网络技术在公共网络上建立起一个逻辑上隔离且专用的通信通道，实现类似于专用网络的功能和安全性。

通过隧道技术、加密技术等手段，使得用户在使用VPN时感觉像是在一个独立、安全的专用网络中进行通信，但实际上数据是在公共网络的基础上传输的，只是在逻辑上实现了专用性和私密性。

##### 4.6.4.1 IPsec

IPsec是IETF制定的一组开放的网络安全协议，在IP层对数据包进行认证、加密操作来保证通信数据的完整性、安全性。减少数据泄漏和被窃听的风险，保障了用户业务数据传输的安全。

    - DES：使用64bit的密钥对一个64bit的明文块进行加密

    - ESP：ESP封装安全载荷协议：除提供AH认证头协议的所有功能之外，还可对IP报文净荷进行加密。ESP协议允许对IP报对IP头的内容进行保护

    - 传输模式：在IP报文头和上层协议报文头之间插入一个IPSec报文头（AH或ESP）。在这种模式下，原IP报文头不变，只是IP协议字段被改为AH或ESP，并重新计算IP报文头校验和。传输模式适用于两台主机，或者是一台主机和一个安全网关之间的通讯

    - DPD超时时间：当接收端触发DPD查询，主动向对端发送请求报文进行检测，超过超时时间仍没有收到对端的DPD回应报文时，将删除此IPsec SA。合法值：10-3600，单位：秒
    - DPD重试间隔：用于IPSec邻居状态的检测时间间隔。启动DPD功能后，当接收端在触发DPD的时间间隔内收不到对端的IPSec加密报文时，能够触发DPD查询，主动向对端发送请求报文，对IKE对等体是否存在进行检测。合法值：1-60，单位：秒

<p align="center"><strong>图 4-96 IPsec高级选项</strong></p>

- IPsec高级选项（第2阶段）
  - 完美前向加密（PFS）：PFS是一种安全特性，指一个密钥被破解，并不影响其他密钥的安全性，因为这些密钥间没有派生关系。IPSec第二阶段的密钥是从第一阶段的密钥导出的，当第一阶段IKE密钥被窃取后，攻击者将可能收集到足够的信息来导出第二阶段IPSec SA的密钥，PFS通过执行一次额外的DH交换，保证第二阶段密钥的安全。
  - IPsec SA生命周期：设置IPSec SA的生存周期。IPSec协商建立SA时，采用本端设置的生存周期和对端的生存周期中较小的一个
  - IPsec SA空闲超时时间：当IPsec建立成功后在配置的空闲时间内没有数据传输，则IPSec SA就会失效。IPSec SA快要失效前，IPSec协商建立新的SA，这样在旧的SA失效前新的SA就已经准备好。

- Tunnel高级选项
  - 隧道启动方式：设置IPsec启用方式
    - 自动连接：应用IPSec策略后，自动完成IKE的协商建立IPSec隧道。常用于客户端模式
    - 仅响应连接：只会被动接收IPSEC请求，不会主动发起连接。常用于server模式
    - 按需连接：接口有IPsec定义的报文通过时，才完成IKE的协商，建立IPSec隧道
  - 重启接口前连接失败次数：当隧道连接失败次数达到设定的值以后，重启建立隧道的物理接口；默认为0，即失败后也不重启接口，合法值：1-12次
  - 重启设备前连接失败次数：当隧道连接失败次数达到设定的值以后，重启当前设备；默认为0，即失败后也不重启设备，合法值：1-32次
  - 本地/远端发送证书规则：可选择请求时发送证书、总是发送证书、不发送证书
    - 总是发送证书：有些ipsec服务不会发送"请求证书"请求，但是它的本地又没有保存对端的证书，那么对端就必须要配置成"总是发送证书"才能建立IPSEC
    - 请求时发送证书：只有当对端发送请求时，才发送本地证书
    - 不发送证书：不管对端是否发送请求，本地都将把自己的证书发送到对端
  - ICMP探测
    - ICMP探测服务器：IPsec探测的对端主机地址
    - ICMP探测本地地址：IPsec保护流量的源地址

    - ICMP探测最大重试次数：设置ICMP探测失败时的最大重试次数（达到最大次数后会重新启动IPsec服务）

##### 4.6.4.2 GRE

通用路由封装（GRE）定义了在任意一种网络层协议上封装任意一个其它网络层协议的协议。GRE可以作为VPN的三层隧道协议，为VPN数据提供透明传输通道。简单来说，GRE是一种隧道技术，提供了一条通路使封装的数据报文能够在这个通路上传输，在隧道的两端分别对数据报进行封装及解封装。

<p align="center"><img src="./img/WnWQlWcKwJH_STyE/1720522741408-cd5da5c6-3d60-4e77-803a-2126a0068eaf-486625.png" alt="GRE配置"></p>

<p align="center"><strong>图 4-101 GRE配置</strong></p>

GRE参数说明如下：

- 启用：启用或禁用GRE功能
- 接口标识：设置GRE隧道名称，范围1-100
- 网络类型：选择GRE网络类型
- 本地虚拟IP：设置本地虚拟IP地址
- 对端虚拟IP：设置对端虚拟IP地址，若为子网类型，此项为本地子网掩码
- 源地址类型：选择源地址类型，并设置对应类型的IP地址或接口名称
- 本地IP地址：配置GRE隧道的源地址
- 对端地址：配置GRE隧道的目的地址
- 密钥：设置隧道的认证密钥，两端配置需一致
- MTU：设置GRE隧道的最大传输单元，以字节为单位
- 启用NHRP：下一跳解析协议，用于连接到非广播多路访问（NBMA）式子网络的源站决定到达目标站间的"NBMA下一跳"的互联网络层地址和NBMA子网地址
  - NHS地址：对端NHS服务器地址
  - 认证密钥：NHRP的认证密钥
  - 维持时间：合法值1-65535
  - 禁止NHRP Purge消息：启用/禁止

##### 4.6.4.3 OpenVPN

在OpenVPN中，如果用户访问一个远程的虚拟地址（属于虚拟网卡配用的地址系列，区别于真实地址），则操作系统会通过路由机制将数据包（TUN模式）或数据帧（TAP模式）发送到虚拟网卡上，服务程序接收该数据并进行相应的处理后，通过SOCKET从外网上发送出去，远程服务程序通过SOCKET从外网上接收数据，并进行相应的处理后，发送给虚拟网卡，则应用软件可以接收到，完成了一个单向传输的过程，反之亦然。

Scep（Simple Certificate Enrollment Protocol）简单证书注册协议，是cisco和Verisign一起制定的设备证书管理的通信协议。它利用现有的PKCS#7和PKCS#10协议技术结合，享有广泛的支持客户端和CA的实现。

**证书管理：**

<p align="center"><img src="./img/WnWQlWcKwJH_STyE/1720522742429-0f235852-3ea7-4729-a546-b724c133306e-207018.png" alt="证书管理"></p>

- 请求状态：当前设备申请证书的状态，分为initialize、Enrolling、Re-Enrolling、Complete四种状态
- 证书保护密钥：申请证书时设置的证书保护密钥，起到数字证书加密作用，当导入和导出证书时配置的保护密钥和证书申请时的密钥一致才能使用

- FQDN：设置证书的FQDN（Fully Qualified Domain Name，合格域名）FQDN是实体在网络中的唯一标识，由一个主机名和域名组成，可被解析为IP地址。例如，www是一个主机名，whatever.com是一个域名，则www.whatever.com就是一个FQDN
- 单位名1：配置证书所属的单位名称1
- 单位名2：配置证书所属的单位名称2
- 域名：配置证书的合格域名
- 序列号：配置申请证书的序列号
- 认证密码：设置证书申请时的挑战密码，挑战码是指撤销证书时使用的密码（可选）
- 认证密码确认：对认证密码校验，与上面配置一样
- 主机地址：设置证书使用的IP地址
- RSA密钥长度：合法值：128-2048，单位：位
- 查询时间间隔：设备向证书服务器查询当前证书状态的时间间隔，合法值：30-3600，单位：秒
- 查询超时时间：设置设备查询证书状态的最大时间，当达到超时时间后，认为本次证书申请失败。合法值：30-86400，单位：秒
- 证书吊销：启用或禁用证书吊销
  - CRL URL：CRL即证书吊销列表，设置CRL发布点的URL
  - OCSP URL：配置OCSP服务器的URL，一般与证书服务器的URL一致
- 证书：导入相关证书

##### 4.7.1.1 串口设置

根据与IG相连的终端设备的串口参数设置IG串口的参数，实现IG与终端设备的正常通信。

<p align="center"><img src="./img/WnWQlWcKwJH_STyE/1720522743143-81c347c6-07be-4c85-8124-adcca6fa542a-628240.png" alt="串口设置"></p>

<p align="center"><strong>图 4-109 串口设置</strong></p>

串口设置参数说明如下：

- 串口类型：IG不同系列的串口类型及数量不相同，根据实际情况进行选择
- 波特率：这是一个衡量符号传输速率的参数。它表示每秒钟传送的符号的个数
- 数据位：设置通信中实际数据位的参数
- 校验位：在串口通信中的检错方式，一般是奇偶校验或者无校验两种方式
- 停止位：用于表示单个包的最后一位。典型的值为1，1.5和2位
- 软件流控：串口通讯的流控提供了由于某种原因不能进行通讯时阻塞通讯的一种机制。流控可以使数据接收设备在不能接收数据时通知数据发送设备，使其停止发送
- 串口描述：自定义描述

**注意：**

- IG的串口参数与相连的终端设备的串口参数必须设置一致
- DTU功能和GNSS串口转发功能不能同时开启

##### 4.7.1.2 DTU1

<p align="center"><img src="./img/WnWQlWcKwJH_STyE/1720522743402-f7be2e2c-9bf7-41eb-8df6-0ffb529e456e-755197.png" alt="DTU1配置"></p>

<p align="center"><strong>图 4-110 DTU1配置</strong></p>

DTU 1参数说明如下：

- 启用：点选启用
- DTU协议：可选择透明传输、TCP服务器、RFC2217模式、IEC101转104、Modbus网桥、DC协议
  - 透明传输和TCP服务器：若选用透明传输设备作客户端，则选用TCP服务器设备作服务器端
  - RFC2217模式：选用此模式后不用再设置串口配置
  - IEC101转104：适用于电力行业，功能和TCP相似
- 传输协议：可选TCP协议和UDP协议
- 连接类型：可选长连接和短连接
  - 长连接：TCP客户端和TCP服务器建立连接后一直保持TCP连接
  - 短连接：TCP客户端和TCP服务器建立连接后，当空闲时间内没有数据传输，自动断开与服务器TCP连接
- 心跳间隔：TCP客户端和TCP服务器建立连接后，定期发送TCP心跳报文的时间间隔，用于检测客户端和服务器之间的连通性
- 心跳重试次数：当TCP心跳超时后，设备重新发送TCP心跳，达到设置的心跳重试次数后，重新建立TCP连接。
- 串口缓存帧个数：串口收发数据时设置的串口缓存大小，默认4K
- 串口分帧长度：串口发送数据时，设置的一帧数据的大小。当达到设定值后开始发送。合法值：1-1024，单位：字节
- 串口分帧间隔：当串口发送数据时候，当发送间隔大于设置的分帧间隔后，设备会自动分帧发送，合法值：10-65535，单位：毫秒
- 最小重连间隔：用户自定义最小重连间隔时间。设备启动连接时如果没连接成功会按照此最小重连间隔时间重连。直到达到用户自定义的最大重连间隔。合法值：15-60，单位：秒
- 最大重连间隔：用户自定义最大重连间隔时间。设备启动连接且连接时间达到最大重连间隔时间后，每隔此间隔时间（即用户自定义的最大重连间隔时间）连接一次。合法值：60-3600，单位：秒
- 多中心策略：可选择并发、轮询
  - 并发：同时去连接目的IP地址列表中的中心
  - 轮询：先连接列表前边的中心，若连接上就不再连接后边的；若没有连接上则按照自前往后的顺序去连接，直到连接上一个中心为止
- 源接口：一般用户不用选择
- 本地IP地址：源接口选择IP时对应的设备接口IP地址。一般用户不用配置，可为空
- DTU标识：自定义，成功连接服务器后自动向服务器发送的DTU标识。也可以不配置，保持空状态
- 调试日志：点选开启，开启后将会打印DTU相关日志
- 开启ReportID：点选开启
- 心跳间隔时间：启用ReportID才需设置此项，合法值：1-65535，单位：秒
- 心跳包内容：启用ReportID才需设置此项
- 目的IP地址
  - 服务器地址：自定义设备要连接的服务器IP地址
  - 服务器端口：自定义设备要连接的服务器端口

说明：目的IP地址最多可以设置10个。

## 第5章 典型应用

### 案例1：工业远程监控

**场景描述**：在工厂环境中，IG502作为边缘网关，通过蜂窝网络或以太网将现场PLC、传感器等设备的数据采集并上传至云平台，实现设备远程监控和运维管理。

**网络拓扑**：

```
[现场设备: PLC/传感器/仪表]
        |
        | RS232/RS485
        |
    [IG502] ----蜂窝网络/以太网----> [InHand Device Live云平台]
        |
        | Web管理
        |
    [运维人员PC]
```

**设备角色**：本设备作为边缘网关，负责采集现场设备数据、协议转换、数据预处理及上云传输。

**配置步骤**：

1. 将现场设备通过RS232/RS485串口连接到IG502的串口端子。
2. 将蜂窝天线安装到IG502的天线接口，插入SIM卡。
3. 配置IG502的蜂窝网络参数（详见[场景1：蜂窝联网](#场景1蜂窝联网)）。
4. 配置DTU功能，设置串口参数与现场设备一致，选择透明传输或Modbus网桥模式，配置目的服务器地址为云平台地址。
5. 在【系统管理】→【设备云平台】中配置连接到InHand Device Live Manager（详见[场景6：设备云平台上云](#场景6设备云平台上云)）。
6. 登录InHand Device Live云平台，在设备列表中查看IG502在线状态，并通过云平台远程访问设备Web界面。

**参考章节**：

- [蜂窝网络配置](#场景1蜂窝联网)
- [DTU配置](#471-dtu)
- [设备云平台上云](#场景6设备云平台上云)

## 附录 安全注意事项

1. 设备应在规定的温度和湿度范围内使用（工作温度-25~70℃，环境湿度5~95%无凝霜）。
2. 请勿在易燃易爆环境中使用本设备。
3. 电源连接前请确认电压符合设备规格要求（12-48VDC）。
4. SIM卡和Micro SD卡不支持热插拔，操作前需断开设备电源。
5. 在断开USB大容量存储设备之前，需输入sync同步命令，以防止数据丢失。
6. 断开存储设备时，需从/mnt/usb/sda_<num>目录下退出，否则自动卸载过程将会失败。

> **警告**：非专业人员请勿打开设备外壳，存在触电风险。

## FAQ 常见问题解答

### 问题1：IG502与其他型号（如IG504、IG532）有什么区别？

IG502、IG504、IG532均为映翰通IG系列边缘计算网关，主要区别在于硬件接口配置和性能。IG502为高性价比型号，尺寸小巧；IG504/IG532在接口数量和性能上有所扩展。具体规格差异请参见各型号产品规格书。

### 问题2：如何确认IG502的固件版本？

登录Web管理界面后，在【概览】页面的"系统信息"区域可查看固件版本号。也可在【高级功能】→【管理】→【系统】页面查看详细的版本信息。

### 问题3：蜂窝网络无法拨号成功怎么办？

1. 物理环境：检查SIM卡是否插到正确的卡槽位，蜂窝天线是否全部安装。
2. APN设置：确保APN的配置信息与运营商提供的一致。
3. 检查信号状态：查看信号状态指示灯或Web界面的蜂窝信号强度。
4. 检查设备连通性：登录设备本地界面，通过系统自带的ICMP工具ping 8.8.8.8测试是否连通。
5. 判断SIM卡是否正常：取出SIM卡，安装在手机中查看是否可以正常联网。
6. 重启：尝试将设备断电，等待几秒钟，然后重新连接电源，重试网络连接。
7. 恢复出厂：将设备恢复出厂，然后重新尝试。

### 问题4：Web界面登录密码忘记怎么办？

IG502的初始登录密码为设备铭牌上的密码。如已修改密码且遗忘，需通过硬件方式恢复出厂设置（长按RESET键），恢复后密码将恢复为铭牌初始密码。恢复出厂后所有配置将被清除，需重新配置。

### 问题5：Python App开发需要哪些环境？

需先在【边缘计算】→【Python边缘计算】页面启用Python引擎并安装SDK。开发环境推荐使用映翰通提供的Python SDK，具体开发指南请参考[Python开发快速入门](http://sdk.ig.inhand.com.cn/zh_CN/latest/MobiusPi-Python-QuickStart-CN.html)。

### 问题6：如何备份和恢复设备配置？

在【系统管理】→【配置管理】页面，可导出startup-config或running-config到本地备份。需要恢复时，点击"导入startup-config"选择备份文件导入，导入后重启设备即可生效。建议定期备份配置以防止意外丢失。
