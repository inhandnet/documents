### 2.3 网络

### 2.3.1 网络接口

#### 2.3.1.1 蜂窝网

此页面展示了网关当前的蜂窝网状态及配置信息。您可以通过”启用蜂窝网“按钮对蜂窝网参数进行配置。

![1720522717399-f74fd77b-8eb5-4ab5-8952-05721d6530b5.png](./img/WnWQlWcKwJH_STyE/1720522717399-f74fd77b-8eb5-4ab5-8952-05721d6530b5-119419.png)

蜂窝网参数说明如下：

- 启用蜂窝网：启用/关闭蜂窝网功能
- 拨号参数集：拨号参数集，用于专用网卡拨号时配置APN，用户名，密码以及认证方式等信息。如果不是专网卡，通常不需要修改这里的配置。拨号参数集最多可以添加10条记录。
  - 网络类型：用户选择设备所使用的移动网络类型，可选择GSM、CDMA。
  - APN：APN（Access Point Name）用来标识 WCDMA/LTE 网络的业务种类，WCDMA/LTE 系统根据用户连接 WCDMA/LTE 网络的 APN 提供相应的服务。由运营商提供(CDMA2000 系列不设置此项)
  - 拨号号码：拨号使用的拨号串。拨号串由运营商提供，请向运营商获取。
    - 当 3G/LTE 数据卡支持 WCDMA 或 LTE 标准时，缺省拨号串为*99***1# 。
    - 当 3G 数据卡支持 CDMA2000 标准时，缺省拨号串为#777。
  - 认证方式
    - 自动：自动选择一种认证方式
    - PAP：密码认证协议，通过两次握手提供一种简单明文认证方式。
    - CHAP：挑战握手认证协议，通过三次握手确认摘要信息从而进行安全认证。
    - MS-CHAP：微软公司的 CHAP 标准。
    - MS-CHAPv2：MS-CHAP 升级版本，它要求双向验证。
  - 用户名：指定接入外部 PDN 网络用户的用户名。由运营商提供。缺省用户名为gprs。
  - 密码：指定接入外部 PDN 网络用户的密码。由运营商提供。缺省用户名为gprs。

![1720522717673-6826f00c-ef1c-4ed2-a116-43e662157cc6.png](./img/WnWQlWcKwJH_STyE/1720522717673-6826f00c-ef1c-4ed2-a116-43e662157cc6-366141.png)

- 启用双SIM卡： 启用双卡功能。为了提高网络的可靠性，支持双卡单拨。设备上需要插入2张SIM卡，默认是关闭的。
  - 选择主卡：支持SIM1、SIM2、随机、顺序四种方式
  - 最大拨号次数：当 SIM1 在设置的最大拨号次数内一直没拨号成功,设备将切换到 SIM2 拨号
  - 最小连接时间：当设备拨号连接成功时间小于设置的最小连接时间时，设备的拨号次数会累计。当大于设置的最小连接时。设备的拨号次数将清零。0 是禁用此功能
  - 备卡超时时间：若当前使用的为备卡，备卡拨号成功后，当达到设定的备卡超时时间后，设备会切换到主卡拨号
- 网络选择方式：可选择自动、2G、3G、4G、5G等网络。（（备注：不同设备支持的网络类型不同，具体可查看对应的产品规格书））用户根据所使用设备及 SIM 卡适用的情况，可选择特定的网络方式，或使用自动方式，设备可自行注册到适用当前网络状况的网络方式。
- 拨号参数集：拨号策略选择，对应于拨号参数集配置索引项，若不是专用网卡，选择默认即可。
- 启用漫游：勾选启用漫游功能，在漫游状态下可以正常拨号上网，当取消漫游选项，漫游的 SIM 卡不能拨号上网；使用本地卡时，勾选漫游和取消漫游功能都不影响 SIM 卡拨号上网。
- PIN code：PIN 码即 SIM 卡的个人识别密码。如果启用 PIN 码，当不设置 PIN 码或设置错误的 PIN 码，设备拨号失败；设置正常的 PIN 码，设备可以正常拨号上网。

![1720522717878-21bf29c4-1511-4e53-a7e1-fe621992edbf.png](./img/WnWQlWcKwJH_STyE/1720522717878-21bf29c4-1511-4e53-a7e1-fe621992edbf-064375.png)

- 静态IP：拨号时是否使用静态 IP，可以手动指定设备的 IP 地址。设备每次拨号都获得配置的静态 IP。
- 重拨间隔：设备每次掉线重新拨号等待的时间。

![1720522718089-02b5d806-ab94-4a59-ac56-1c45103aea77.png](./img/WnWQlWcKwJH_STyE/1720522718089-02b5d806-ab94-4a59-ac56-1c45103aea77-106580.png)

- ICMP探测：蜂窝网络有时会出现拨号成功并获取到 IP 地址，但却无法上网的情况，通常重新拨号能够解决此问题。ICMP 探测可用于检测网络是否正常，一旦网络异常，就会触发重新拨号以解决网络不通的问题。该功能默认关闭，建议用户在部署蜂窝网络时开启该功能
  - ICMP探测服务器：要探测的远端 IP 地址或域名（同时启用两个 ICMP 探测服务器，建议同时输入 IP 地址或同时输入域名）。设备支持两个 ICMP 探测服务器：主服务器和备份服务器。当配置两个服务器后，首先检测第一个服务器，只有当第一个服务器达到最大重试次数后，系统才会检测第二个服务器。当两个服务器都检测失败的情况下，设备会重新拨号并进行下一轮 ICMP 探测。
  - ICMP探测间隔时间：设备发送 ICMP 探测报文的时间间隔。
  - ICMP探测超时时间：在设置的 ICMP 探测超时时间内，没有收到 ICMP 响应包认为本次 ICMP 探测超时。
  - ICMP探测最大重试次数：设置 ICMP 探测失败时的最大重试次数（达到最大次数后会重新拨号）。
  - ICMP严格探测：关闭ICMP严格探测时，如果拨号接口有流量，则不发送ICMP探测以节省流量；开启ICMP严格探测的时候，无论拨号接口是否有流量，都会发送ICMP探测报文。

![1755064375485-b9ebf795-da0b-4c7f-ab19-5328228f6e26.png](./img/WnWQlWcKwJH_STyE/1755064375485-b9ebf795-da0b-4c7f-ab19-5328228f6e26-666340.png)

- 短信：短信功能用于设置蜂窝参数，查询蜂窝状态，配置io输出以及重启系统。
  - 启用短信：启用/禁用短信功能
  - 回复结果：开启后，当用户通过短信配置蜂窝，io或者重启系统时，设备会回复短信告知配置的结果是否成功。
  - 查询间隔：设备会定期查询收到的短信，这里设置定期查询的间隔
  - 手机白名单：只有在白名单中的手机号才能发送短信设置蜂窝参数等信息。

**短信配置详情如下：**

**1、短信设置蜂窝拨号参数**

短信格式：

:::color2
code=xxx cellular slot=sim1 apn=hello username=xxx password=xxx network=4g dialnumber=*99*1 auth=pap  

:::

:::color2
cellular slot=sim1 apn=hello username=xxx password=xxx network=4g dialnumber=*99*1 auth=pap  

:::

蜂窝拨号参数配置说明如下：

- code: 页面设置的短信密码，若设置了密码，必须以code，必选项。没有设置密码时按第二种格式下发指令。
- cellular：表示设置 cellular 服务相关的参数， cellular关键字的位置需要在短信内容的开始部分; cellular 后面的内容是cellular的配置项.
- slot：标志设置哪张卡的拨号参数，可设置的值为： sim1 , sim2 ；
- 必填参数 apn：配置 apn 信息；
- 非必填参数 username：配置用户名；
- 非必填参数 password：配置密码；
- 非必填参数 auth：认证方式；可设置的值为： none , auto , pap , chap ；严格区分大小写；
- 非必填参数 network：配置网络制式；可设置的值为： 2g , 3g , 4g , auto ；严格区分大小写；
- 非必填参数 dialnumber：拨号号码；非必填参数  

**注意事项：**

1. 非必填参数需要至少填写1项；除了 cellular 必须在短信内容的开始部分，其他参数的顺序没有要 求；每个参数之间用空格分隔；
2. 参数名称和部分参数值严格区分大小写；
3. 如果短信内容中出现不识别的配置项，例如icmp_interval=10，则配置会失败；
4. 短信内容的长度不能超过140个字符；
5. 由于默认参数集不允许修改，当sim1/sim2选用默认参数集的时候，通过短信配置拨号参数会失败  

**2、短信开启双卡**

短信格式：

:::color2
code=xxx cellular enable_dualsim=0

:::

:::color2
cellular enable_dualsim=1

:::

蜂窝拨号参数配置说明如下：

- enable_dualsim：是否开启双卡，0表示关闭，1表示开启

**注意事项：**

1. 参数仅支持0或者1

**3、短信执行切卡**

短信格式：

:::color2
code=xxx cellular switch_to=sim1

:::

:::color2
cellular switch_to=sim2

:::

蜂窝拨号参数配置说明如下：

- switch_to：切换至目标卡，sim1表示切换至sim1卡槽，sim2表示切换至sim2卡槽

**注意事项：**

1. 参数仅支持sim1或者sim2

**4、短信重启系统**

短信格式：

:::color2
code=xxx system command=reboot  

:::

:::color2
 system command=reboot  

:::

重启系统参数说明：

1. system：表示执行系统相关的设置； system 关键字的位置需要在短信内容的开始部分；
2. command：表示需要执行的命令，可取值： reboot ；必填参数  

**5、短信查询拨号信息**

短信格式：

:::color2
code=xxx status object=cellular  

:::

:::color2
 status object=cellular  

:::

查询拨号信息参数说明：

1. status：表示执行状态查询； status 关键字的位置需要在短信内容的开始部分；
2. object：表示需要查询哪个服务的状态，可取值： cellular ；必填参数  

 发送查询短信后，会收到如下内容的短信：  

:::color2
 cellular_status signal=none network=4g hostname=EdgeGateway uptime=150 ip=10.48.104.247

:::

回复内容参数说明如下：

1. cellular_status: 蜂窝的状态
2. signal：蜂窝的信号质量；可取值： none , poor , moderate , good , great
3. network: 网络制式；可取值： 2g , 3g , 4g , unknown
4. hostname：设备的主机名
5. ip：拨号接口的ip地址
6. uptime：设备的启动时长，单位s  

**6、短信设备数字量输出DO**

短信格式：

:::color2
code=xxx io do0=1 do1=1 do2=0 do3=0  

:::

:::color2
 io do0=1 do1=1 do2=0 do3=0  

:::

DO设置参数说明：

1. io：表示设置 io 服务相关的参数， io 关键字的位置需要在短信内容的开始部分
2. do0~do3：配置do0~do3的输出，可以配置多个do，配置项之间没有顺序要求，比如可以这样配置：io do0=1 do1=1 do3=0 do2=0；也可以只配置一个do，比如io do0=1；至少需要配置1个 do；do的取值只能为0或1  

**7、 短信接收DI的变化**

使用该功能，需要在io页面配置接收短信的手机号，同时在拨号页面将该手机号加入白名单中.

 di的值发生变化后用户的手机号会收到如下内容的短信：

 io_status sn=GL5022221013941 di0=0 timestamp=2024-05-15T10:03:12+0000

:::

- 高级设置
  - 初始化命令：可以配置一些 AT 指令查询模块状态。
  - 信号查询间隔：设备拨号成功后，将以设置的查询间隔时间定时查询信号状态。如设置查询间隔时间为 60s。设备拨号成后，拔下设备天线，等到 60s 后设备信号应该降低，在 60s 内设备信号不发生变化。0 表示禁用。
  - 拨号超时时间：在设定的超时时间内，设备未成功拨号，认为拨号超时，设备重新检测模块并重新拨号。
  - MRU：最大接收单元，以字节为单位。
  - MTU：最大传输单元，以字节为单位。
  - 启用缺省的 Asyncmap：启用/禁用缺省的Asyncmap
  - 使用分配的DNS服务器：启用后使用拨号网络连接中分配的DNS服务器
  - 连接检测间隔：每间隔指定时间检测拨号连接是否正常

  - 动作：允许/拒绝报文通过
  - 源IP地址：ACL 规则匹配报文的源地址，为空时表示 any，即所有网段。
  - 源地址反掩码：ACL 规则匹配报文的源地址反掩码，例如：源地址网段为192.168.2.0，其掩码为255.255.255.0，则其反掩码为0.0.0.255。
  - 记录日志：启用后系统会记录关于访问控制方面的日志。
  - 说明：用于对该条ACL规则进行描述，方便一眼看出该ACL的作用。

  - 动作：放行/阻止报文通过
  - 协议：访问控制协议

  - 说明：用于对该条ACL规则进行描述，方便一眼看出该ACL的作用。

网络地址转换规则参数说明如下：

  - DNAT：目的地址转换，将 IP 数据包的目的地址转换成另外一个地址。一般用于从网关外部发往内部的数据。

### 2.5 系统管理

### 2.5.1 系统时间

为了让 IG 与其他设备更好的协同，需精准配置系统时间。配置方式及步骤如下：

- **方法一：使用时区配置系统时间**
    1. 选择“系统管理 > 系统时间”，进入“系统时间”界面。
    2. 在“系统时间”的“时区”中选择网关的时区
    3. 选择后点击“应用”即可

![1720522727710-fad6f290-8c15-4d85-8947-ed2220848590.png](./img/WnWQlWcKwJH_STyE/1720522727710-fad6f290-8c15-4d85-8947-ed2220848590-056860.png)

- **方法二：使用PC的本地时间配置系统时间**
    1. 选择“系统管理 > 系统时间”，进入“系统时间”界面。
    2. 网关会自动获取PC的时间作为本地时间
    3. 点击设备时间中的“同步”，设备时间将与本地时间同步

![1720522727916-dc8699d6-f514-4a55-b187-5db12d32efcd.png](./img/WnWQlWcKwJH_STyE/1720522727916-dc8699d6-f514-4a55-b187-5db12d32efcd-051275.png)

- **方法三：使用手动设置时间的方法配置系统时间**
    1. 选择“系统管理 > 系统时间”，进入“系统时间”界面。
    2. 在“设置时间”中直接手动设置设备时间
    3. 设置完成后点击“应用“即可

![1720522728140-40261843-1814-4b07-b036-eab1f54cf671.png](./img/WnWQlWcKwJH_STyE/1720522728140-40261843-1814-4b07-b036-eab1f54cf671-959640.png)

- **方法四：使用SNTP客户端配置系统时间**
    1. 选择“系统管理 > 系统时间”，进入“系统时间”界面。
    2. 勾选“启用SNTP客户端”
    3. 依次配置各项参数
    4. 点击“提交”应用配置。

![1720522728385-ae73aa5c-d8f1-4918-888a-b66c3bc89c3e.png](./img/WnWQlWcKwJH_STyE/1720522728385-ae73aa5c-d8f1-4918-888a-b66c3bc89c3e-699745.png)

SNTP客户端参数说明如下：

- 启用SNTP客户端：开启/关闭 SNTP 客户端。
- 更新时间间隔：启用SNTP客户端后按照更新时间间隔向服务器同步设备时间。
- SNTP服务器列表
  - 服务器地址：SNTP 服务器地址（域名/IP），最多可填写 10 个服务器。设置多个 SNTP 服务器时，系统将轮询所有 SNTP 服务器，直到找到可用的。
  - 端口：SNTP 服务器的 SNTP 服务端口

**IG也可作为NTP服务器为其他设备同步时间，具体步骤如下：**

1. 选择“系统管理 > 系统时间”，进入“系统时间”界面。
2. 勾选“启用NTP服务器”
3. 依次配置各项参数
4. 点击“提交”应用配置。

![1720522728582-a21f1315-cbd0-4f7b-9a4f-bd309aef040c.png](./img/WnWQlWcKwJH_STyE/1720522728582-a21f1315-cbd0-4f7b-9a4f-bd309aef040c-132901.png)

NTP服务器参数说明如下：

- 启用NTP服务器：开启/关闭 NTP 服务器
- NTP服务器层级：NTP 采用分层同步方式，一般第 n+1 级与第 n 级时钟源进行同步。NTP 最多支持 16 层同步，即 0－15 层。多于 16 层将无法同步。
- 源接口：网关发出 NTP 报文的接口。源接口与源地址不能同时使用。
- 源地址：网关发出 SNTP 报文携带的源地址。 源接口与源地址不能同时使用。
- NTP服务器列表
  - 主NTP服务器：设置多个 NTP 服务器，勾选主 NTP 服务器时，表示我们的设备以该 NTP 服务器进行时间同步。勾选多个则轮询所有已勾选的 NTP 服务器，直到找到可用的。
  - 服务器地址：NTP 服务器地址（域名/IP），最多可填写 10 个服务器。

### 2.5.2 系统日志

你可以选择“系统管理 > 系统日志”，进入“系统日志”页面。该页面包含了网络和IG的大量信息，包括运行状态、配置变化等。

![1720522728979-02b77cdc-2b14-4d13-8c2e-3afbfd8acf02.png](./img/WnWQlWcKwJH_STyE/1720522728979-02b77cdc-2b14-4d13-8c2e-3afbfd8acf02-999602.png)

在“配置”页面，可以设置网关对接远程日志服务器，设置完成后网关将会把所有的系统日志上传到远程日志服务器，这需要主机上的远程日志软件（如:Kiwi Syslog Daemon）的配合。

![1720522729289-07e15a43-3eda-428a-9e43-5e09f537a331.png](./img/WnWQlWcKwJH_STyE/1720522729289-07e15a43-3eda-428a-9e43-5e09f537a331-729171.png)

- 发送到远程日志服务器：开启/关闭设备对接远程日志服务器
- 服务器地址：远程日志服务器的地址
- 端口号：远程日志服务器的端口号

当日志在本地进行存储时，可对日志文件的大小、日志输出的等级进行设置：

![1720522729522-7463aa53-e2f0-4541-8c22-7719a6f365fc.png](./img/WnWQlWcKwJH_STyE/1720522729522-7463aa53-e2f0-4541-8c22-7719a6f365fc-086328.png)

- 历史日志文件大小： 主机被分配用以存储日志的空间大小。
- 历史日志文件严重等级：将设备中的日志按照“紧急>警报>严重>错误>警告>通知>信息>调试”进行排序。设置日志等级为“通知”及以上等级时，则日志文件将输出“通知“以及比”通知“优先级更高的日志。

### 2.5.3 配置管理

选择“系统管理 > 配置管理”，进入“配置管理”界面。你可以在此页面备份配置参数；导入相应的参数配置；使IG恢复出厂设置。

![1720594939169-52ee2c2a-13bd-4bbe-880d-57c153beb151.png](./img/WnWQlWcKwJH_STyE/1720594939169-52ee2c2a-13bd-4bbe-880d-57c153beb151-792836.png)

- 配置管理
  - 自动保存：勾选后，每次修改配置都会自动将running-config中的配置同步到startup-config中，以确保设备断电启机后配置不丢失。
  - 加密：启用后IG在 WEB 上配置的所有带密码的参数都会以加密的方式在配置中显示。提高密码安全性。
- 配置文件操作
  - 导入 startup-config：将配置文件导入到 startup-config 中，重启后会加载所导入的配置。（注意：在此过程中，请务必确保导入配置的合法性与有序性。导入配置时，IG 会过滤格式不合法的命令，并将其余正确的配置重新存储为 startup-config，待系统重启后按顺序执行合法配置。若导入的配置内容未按有效顺序排列，将致使系统无法进入期望的配置状态。）
  - 导出 startup-config：备份 startup-config 到主机，startup-config 为网关开机启动时候的配置。
  - 导出 running-config：备份 running-config 到主机，running-config 为网关当前正在运行的配置。
  - 恢复出厂设置：将IG恢复到出厂配置，网关所有的配置被恢复到默认参数。恢复出厂后需要重新启动IG才能生效。

### 2.5.6 管理工具

为便于管理和配置IG,你可以在“管理工具”页面配置IG的管理和访问方式。配置步骤如下：

- **配置HTTPS**

![1720522731654-b6a71bd4-0133-40db-b7da-111a3ace1051.png](./img/WnWQlWcKwJH_STyE/1720522731654-b6a71bd4-0133-40db-b7da-111a3ace1051-182016.png)

HTTPS参数说明如下：

1. 监听地址：可选Any、127.0.0.1等IP地址。默认选择Any即可，即设备监听任意地址的HTTPS请求
2. 端口：HTTPS访问端口号,默认采用443端口
3. web登录超时：登录web页面的超时时间，默认是300s，当web空闲时间超过300s后会退出web登录；也可通过手动更改web登录超时时间，合法值100-3600，单位：秒
4. 远程控制：开启后可以通过HTTPS远程访问IG
5. 源网络：用于访问设备WEB页面的合法源地址；如果不设置则代表任意地址均可通过HTTPS连接到IG。

- **配置TELNET**

![1720522731849-4d24c8ec-fadc-4148-a3c1-cd6fe1f72523.png](./img/WnWQlWcKwJH_STyE/1720522731849-4d24c8ec-fadc-4148-a3c1-cd6fe1f72523-018340.png)

TELNET参数说明如下：

1. 监听地址：可选Any、127.0.0.1等IP地址。默认选择Any即可，即设备监听任意地址的TELNET请求
2. 端口：TELNET访问端口号,默认采用23端口
3. 远程控制：开启后可以通过TELNET远程访问IG。
4. 源网络：用于访问设备的合法源地址；如果不设置则代表任意地址均可通过TELNET连接到IG。

- **配置SSH**

![1720522732078-382362d3-c243-437f-8ce2-0ee4079d4855.png](./img/WnWQlWcKwJH_STyE/1720522732078-382362d3-c243-437f-8ce2-0ee4079d4855-856384.png)

SSH参数说明如下：

1. 监听地址：可选Any、127.0.0.1等IP地址。默认选择Any即可，即设备监听任意地址的SSH请求
2. 端口：SSH访问端口号,默认采用22端口
3. 超时时间：SSH超时时间，合法值0-120；超时时间内SSH未连接成功则视为本次连接失败
4. 密钥模式：SSH 连接通常使用非对称加密算法来实现安全通信，涉及到公钥和私钥。常见的密钥模式是RSA 模式。
5. 密钥长度：RSA 算法的密钥长度可以根据需求进行选择。一般来说，密钥长度越长，安全性就越高，但生成和处理密钥所需的计算资源也会增加。可选512/1024/2048/4096，默认长度为1024
6. 远程控制：开启后可以通过SSH远程访问IG。
7. 源网络：用于访问设备的合法源地址；如果不设置则代表任意地址均可通过SSH连接到IG。

- **配置开发者模式**

![1720522732306-883de2ba-84f2-42bb-bd88-472f2aa99cca.png](./img/WnWQlWcKwJH_STyE/1720522732306-883de2ba-84f2-42bb-bd88-472f2aa99cca-882678.png)

开发者模式参数说明如下：

1. 用户名：开发者模式登录的用户名，固定为pyuser
2. SSH端口： 开发者模式登录的端口号，默认为222
3. 启用固定密码：启用后可手动设置固定的密码登录设备，否则密码为设备生成的随机密码

#### 3.1.2.1 Radius

RADIUS 协议采用了客户机 / 服务器（ C/S ）工作模式。RADIUS协议实体由三个部分组成：用户端、RADIUS客户端（NAS）、RADIUS服务器。其认证流程如下图所示：

![1720522735071-08ef4552-d966-406f-82fd-284cbf1d4cdf.png](./img/WnWQlWcKwJH_STyE/1720522735071-08ef4552-d966-406f-82fd-284cbf1d4cdf-127996.png)

认证步骤如下：

1、当用户接入网络时，用户发起连接请求，向RADIUS客户端发送用户名和密码。

2、RADIUS客户端向RADIUS服务器发送包含用户名和密码信息的认证请求报文。（NAS和RADIUS服务器采用预共享密钥的方式，用户密码使用该共享密钥通过MD5加密算法进行加密隐藏，增加了密码的安全性）

3、RADIUS服务器对用户身份的合法性进行检验：

（1）如果用户身份合法，RADIUS服务器向RADIUS客户端返回认证接受报文，允许用户进行下一步动作。由于RADIUS协议合并了认证和授权的过程，因此认证接受报文中也包含了用户的授权信息。

（2）如果用户身份不合法，RADIUS服务器向RADIUS客户端返回认证拒绝报文，拒绝用户访问接入网络。

4、RADIUS客户端通知用户认证是否成功。

#### 3.1.2.3 LDAP

轻量级目录存取协议 LDAP （Lightweight Directory Access Protocol）是基于 TCP/IP 的目录访问协议，主要用于存储不常变数据，如电子邮件地址、联系人列表等。LDAP 定义多种操作，通过绑定和查询操作可实现用户认证和授权功能。LDAP 基于 Client/Server 结构，目录信息存于服务器上，且不支持计费。LADP认证流程如下图所示：

![1720522736373-c9d7844b-d435-4d92-87dd-f41e2c378551.png](./img/WnWQlWcKwJH_STyE/1720522736373-c9d7844b-d435-4d92-87dd-f41e2c378551-136276.png)

LADP认证说明如下：

1、当用户需要访问LDAP服务器时，用户输入用户名和密码，向设备发起认证请求

2、设备获取到用户的用户名和密码，以管理员DN和管理员密码为参数向LDAP服务器发送管理员绑定请求报文以获得查询权限。

3、LDAP服务器收到管理员绑定请求报文后，验证管理员DN和管理员密码是否正确。如果管理员DN和管理员密码正确，则绑定成功，LDAP服务器向设备发送管理员绑定响应报文。

4、设备收到绑定响应报文后，以用户输入的用户名为参数构造过滤条件，向LDAP服务器发送用户DN查询请求报文。例如：构造过滤条件为CN=User2。

5、LDAP服务器收到用户DN查询请求报文后，根据报文中的查询起点、查询范围、以及过滤条件，对用户DN进行查找。如果查询成功，则向设备发送查询成功的响应报文。查询得到的用户DN可以是一个或多个。

6、设备根据查询得到的用户DN和用户输入的密码为参数，向LDAP服务器发送用户绑定请求报文。

7、LDAP服务器收到用户绑定请求报文后，检查用户输入的密码是否正确。

（1）如果用户输入的密码正确，则向设备发送绑定成功的绑定响应报文。

（2）如果用户输入的密码不正确，则向设备发送绑定失败的响应报文。设备以查询到的下一个用户DN为参数，继续向LDAP服务器发送绑定请求，直至有一个DN绑定成功。如果所有用户DN都绑定失败，则设备通知用户认证失败。

8、认证成功后，设备通知用户登录成功。

**LADP认证配置如下图所示：**

![1720522736676-9dfac6cf-8511-4ba8-98b0-1085a6630a07.png](./img/WnWQlWcKwJH_STyE/1720522736676-9dfac6cf-8511-4ba8-98b0-1085a6630a07-985533.png)

LDAP参数说明如下：

- 名称：用户自定义服务器列表名称
- LDAP服务器：服务器地址（域名/IP），最多支持 10 条
- 端口：与服务器端口保持一致

### 3.3 链路备份

为了保持网络的稳定性，在设备组成的网络环境中，通常都使用一些备份连接，以提高网络的健壮性、稳定性，这里的备份连接也称为备份链路或者冗余链路。

### 3.3.1 SLA

InHand SLA 基本原理：1.对象跟踪：对指定对象的可达性进行跟踪。2.SLA probe：对象跟踪功能可使用 InHand SLA 向对象发出不同类型的探测。3.使用静态路由和跟踪选项。SLA配置步骤如下：

1. 定义一个或多个 SLA 操作（探测）。
2. 定义一个或多个跟踪(Track)对象，以跟踪 SLA 操作的状态 。
3. 定义与跟踪对象相关联的措施。

![1720522737775-6317cccf-d293-48a3-84c2-3c74859346a6.png](./img/WnWQlWcKwJH_STyE/1720522737775-6317cccf-d293-48a3-84c2-3c74859346a6-720614.png)

SLA参数说明如下：

- 标识：SLA 的索引或 ID，用户自定义也可自动生成最多可添加 10 条 SLA
- 类型：探测的类型，默认为 icmp-echo，通过简单发送一个 ICMP-ECHO（Type8）数据包到目标主机，如果 ICMP-ECHO-Reply（ICMPtype0）数据包接收到，说明主机是存活状态
- 目的地址：被探测的 IP 地址
- 数据大小：用户自定义探测数据包的大小
- 探测间隔（秒）：设备按照设置的探测间隔定时发送 ICMP 探测。
- 超时（毫秒）：设备发送 ICMP 探测后在配置的超时时间内没有收到返回包认为探测失败，进行下一次探测。
- 次数：即探测失败多少次为链路故障，当达到配置的探测次数后，认为 SLA 探测失败，这时 SLA状态为 DOWN 状态。
- 生命周期：默认为 forever（即配置后永远生效），用户不可更改
- 开始时间：默认为now（配置后立即生效）

### 3.3.2 Track模块

Track 的用途是实现联动功能。联动功能由应用模块、Track 模块和监测模块三部分组成。Track 模块位于应用模块和监测模块之间，主要功能是屏蔽不同监测模块的差异，为应用模块提供统一的接口。

![1720522738008-0cf2ad83-d85d-4806-8cd0-80eaf24bc670.png](./img/WnWQlWcKwJH_STyE/1720522738008-0cf2ad83-d85d-4806-8cd0-80eaf24bc670-533820.png)

Track 模块参数说明如下：

**Track表：**

- 标识：Track 的索引或 ID，用户自定义也可自动生成，最多可创建 10 条
- 类型：可选择sla 或 interface
- SLA标识：定义过的 SLA 的索引或 ID。类型选择为 interface 时，此项不可用
- 接口：类型选择为 sla 时，此项不可用
- 异常状态延时：当接口或 sla 状态为 DOWN 时，需要多久 track 才显示异常。0 表示立即显示，单位：秒
- 正常状态延时：当故障恢复时，可根据设置的时间(0 代表立即切换)，延迟切换，而不是立即切换

**Track控制表：**

- 标识：Track 的索引或 ID，用户自定义也可自动生成，最多可创建 10 条
- 控制服务：默认为ipsec，即此项Track用于检测IPSec连接建立状态
- 控制方式：
  - positive-start/negative-stop：当与 IPSec 相关的探测或监测结果为“positive”（IPSEC隧道状态正常）时，将启动相应的操作或进程；而当结果为“negative”（IPSEC隧道状态异常）时，将停止相应的操作或进程。
  - positive-stop/negative-start：当与 IPSec 相关的探测或监测结果为“positive”（IPSEC隧道状态正常）时，将停止相应的操作或进程；而当结果为“negative”（IPSEC隧道状态异常）时，将开始相应的操作或进程。

- 备份接口：等待切换的接口

- Track标识：跟踪探测，定义过的 Track 的索引或 ID。

### 3.4 VPN

VPN是在公共网络（如互联网）基础上构建的“虚拟”私人专用通信网络。所谓“虚拟”，是指它并非像传统的专用网络那样拥有独立的物理链路和基础设施，而是通过网络技术在公共网络上建立起一个逻辑上隔离且专用的通信通道，实现类似于专用网络的功能和安全性。

通过隧道技术、加密技术等手段，使得用户在使用 VPN 时感觉像是在一个独立、安全的专用网络中进行通信，但实际上数据是在公共网络的基础上传输的，只是在逻辑上实现了专用性和私密性。

### 3.4.1 IPsec

IPsec 是 IETF 制定的一组开放的网络安全协议，在 IP 层对数据包进行认证、加密操作来保证通信数据的完整性、安全性。减少数据泄漏和被窃听的风险，保障了用户业务数据传输的安全。

#### 3.4.1.1 IPsec配置

![1720522738468-a984c3d9-fa04-4680-91c3-ccfda5b1902a.png](./img/WnWQlWcKwJH_STyE/1720522738468-a984c3d9-fa04-4680-91c3-ccfda5b1902a-694492.png)

IPsec 配置参数说明如下：

- IKEv1策略
  - 标识：自定义 IKEv1 策略标识
  - 加密算法：可选择3DES、DES、AES128、AES192、AES256
    - 3DES: 使用三个 64bit 的 DES 密钥对明文进行加密
    - DES：使用 64bit 的密钥对一个 64bit 的明文块进行加密
    - AES：使用 128bit、192bit 或 256bit 密钥长度的 AES 算法对明文进行加密
  - 哈希算法：可选择MD5、SHA1、SHA2-256、SHA2-384、SHA2-512
    - MD5：通过输入任意长度消息，产生 128bit 的消息摘要
    - SHA1：输入长度小于 128bit 消息，产生 160bit 消息摘要
    - SHA2-256：输出 256bit
    - SHA2-384：输出 384bit
    - SHA2-512：输出 512bit
  - DiffieHellman密钥交换：Diffie-Hellman 算法是一种公开密钥算法。通信双方在不传送密钥的情况下通过交换一些数据，计算出共享的密钥。加密的前提是交换加密数据的双方必须要有共享的密钥。IKE 的精髓在于它永远不在不安全的网络上直接传送密钥，而是通过一系列数据的交换，最终计算出双方共享的密钥。即使第三者（如黑客）截获了双方用于计算密钥的所有交换数据，也不足以计算出真正的密钥
  - 生命周期：设置 IKE SA 的存活时间，在设置的存活时间超时前会提前协商另一个 SA 来替换旧的 SA。
- IKEv2策略
  - 标识：自定义 IKEv2 策略标识
  - 加密算法：可选择3DES、DES、AES128、AES192、AES256
    - 3DES: 使用三个 64bit 的 DES 密钥对明文进行加密
    - DES：使用 64bit 的密钥对一个 64bit 的明文块进行加密
    - AES：使用 128bit、192bit 或 256bit 密钥长度的 AES 算法对明文进行加密
  - integrity：可选择MD5、SHA1、SHA2-256、SHA2-384、SHA2-512
    - MD5：通过输入任意长度消息，产生 128bit 的消息摘要
    - SHA1：输入长度小于 128bit 消息，产生 160bit 消息摘要
    - SHA2-256：输出 256bit
    - SHA2-384：输出 384bit
    - SHA2-512：输出 512bit
  - DiffieHellman密钥交换：Diffie-Hellman 算法是一种公开密钥算法。通信双方在不传送密钥的情况下通过交换一些数据，计算出共享的密钥。加密的前提是交换加密数据的双方必须要有共享的密钥。IKE 的精髓在于它永远不在不安全的网络上直接传送密钥，而是通过一系列数据的交换，最终计算出双方共享的密钥。即使第三者（如黑客）截获了双方用于计算密钥的所有交换数据，也不足以计算出真正的密钥
  - 生命周期：设置 IKE SA 的存活时间，在设置的存活时间超时前会提前协商另一个 SA 来替换旧的 SA。
- IPsec策略
  - 名称：设置 IPSec 策略的名称。配置 IPSec 策略成功后，该参数不可修改。
  - 封装：IPSec 协议中的 AH 协议定义了认证的应用方法，提供数据源认证和完整性保证；ESP 协议定义了加密和可选认证的应用方法，提供数据可靠性保证
    - AH：提供数据源认证、数据完整性校验和报文防重放功能。发送端对 IP 头的不变部分和 IP 净荷进行离散运算，生成一个摘要字段。接收端根据接收的 IP 报文，对报文重新计算摘要字段，通过摘要字段的比较，判别报文在网络传输期间是否被篡改
    - ESP：ESP 封装安全载荷协议：除提供 AH 认证头协议的所有功能之外，还可对 IP 报文净荷进行加密。ESP 协议允许对IP 报对 IP 头的内容进行保护
  - 加密算法：可选择3DES、DES、AES128、AES192、AES256
    - 3DES: 使用三个 64bit 的 DES 密钥对明文进行加密
    - DES：使用 64bit 的密钥对一个 64bit 的明文块进行加密
    - AES：使用 128bit、192bit 或 256bit 密钥长度的 AES 算法对明文进行加密
  - 认证方式：可选择MD5、SHA1、SHA2-256、SHA2-384、SHA2-512
    - MD5：通过输入任意长度消息，产生 128bit 的消息摘要
    - SHA1：输入长度小于 128bit 消息，产生 160bit 消息摘要
    - SHA2-256：输出 256bit
    - SHA2-384：输出 384bit
    - SHA2-512：输出 512bit
  - IPsec模式：IPSec 协议的封装模式
    - 隧道模式：在原始 IP 报文头外封装一个 IPSec 报文头（AH或 ESP），然后在最外层封装新的 IP 报文头，原 IP 报文被当作有效载荷的一部分受到 IPSec 的保护。隧道模式一般用在两个安全网关之间。在一个安全网关被加密的报文，只有到达另一个安全网关才能够被解密
    - 传输模式：在 IP 报文头和上层协议报文头之间插入一个 IPSec 报文头（AH 或 ESP）。在这种模式下，原 IP 报文头不变，只是 IP 协议字段被改为 AH 或 ESP，并重新计算 IP 报文头校验和。传输模式适用于两台主机，或者是一台主机和一个安全网关之间的通讯
- IPsec隧道配置

- 基本参数
  - 对端地址：设置对端 IKE 对等体的 IP 地址或域名（当 IG为 server时配置为 0.0.0.0）
    - 接口名称：设置应用IPSec策略的接口
    - IKE版本：设置 IKE 协议使用的版本号，支持 IKEv1 和 IKEv2
    - IKEv1策略：在 IKEv1 策略列表里定义过的策略标识
    - IKEv2策略：在 IKEv2 策略列表里定义过的策略标识
    - IPsec策略：在 IPsec 策略列表里定义过的策略标识
    - 认证方式：两种认证方式可选，共享密钥和数字证书
            + 共享密钥：用户输入共享密钥
            + 数字证书：用户需在证书管理页面导入相应的有效证书
    - 协商模式：设置 IKEv1 的协商模式
            + 主模式：主模式将密钥交换信息与身份认证信息相分离。这种分离保护了身份信息，从而提供了更高的安全性
            + 野蛮模式：野蛮模式缺少身份认证，但可以满足某些特定的网络环境需求。如果无法预先知道发起者的地址、或者发起者的地址总在变化，而双方都希望采用预共享密钥认证方法来创建IKE SA，就可以用野蛮模式
    - 本地子网地址：IPESC 定义的感兴趣流中的源网络

- IKE高级选项（第1阶段）
  - 本地标识：设置 IKE 协商中本端设备的身份类型
            + IP地址：填写对端建立 IPsec 接口的地址
            + FQDN：使用字符串作为本端身份
            + User FQDN：使用全域名形式作为本端身份
    - 对端标识：设置 IKE 协商中对端设备的身份类型
            + IP地址：使用接口的 IP 地址作为本端的身份，和对端进行 IKE协商，交互身份信息
            + FQDN：设置 IKE 协商中对端设备身份使用的名称，需要和对端设备上设置的保持一致
            + User FQDN：与对端配置的全域名一致
    - IKE连接检测（DPD）：设置是否启用对等体存活检测 DPD 功能
            + DPD超时时间：当接收端触发 DPD 查询，主动向对端发送请求报文进行检测，超过超时时间仍没有收到对端的 DPD 回应报文时，将删除此 IPsec SA。合法值：10-3600 单位：秒
            + DPD重试间隔：用于 IPSec 邻居状态的检测时间间隔。启动 DPD 功能后，当接收端在触发 DPD 的时间间隔内收不到对端的 IPSec 加密报文时，能够触发 DPD 查询，主动向对端发送请求报文，对 IKE 对等体是否存在进行检测。合法值：1-60，单位：秒

- IPsec高级选项（第2阶段）
  - 完美前向加密（PFS）：PFS 是一种安全特性，指一个密钥被破解，并不影响其他密钥的安全性，因为这些密钥间没有派生关系。IPSec 第二阶段的密钥是从第一阶段的密钥导出的，当第一阶段 IKE 密钥被窃取后，攻击者将可能收集到足够的信息来导出第二阶段 IPSec SA 的密钥，PFS 通过执行一次额外的 DH 交换，保证第二阶段密钥的安全。
  - IPsec SA生命周期：设置 IPSec SA 的生存周期。IPSec 协商建立 SA 时，采用本端设置的生存周期和对端的生存周期中较小的一个
  - IPsec SA空闲超时时间：当 IPsec 建立成功后在配置的空闲时间内没有数据传输，则IPSec SA 就会失效。IPSec SA 快要失效前，IPSec 协商建立新的 SA，这样在旧的 SA 失效前新的 SA 就已经准备好。

![1720522740482-27d8e941-2153-447e-8c76-0b57d50abe32.png](./img/WnWQlWcKwJH_STyE/1720522740482-27d8e941-2153-447e-8c76-0b57d50abe32-078756.png)

- Tunnel高级选项
  - 隧道启动方式：设置 IPsec 启用方式
            + 自动连接：应用 IPSec 策略后，自动完成 IKE 的协商建立 IPSec 隧道。常用于客户端模式
            + 仅响应连接：只会被动接收 IPSEC请求，不会主动发起连接。常用于 server 模式
            + 按需连接：接口有 IPsec 定义的报文通过时，才完成 IKE 的协商，建立 IPSec 隧道
    - 重启接口前连接失败次数：当隧道连接失败次数达到设定的值以后，重启建立隧道的物理接口；默认为0，即失败后也不重启接口，合法值：1-12次
    - 重启设备前连接失败次数：当隧道连接失败次数达到设定的值以后，重启当前设备；默认为0，即失败后也不重启设备，合法值：1-32次
    - 本地/远端发送证书规则：可选择请求时发送证书、总是发送证书、不发送证书
            + 总是发送证书：有些 ipsec 服务不会发送“请求证书”请求，但是它的本地又没有保存对端的证书，那么对端就必须要配置成“总是发送证书”才能建立 IPSEC
            + 请求时发送证书：只有当对端发送请求时，才发送本地证书
            + 不发送证书：不管对端是否发送请求，本地都将把自己的证书发送到对端
    - ICMP探测
            + ICMP 探测服务器：IPsec 探测的对端主机地址
            + ICMP 探测本地地址：IPsec 保护流量的源地址
            + ICMP 探测间隔时间：设备发送 ICMP 探测报文的时间间隔
            + ICMP 探测超时时间：在设置的 ICMP 探测超时时间内，没有收到 ICMP 响应包认为本次ICMP 探测超时
            + ICMP 探测最大重试次数：设置 ICMP 探测失败时的最大重试次数（达到最大次数后会重重新启动 IPsec 服务）

### 3.4.2 GRE

通用路由封装（GRE）定义了在任意一种网络层协议上封装任意一个其它网络层协议的协议。GRE 可以作为 VPN 的三层隧道协议，为 VPN 数据提供透明传输通道。简单来说，GRE是一种隧道技术，提供了一条通路使封装的数据报文能够在这个通路上传输，在隧道的两端分别对数据报进行封装及解封装。

![1720522741408-cd5da5c6-3d60-4e77-803a-2126a0068eaf.png](./img/WnWQlWcKwJH_STyE/1720522741408-cd5da5c6-3d60-4e77-803a-2126a0068eaf-486625.png)

GRE参数说明如下：

- 启用：启用或禁用 GRE 功能
- 接口标识：设置 GRE 隧道名称，范围 1-100
- 网络类型：选择 GRE 网络类型
- 本地虚拟IP：设置本地虚拟 IP 地址
- 对端虚拟IP：设置对端虚拟 IP 地址，若为子网类型，此项为本地子网掩码
- 源地址类型：选择源地址类型，并设置对应类型的 IP 地址或接口名称
- 本地IP地址：配置 GRE 隧道的源地址
- 对端地址：配置 GRE 隧道的目的地址
- 密钥：设置隧道的认证密钥，两端配置需一致
- MTU：设置 GRE 隧道的最大传输单元，以字节为单位
- 启用NHRP：下一跳解析协议，用于连接到非广播多路访问 （NBMA）式子网络的源站决定到达目标站间的 “ NBMA 下一跳 ”的互联网络层地址和 NBMA 子网地址
  - NHS地址：对端 NHS 服务器地址
  - 认证密钥：NHRP 的认证密钥
  - 维持时间：合法值1-65535
  - 禁止 NHRP Purge 消息：启用/禁止
- IPsec Profile：禁用，与 IPsec 扩展结合使用
- 说明：GRE 隧道的描述信息

**说明：**

- NHRP 只用于 DMVP 网络，一般 GRE 不需要开启 NHRP。
- GRE 一般用于两端地址都是固定公网情况。

### 3.4.3 OpenVPN

在 OpenVPN中，如果用户访问一个远程的虚拟地址（属于虚拟网卡配用的地址系列，区别于真实地址），则操作系统会通过路由机制将数据包（TUN 模式）或数据帧（TAP 模式）发送到虚拟网卡上，服务程序接收该数据并进行相应的处理后，通过 SOCKET 从外网上发送出去，远程服务程序通过 SOCKET 从外网上接收数据，并进行相应的处理后，发送给虚拟网卡，则应用软件可以接收到，完成了一个单向传输的过程，反之亦然。

- 高级选项
  - 虚拟IP 起始地址：客户端的虚拟 IP 起始地址通常是由服务器端进行分配和设置的。

  - 虚拟IP个数：服务器可分配的虚拟IP个数

    - Tun：大多时候被用于基于 IP 协议的通讯
    - Tap：允许完整的以太网帧通过 Openvpn 隧道,提供非 ip 协议的支持
  - 网络类型：可选择net30、p2p、subnet（接口类型为tap时可忽略此项）
    - net30：从 pool 中选择 4 个掩码为 30 的 ip，将中间两个 ip 中的大者作为 client 的虚拟网卡 ip；将小者作为 client 的对端 ip
    - p2p：从 pool 中选择一个 ip 作为 client 的虚拟网卡 ip,将自己的实际虚拟网卡 ip 作为 client 的对端 ip
    - subnet：从 pool 中选择一个 ip 作为 client 的虚拟网卡 ip,将自己的子网掩码作为 client 的子网掩码
  - 桥接接口：用于连接多个虚拟机或网络设备，使它们可以在同一网络中通信（当接口类型配置为tun模式时可忽略此项）
  - 加密算法：OpenVPN 传输数据时使用的加密协议，必须与服务器保持一致
  - HMAC：OpenVPN 传输数据时采用的校验方式，校验不通过数据传输失败，必须与服务器一致
  - LZO压缩：OpenVPN 数据传输时采用的压缩形式
  - 重定向网关：使 Client 的默认网关指向 OpenVPN，让 Client 的所有流量都通过 OpenVPN 接口转发
  - Remote Float：允许远端改变它的 IP 地址/端口
  - 连接检测时间间隔：OpenVPN 建立成功后，发送连接检测报文的时间。
  - 连接检测超时时间：OpenVPN 连接检测失败后，当达到最大失败次数后，L2TP 重新建立连接。
  - MTU：OpenVPN 接口的最大传输单元，单位：字节
  - 调试日志：点选启用
  - 专家配置：配置 OpenVPN 扩展参数
  - 导入配置：选择需要导入的 OpenVPN 配置文件

Scep（Simple Certificate Enrollment Protocol）简单证书注册协议，是 cisco 和Verisign 一起制定的设备证书管理的通信协议。它利用现有的 PKCS#7 和 PKCS#10 协议技术结合，享有广泛的支持客户端和 CA 的实现。

证书管理参数说明如下：

- 请求状态：当前设备申请证书的状态，分为 initialize Enrolling、Re-Enrolling、Complete 四种状态
- 证书保护密钥：申请证书时设置的证书保护密钥，起到数字证书加密作用，当导入和导出证书时配置的保护密钥和证书申请时的密钥一致才能使用

#### 3.5.1.1 串口设置

根据与IG相连的终端设备的串口参数设置IG串口的参数，实现IG与终端设备的正常通信。

串口设置参数说明如下：

- 串口类型：IG不同系列的串口类型及数量不相同，根据实际情况进行选择
- 波特率：这是一个衡量符号传输速率的参数。它表示每秒钟传送的符号的个数
- 数据位：设置通信中实际数据位的参数
- 校验位：在串口通信中的检错方式，一般是奇偶校验或者无校验两种方式
- 停止位：用于表示单个包的最后一位。典型的值为 1，1.5 和 2 位
- 软件流控：串口通讯的流控提供了由于某种原因不能进行通讯时阻塞通讯的一种机制。流控可以使数据接收设备在不能接收数据时通知数据发送设备，使其停止发送
- 串口描述：自定义描述

**注意：**

- IG的串口参数与相连的终端设备的串口参数必须设置一致
- DTU 功能和 GNSS 串口转发功能不能同时开启

#### 3.5.1.2 DTU1

![1720522743402-f7be2e2c-9bf7-41eb-8df6-0ffb529e456e.png](./img/WnWQlWcKwJH_STyE/1720522743402-f7be2e2c-9bf7-41eb-8df6-0ffb529e456e-755197.png)

DTU 1 参数说明如下：

- 启用：点选启用
- DTU协议：可选择透明传输、TCP 服务器、RFC2217模式、IEC101 转 104、Modbus 网桥、DC 协议
  - 透明传输和 TCP 服务器：若选用透明传输设备作客户端，则选用 TCP 服务器设备作服务器端
  - RFC2217 模式：选用此模式后不用再设置串口配置
  - IEC101 转 104：适用于电力行业，功能和 TCP 相似
- 传输协议：可选TCP 协议和 UDP 协议
- 连接类型：可选长连接和短连接
  - 长连接：TCP 客户端和 TCP 服务器建立连接后一直保持 TCP 连接
  - 短连接：TCP 客户端和 TCP 服务器建立连接后，当空闲时间内没有数据传输，自动断开与服务器TCP 连接
- 心跳间隔：TCP 客户端和 TCP 服务器建立连接后，定期发送TCP心跳报文的时间间隔，用于检测客户端和服务器之间的连通性
- 心跳重试次数：当 TCP 心跳超时后，设备重新发送 TCP 心跳，达到设置的心跳重试次数后，重新建立 TCP 连接。
- 串口缓存帧个数：串口收发数据时设置的串口缓存大小，默认 4K
- 串口分帧长度：串口发送数据时，设置的一帧数据的大小。当达到设定值后开始发送。合法值：1-1024，单位：字节
- 串口分帧间隔：当串口发送数据时候，当发送间隔大于设置的分帧间隔后，设备会自动分帧发送，合法值：10-65535，单位：毫秒
- 最小重连间隔：用户自定义最小重连间隔时间。设备启动连接时如果没连接成功会按照此最小重连间隔时间重连。直到达到用户自定义的最大重连间隔。合法值：15-60，单位：秒
- 最大重连间隔：用户自定义最大重连间隔时间。设备启动连接且连接时间达到最大重连间隔时间后，每隔此间隔时间（即用户自定义的最大重连间隔时间）连接一次。合法值：60-3600，单位：秒
- 多中心策略：可选择并发、轮询
  - 并发：同时去连接目的 IP 地址列表中的中心
  - 轮询：先连接列表前边的中心，若连接上就不再连接后边的；若没有连接上则按照自前往后的顺序去连接，直到连接上一个中心为止
- 源接口：一般用户不用选择
- 本地IP地址：源接口选择 IP 时对应的设备接口 IP 地址。一般用户不用配置，可为空
- DTU标识：自定义，成功连接服务器后自动向服务器发送的 DTU 标识。也可以不配置，保持空状态
- 调试日志：点选开启，开启后将会打印DTU相关日志
- 开启ReportID：点选开启
- 心跳间隔时间：启用 ReportID 才需设置此项合法值：1-65535，单位：秒
- 心跳包内容：启用 ReportID 才需设置此项
- 目的IP地址
  - 服务器地址：自定义设备要连接的服务器 IP 地址
  - 服务器端口：自定义设备要连接的服务器端口

说明：目的 IP 地址最多可以设置 10 个。
