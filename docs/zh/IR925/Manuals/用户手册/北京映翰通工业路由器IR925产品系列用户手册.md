# 第1章 认识设备

## 1.1 概述

IR925系列工业级蜂窝路由器是一款集成高速5G、Wi-Fi 6、虚拟专用网络（VPN）等核心技术的物联网专用设备，其不仅提供不间断网络接入能力与全面安全功能，构建安全可靠的业务数据链路，还嵌入硬件看门狗并支持多层链路检测机制，确保通信可靠稳定。同时，IR925搭载针对物联网行业应用的智能软件服务，可灵活适配各类场景需求。该系列产品深度结合InHand DeviceLive云平台，支持用户通过云端统一集中管理设备、实时掌握运行状态，大幅节省部署与运维成本、提升管理效率，尤其适用于无人值守设备与站点的联网场景，能为企业数字化转型及大规模物联网部署提供一站式、高可靠的联网解决方案。

按照以下步骤进行操作将设备恢复出厂设置：

1. 设备上电，等待SYS指示灯熄灭的瞬间长按Reset按钮5-10秒，然后松开Reset按钮，此时SYS指示灯红色闪烁。
2. 再次长按Reset按钮，直到SYS指示灯红色常亮，然后松开Reset按钮，此时设备运行恢复出厂程序。

# 第2章 安装与首次使用

## 2.1 安装前准备

在进行设备安装前，需确认以下环境要求和工具准备：

| 项目 | 要求 |
|------|------|
| 电源 | 需使用原装电源适配器 |
| 环境温度 | 需满足规格书所要求的温度范围 |
| 环境湿度 | 需满足规格书所要求的湿度范围 |
| 网络环境 | 需准备可用的蜂窝/有线/Wi-Fi网络 |
| SIM卡 | 如使用蜂窝网络，需准备运营商SIM卡 |
| 网线 | 如使用有线网络，需准备标准网线 |
| 电脑 | 用于访问设备Web管理界面 |

> **注意**：安装设备时禁止将设备安装在强电磁干扰环境下，避免与高功率设备过于接近。

## 2.2 安装指南

### 2.2.1 天线安装与SIM卡插入

1. 安装蜂窝和Wi-Fi天线并插入SIM卡。

> **警告**：插拔SIM卡前，需拔掉电源，以免造成数据丢失或设备损坏。

### 2.2.2 电源与网络连接

1. 连接电源和网线，任意LAN口连接PC。

### 2.2.3 PC网络配置

将PC的IP地址设置为与工业路由器处于同一网段。

- 设备LAN口默认启用DHCP Server功能，将PC设置为"自动获取IP地址"，PC即可通过IR925获取IP地址，需确认该地址是否处于192.168.2.0网段。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1747990777977-96d6b368-5f68-41c1-a456-43b0e1d5e058-091388.webp" alt="PC自动获取IP地址设置"></p>

<p align="center"><strong>图 2-1 PC自动获取IP地址设置</strong></p>

- 若PC无法通过DHCP的方式自动获取IP地址，则需手动将PC的IP地址配置为192.168.2.2~192.168.2.254中的任一地址；默认网关为192.168.2.1，子网掩码为255.255.255.0，DNS设置为8.8.8.8或从运营商获取的DNS地址。

完成安装后，需进行以下检查：

- [ ] 设备已上电，系统状态灯正常显示

# 第3章 常用场景配置

## 场景1：蜂窝联网

**目标**：通过5G/4G蜂窝网络接入互联网。

**前提**：已插入SIM卡并安装天线，设备已上电。

**预计用时**：约5分钟。

**操作步骤**：
1. 安装蜂窝天线并插入SIM卡。
2. 设备上电后，IR925将自动建立拨号连接并接入互联网。
3. 如需配置APN参数，进入【Internet】→【蜂窝连接】菜单，点击"编辑"按钮，选择"拨号参数集"为手动后进行配置。

**验证方法**：
1. 检查蜂窝状态灯，确认蜂窝连接成功（绿色常亮）。
2. 检查蜂窝信号灯，确认信号质量。
3. 在设备Web界面的【仪表盘】中查看联网状态。

**常见问题**：
- 无法连接蜂窝网络：检查SIM卡是否正确插入、APN参数是否正确。
- 信号弱：检查天线是否连接、调整设备位置。

## 场景2：有线接入网络

**目标**：通过有线方式接入互联网。

**前提**：已准备网线，上行设备（如光猫、上级路由器）可用。

**预计用时**：约5分钟。

**操作步骤**：
1. 用网线将IR925的WAN1/LAN1端口与上行设备连接。
2. 如需配置第二条有线链路，在【Internet】→【上行链路列表】中手动添加WAN链路，并选择合适的联网方式（该端口支持DHCP/静态IP/PPPoE三种有线接入模式）。
3. 完成设置后，用网线将IR925的LAN2/WAN2口与上行设备连接。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1749710233048-e12fb9e5-fd1e-4a4a-b1b5-d6f6da056cad-358189.webp" alt="WAN/LAN切换配置"></p>

<p align="center"><strong>图 3-1 WAN/LAN切换配置</strong></p>

**验证方法**：
1. 在【仪表盘】→【接口状态】中查看WAN接口状态。
2. 在【状态】→【链路监控】中查看链路健康状态。

**常见问题**：
- 无法获取IP地址：检查网线连接、确认DHCP服务器正常。
- PPPoE拨号失败：检查账号密码是否正确。

## 场景3：Wi-Fi(STA)接入网络

**目标**：通过Wi-Fi方式接入互联网。

**前提**：现场有可用的Wi-Fi接入点（AP）。

**预计用时**：约5分钟。

**操作步骤**：
1. 进入【Internet】→【上行链路列表】菜单，点击"添加"按钮。
2. 选择"Wi-Fi(STA)"，选择合适的频段，填入需连接的AP的SSID信息和密码。
3. 为保证连接的有效性，需核对AP的认证策略和加密策略，IR925需与其一致。
4. 完成设置后，IR925会主动向AP发起连接。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1749710480110-4da9e645-72dd-4f70-9797-25c8a44e5637-000805.webp" alt="Wi-Fi(STA)配置"></p>

<p align="center"><strong>图 3-2 Wi-Fi(STA)配置</strong></p>

**验证方法**：
1. 在【仪表盘】→【接口状态】中查看Wi-Fi(STA)接口状态。
2. 在【状态】→【链路监控】中查看链路健康状态。

**常见问题**：
- 无法连接AP：检查SSID和密码是否正确、确认AP信号强度。
- 认证失败：核对AP的认证策略和加密策略是否与IR925配置一致。

**常见问题**：
- 设备无法上线：检查设备是否已联网、确认序列号填写正确。

## 4.1 设备监控

### 4.1.1 设备总览

## 4.2 Internet

用户可在"Internet"功能下配置每个上行接口的参数和操作模式。IR925支持有线、蜂窝、Wi-Fi三种接入网络模式。其最多可以支持三个上行链路，包括WAN、蜂窝和Wi-Fi（STA）。WAN和Wi-Fi（STA）接口需要手动添加，并可根据需要删除；蜂窝上行链路不可删除。

### 4.2.1 有线连接

IR925支持三种有线连接方式接入网络：DHCP、静态IP和PPPoE。用户可通过单击"编辑"按钮来修改连接方式。默认选中"DHCP"模式。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1750753317134-ee006e78-506a-4a29-b425-97c805176fb4-261842.webp" alt="有线连接配置"></p>

<p align="center"><strong>图 4-17 有线连接配置</strong></p>

- DHCP：设备的WAN接口默认开启DHCP服务，用户只需用网线将该接口与互联网（如光猫、上级路由器等网络接入设备）连接，即可实现自动联网。
- 静态IP：用户可手动配置IP地址，该地址既可以是从运营商处获取的地址，也可以是与上行设备（如光猫、上级路由器）处于同一网段的地址；配置完成后，路由器将通过所指定的静态IP接入网络。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1750753373100-047496f5-575a-4ff9-839b-37fa5c299918-465751.webp" alt="静态IP配置"></p>

<p align="center"><strong>图 4-18 静态IP配置</strong></p>

- PPPoE：用户可在设备中设置宽带拨号参数（如运营商提供的账号、密码），配置完成后，路由器将自动通过PPPoE拨号方式接入网络。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1750753392588-231925d5-a0aa-42f2-a38e-b164fa30b4cf-823223.webp" alt="PPPoE配置"></p>

<p align="center"><strong>图 4-19 PPPoE配置</strong></p>

- 删除WAN1接口后，原WAN1接口角色将会切换回LAN1。
- 删除WAN1接口后，与WAN1接口相关的静态路由、入站/出站规则、端口转发、策略路由、流量整形相关配置都将被删除。
- 添加WAN1接口后，原LAN1接口角色将切换成WAN1。

### 4.2.2 无线连接

IR925支持作为客户端接入现场AP的网络，点击【Internet】菜单中"添加"按钮，选择"Wi-Fi(STA)"，填写SSID名称和密码等参数。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1750753668506-c1d0647f-4f6a-4ac8-85da-30892cf9cec5-028071.webp" alt="Wi-Fi(STA)添加"></p>

<p align="center"><strong>图 4-20 Wi-Fi(STA)添加</strong></p>

- Wi-Fi(STA)添加后，IR925在Wi-Fi功能中相同频段的SSID将被禁用且状态字段不可修改。
- Wi-Fi(STA)删除后，IR925在Wi-Fi功能中相同频段的SSID状态字段可以修改。
- Wi-Fi(STA)删除后，与Wi-Fi(STA)接口相关的静态路由、入站/出站规则、端口转发、策略路由、流量整形的配置都将被删除。

### 4.2.3 蜂窝连接

通常情况下，按照说明安装SIM卡和Wi-Fi天线后，IR925路由器将在上电后自动拨号并接入网络。当用户需要配置APN参数时，点击"Cellular"后的"编辑"按钮进入配置界面，选择"拨号参数集"为手动即可进行配置。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1750754710878-7f4d9c23-1bef-444b-935d-acab8bf6da92-338900.webp" alt="蜂窝连接配置"></p>

<p align="center"><strong>图 4-21 蜂窝连接配置</strong></p>

IR925在支持蜂窝上网的基础上，新增流量策略功能。策略启用后，SIM卡在流量到达阈值后执行对应动作，流量统计将在次月起始日重置。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1750758699673-028e7050-634b-409d-a0c2-a96c245b7af6-892299.webp" alt="流量策略"></p>

<p align="center"><strong>图 4-22 流量策略</strong></p>

- 动作：SIM卡流量到达阈值后触发的动作。
  - 通知：生成流量到达阈值的事件，但不会停止转发业务流量。
  - 仅保留云管理：生成流量到达阈值的事件，只转发云端管理流量，不转发访问Internet的业务流量。
  - 切换SIM：生成流量到达阈值的事件，并切换到另一张SIM卡拨号上网。

- 部分专网环境下，需要手动关闭【Internet】菜单下的"链路探测"功能，避免因探测不通导致蜂窝不能正常工作。
- 部分情况下，需要手动配置蜂窝接口的子网掩码，以保证ARP功能正常工作。
- 插拔SIM卡前，需拔掉电源，以免造成数据丢失或设备损坏。

### 4.2.5 上行链路设置

用户可通过"Internet>上行链路设置"功能配置链路检测的探测地址，并配置不同上行接口之间的协作模式。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1747990789787-27fc65cd-868f-4798-bd12-cf5bfc80bde0-576873.webp" alt="上行链路设置"></p>

<p align="center"><strong>图 4-24 上行链路设置</strong></p>

- 链路探测开关：设备默认开启链路探测功能，默认的探测地址为DNS地址。当链路探测功能关闭后，用户无法在【状态】菜单下查看各上行接口的延迟、抖动、丢包、信号强度等信息。

- 对Internet菜单的修改可能导致设备联网中断，需谨慎操作！
- 当链路探测地址为空时，默认探测上行接口获取的DNS地址。填写探测地址后，所有上行接口只会探测用户填写的地址。
- 当路由器处于链路备份模式时，用户可自定义探测参数，设备将根据使能的探测项进行链路切换。未使能探测项时，上行链路将按照优先级顺序由上至下依据链路连通性触发切换。
- 当设备处于负载均衡模式时，所有可以正常工作的上行链路都会转发业务流量。

## 4.3 本地网络

用户可在【本地网络】功能中自定义本地子网，包括配置本地局域网的地址段、VLAN ID、DHCP服务等参数。完成配置后，用户需要进一步在【接口管理】中将这些配置应用到设备LAN口，或在Wi-Fi设置中将其应用到目标SSID。这一系列操作旨在确保客户端设备能够按照规划的网络地址顺利连接到本地网络。

点击"添加/编辑"按钮，添加新的本地网络或对已有本地网络进行编辑。

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1757986113943-b2f4b842-06ff-444a-8933-f7e361ccaa72-713808.webp" alt="本地网络添加/编辑-1"></p>

<p align="center"><strong>图 4-26 本地网络添加/编辑（1）</strong></p>

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1757986133686-feab6189-df59-4bf8-9684-1aa8facde959-433807.webp" alt="本地网络添加/编辑-2"></p>

<p align="center"><strong>图 4-27 本地网络添加/编辑（2）</strong></p>

- 名称：用于标识网络，用户可通过此名称选择在"Wi-Fi"和"接口管理"中应用的网络。
- 模式：选择当前子网是2层透明模式或3层IP模式，默认"IP模式"。
- VLAN：将局域网划分到不同虚拟逻辑的网络中。默认所有的接口和Wi-Fi使用default（VLAN1）网络。

- 默认的本地网络不可删除且仅支持修改IP地址/掩码和DHCP Server配置。
- 已添加的本地网络模式不支持修改。
- VLAN Only模式用于二层透传，无需配置IP地址/掩码、DHCP Server。

## 4.4 Wi-Fi

Wi-Fi是一种广泛使用的无线通信技术，用于将计算机、智能手机、平板电脑和其它设备连接到互联网或局域网络。Wi-Fi技术允许设备通过无线信号在一定范围内传输数据，提供了无需物理连接即可访问网络的便利性。

### 4.4.1 SSIDs

## 4.6 VPN

虚拟专用网络（VPN）是一种加密技术，用于在公共互联网上创建安全的、私人的网络连接。其允许用户通过互联网安全地访问私人网络资源，无论其身在何处。VPN通过加密通信数据，确保通信的机密性和安全性，防止未经授权的访问。这种技术对于连接到公司网络、维护在线隐私以及访问受限制的内容非常有用。VPN具有广泛的应用领域，包括企业、个人和移动设备领域。其是保护隐私和数据安全的关键工具之一。

### 4.6.1 IPSec VPN

IPSec（Internet Protocol Security）VPN是一种用于加强网络通信安全性的协议套件，旨在通过加密和认证来保护数据的传输。其广泛应用于建立安全的远程访问、站点到站点连接以及虚拟专用网络（VPN）。

点击"VPN>IPSec VPN"下的"添加"按钮，添加新的IPSec VPN。

<p align="center"><strong>图 4-37 IPSec VPN添加</strong></p>

<p align="center"><strong>图 4-38 IPSec VPN参数（1）</strong></p>

两端配置完成后，隧道即可建立。用户可在【状态>VPN】菜单中查看隧道建立状态。下方仅列出部分必填项的说明。

- 名称：路由器创建的IPSec VPN名称标识，主要用于本地管理。
- IKE版本：设置IKE协议使用的版本号，支持IKEv1和IKEv2。
- 预共享密钥：在IKE协商的两端设备需要配置相同的认证密钥。
- Internet接口：本地建立IPSec VPN使用的上行接口。
- 隧道模式：设置IPSec对IP报文的封装模式，支持隧道模式和传输模式。
- 对端地址：与IR925建立隧道的对端通信地址。

IR925在用IPSec VPN建立连接时，默认以使用公网地址的设备作为服务器，IPSec服务器侧的对端地址需要配置为0.0.0.0，IPSec客户端侧的对端地址填写IPSec服务器侧的公网地址。

- 本地子网：填写需要通过IR925 IPSec VPN隧道通信的网段地址。
- 对端子网：填写隧道另一端需要通过IPSec VPN隧道通信的网段地址。
- IKE策略：支持设置IKE协议的配置。

    - 取值范围：DES、3DES、AES128、AES192、AES256，默认"AES128"
  - 认证方式：设置IKE使用的认证算法。

- IPSec策略：支持设置IPSec参数配置。
  - 安全协议：设置ESP协议使用的安全协议。
    - 取值范围：DES、3DES、AES128、AES192、AES256，默认使用"AES128"
  - 加密方式：设置ESP协议的加密算法。
    - 取值范围：MD5、SHA1、SHA2-256、SHA2-384、SHA2-512,默认"SHA1"
  - PFS组：IPSec使用安全策略发起一个协商时，在阶段2的协商中进行一次附加的密钥交换以提高通讯的安全性参数。
    - 取值范围：1、2、5、14、15、16、19、20
  - 超时时间：设置IPSec SA老化时间，默认86400秒。

## 4.7 安全

在【安全】菜单下，用户可以根据需要设置防火墙、策略路由、流量整形相关的高级功能。

### 4.7.1 防火墙

防火墙当前包括：入站规则、出站规则、端口转发、MAC地址过滤、域名过滤等功能，用户可根据具体的业务需求进行配置。

**入站/出站规则**

用户可通过"安全>防火墙>出站规则/入站规则"功能来实现基于接口的流量进出控制。例如，当遭受来自特定源IP地址的大量攻击行为时，可以使用防火墙入站规则来限制来自该IP地址的流量。

此外，IT人员还可通过防火墙出站规则来限制某些用户访问外部网络的权限。入站规则和出站规则在可配置的内容上相同，唯一的区别是默认规则。以下是添加入站规则的示例：

<p align="center"><img src="./img/XpDw9oJ5ZXG77a4u/1757992289495-2566b066-3fb0-4b9b-8e53-d5db9b002d56-712186.webp" alt="入站/出站规则"></p>

<p align="center"><strong>图 4-42 入站/出站规则</strong></p>

- 名称：设置入站规则的名称，用于本地标识。
- 状态：规则的功能开关
- 接口：对于出站规则，指流量流出路由器的上行接口。对于入站规则，指流量进入路由器的上行接口。
- 协议：匹配流量的协议类型，支持Any、TCP、UDP、ICMP、自定义。
- 源：匹配流量的源地址，支持自定义，默认Any。
- 目的：匹配流量的目的地址，支持自定义，默认Any。
- 动作：入站规则/出站规则的对匹配流量的动作，支持允许和拒绝。

- 出站规则：通过路由器访问外部的流量管理规则，默认全部放行。
- 支持通过拖动"优先级"一栏的图标调整入站规则/出站规则的优先级。

 **端口转发**

端口转发也被称为端口映射或端口重定向，用于将网络数据包从一个网络端口（或地址）重定向到另一个网络端口或地址。用户可在"安全>防火墙>端口转发"配置端口转发规则，当外部流量访问路由器的特定端口时，设备会将数据转发至内部客户端的相应端口上，实现从外部访问路由器内部的服务。

例如：当需要从外部访问内部192.168.2.10客户端的1024端口服务时，将此客户端的端口映射到WAN1接口下的1024端口，外部网络只需在浏览器输入"https://WAN1地址:1024"即可访问目标设备数据，其中WAN1地址为WAN1接口的IP地址。

<p align="center"><strong>图 4-43 端口转发</strong></p>

- 名称：端口转发规则的名称，用于本地标识。
- 状态：端口转发规则的功能开关。
- 接口：对内部客户端提供映射功能的上行接口，上行接口需要公网地址支持。
- 协议：端口映射流量的协议类型，支持TCP、UDP、TCP&UDP。
- 公网端口：对内部客户端提供映射功能的上行接口的端口号，需要与本地端口输入范围保持一致。
- 本地地址：外部网络需要访问的位于路由器下的目标设备地址。
- 本地端口：外部网络需要访问的目标设备的端口，需要与公网端口输入范围保持一致。

**网络地址转换**

网络地址转换是一种将IP数据包在通过路由器时重新包内源IP地址或目的IP地址的技术，可以将私网IP地址转换为合法IP地址。用户可在"安全>防火墙>网络地址转换"中配置源IP地址转换或目的IP地址转换规则。

- DNAT：目的IP地址转换，以可将IP数据包中目的IP地址字段的值转换为设定值。

- TCP：地址转换规则在TCP报文中生效。
- UDP：地址转换规则在UDP报文中生效。

- 目的：数据包的目的IP地址。
- 转换后地址：根据所选择的地址转换类型，设置需要转换的值。

**MAC地址过滤**

MAC地址过滤是指禁止（或允许）MAC地址列表中的设备上网，即通过路由器上的MAC地址过滤功能控制局域网内设备的上网请求。用户可在"安全>防火墙>MAC地址过滤"中配置MAC地址过滤规则。

### 4.8.3 DNS Server

DNS（Domain Name System）服务器是一种关键的网络组件，其负责将人类可读的域名（如www.example.com）转换为计算机能理解的IP地址（如192.168.1.1）。DNS服务器充当了互联网上的地址簿，其帮助计算机和设备找到其他设备的位置，并确保信息能够在网络上正确传递。

### 4.8.5 Dynamic DNS

### 4.8.6 GNSS

GNSS即全球导航卫星系统（Global Navigation Satellite System），是一个由多卫星星座组成的系统，旨在为全球范围内的用户提供高精度的定位、导航和授时服务。通过接收多颗卫星发射的信号，设备能够精准计算出自身所处的位置（经度、纬度、海拔），规划行进路线，并获取精准的时间信息。目前全球主要的GNSS系统有中国的北斗卫星导航系统（BDS）、美国的全球定位系统（GPS）、俄罗斯的格洛纳斯卫星导航系统（GLONASS）以及欧盟的伽利略卫星导航系统（Galileo）。本设备集成的GNSS功能可同时接收多个系统信号，以提升定位的准确性和稳定性。

用户可通过"服务>GNSS"进入配置页面。在这里可以开启或关闭GNSS功能，配置GNSS IP转发和GNSS串口转发。开启GNSS后，还可以启用"更新设备时间"实现设备校时。

## 案例1：工业现场远程监控组网

**场景描述**：在工业现场，IR925作为边缘网关，通过蜂窝网络将现场PLC、传感器等设备数据上传至云平台，实现远程监控和运维。

**设备角色**：本设备作为边缘网关，负责将现场有线/无线设备接入互联网，并通过VPN保障数据传输安全。

**配置步骤**：
1. 按照[安装与首次使用](#第2章-安装与首次使用)完成设备安装和蜂窝联网。
2. 在【Internet】菜单中确认蜂窝连接正常。
3. 如需数据传输加密，在【VPN】菜单中配置IPSec VPN或L2TP VPN。
4. 将设备添加至Device Live平台，实现远程监控。

# 附录 安全注意事项

1. 需使用原装电源适配器，避免因电源适配器和设备不匹配造成设备损坏。
2. 安装设备时禁止将设备安装在强电磁干扰环境下，避免与高功率设备过于接近。安装完成后需确认设备是否易发生晃动，避免因设备跌落造成设备损坏。
3. 需确认设备使用环境满足规格书所要求的温度和湿度。
4. 需定期检查设备线缆，包括网线，电源适配器连接线，保持线材清洁，如有发生破损需及时更换。
5. 清洁设备时，避免使用化学试剂直接喷洒在设备表面，避免损害外壳或内部组件。需使用柔性的布料清洁产品。
6. 禁止私自拆修或改装设备，这将会导致安全问题并且设备会失去保修资格。
