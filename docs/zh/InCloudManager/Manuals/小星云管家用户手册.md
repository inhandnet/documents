## 快速指南

### 机构管理

### 名词解释

| **名词** | **说明** |
| --- | --- |
| **机构/企业** | 在系统中，使用邮箱注册成功后，即创建了一个机构/企业。各个机构/企业之间的数据是完全隔离的。 |
| **组织** | 注册机构/企业成功后，您的机构/企业默认为是一级组织，您可以根据实际业务需求，在一级组织下创建其他组织。一级组织拥有最高数据权限，能查看当前机构的所有数据。下级组织只能看到当前组织及其下级组织的数据，无法看到上级组织的数据。 |
| **内部用户** | 属于当前机构/企业的用户。 |
| **外部用户** | 不属于当前机构/企业的用户，他们无法访问当前机构/企业的数据。但是由于业务需求，可以通过邀请的方式，使其能访问到当前机构/企业的数据，如提供技术支持。 |

### 组织管理

进入映翰通云服务的【**用户管理**】页面，可以管理当前机构的组织关系及用户。映翰通云服务支持多级组织，当您需要将业务根据区域、场所等划分管理时，您可以创建多个子组织，将业务单独管理。

:::color3
┃**说明：** 同等级的组织数据互相隔离；上级组织能看到下级组织的数据。

:::

![1722396831829-bbcdba37-1a95-4923-9f36-dd212dac680f.png](./img/ejt7ppogIACxPqAy/1722396831829-bbcdba37-1a95-4923-9f36-dd212dac680f-820747.png)

**1.1 添加组织**

点击“**新增**”，输入组织名称并选择上级组织，点击“**确定**”，即可创建一个组织。

:::color3
┃**说明：** 一个机构最多创建五级组织。

:::

![1722396831743-79119837-58b4-44fc-91d7-ff44ee70b663.png](./img/ejt7ppogIACxPqAy/1722396831743-79119837-58b4-44fc-91d7-ff44ee70b663-394767.png)

| **字段名称** | **说明** |
| --- | --- |
| 组织名称 | 自定义的组织的名称。 |
| 上级组织 | 当前组织隶属于哪个组织。上级组织能看到下级组织的所有数据。 |
| 联系电话 | 组织的联系人电话。 |
| 联系邮箱 | 组织的联系人邮箱。 |
| 备注 | 组织的描述信息。 |

**1.2 添加用户**

在用户列表，点击“**添加**”，录入用户信息。添加成功后，用户即可使用账号和密码登录到平台。用户的页面权限由角色决定。用户看到的数据为当前所在组织有权限的数据。如：用户在三级组织A，则只能看到三级组织A及其下级组织的数据，无法看到一级组织、二级组织及其他三级组织的数据。

![1722396831806-2c12d178-e084-415e-a8d8-682be7958bdb.png](./img/ejt7ppogIACxPqAy/1722396831806-2c12d178-e084-415e-a8d8-682be7958bdb-431636.png)

| **字段名称** | **说明** |
| --- | --- |
| 用户名称 | 用户的名称。 |
| 登录邮箱 | 用户的邮箱信息，用于登录平台和接收平台的消息。 |
| 所属组织 | 为当前账号所在的组织。用户可以查看当前所在组织及其下级组织内的设备等资源。 |
| 应用 | 映翰通云服务提供的所有应用，包括小星云管家、DeviceLive等。 |
| 用户角色 | 会根据您所选的应用展示对应的角色。小星云管家包含四个角色，见角色部分。 |
| 登录密码 | 用于登录的密码。为了您的账号安全，请按照系统要求设置登录密码。请妥善保管账号密码并定期修改密码。 |

**1.3 角色**

小星云管家的角色包含：系统管理员、组织管理员、设备管理员、只读用户，各个角色的功能权限如下：

| **角色** | **功能权限** |
| --- | --- |
| 系统管理员 | 用户能查看系统所有数据，并拥有所有功能的操作权限。 |
| 组织管理员 | 1. 用户在小星云管家具有所有的功能权限，但在映翰通云服务中仅具备控制台、用户管理和个人信息页面的功能权限；<br/>2. 用户只能查看到当前组织及其下级组织的数据。 |
| 设备管理员 | 1. 用户拥有总览、设备、网络、分组、客户端、告警、报表、消息页面及映翰通云服务的控制台、个人信息页面的功能权限；<br/>2. 用户只能查看到当前组织及其下级组织的数据。 |
| 只读用户 | 1. 用户拥有总览、设备、网络、分组、客户端、告警、消息页面及映翰通云服务的控制台、个人信息页面的查看权限，无编辑管理权限；<br/>2. 用户可以为当前权限范围内数据生成报表，并下载查看；<br/>3. 用户只能查看到当前组织及其下级组织的数据。 |

**1.4 组织数据管理**

上级组织的管理员可以将有权限的资源划分给下级组织，包括：设备、分组、网络、用户等。

**1.4.1 转移设备**

您可以将组织内的设备转移给下级组织或某个分组，操作步骤如下：

1. 进入小星云管家的【**设备**】页面；

2. 选择需要转移的**设备**；

3. 点击“**移动**”，选择目标组织，点击“保存”。

![1722396831869-137218a0-a603-4056-b79a-000f23d0574f.png](./img/ejt7ppogIACxPqAy/1722396831869-137218a0-a603-4056-b79a-000f23d0574f-630113.png)

**1.4.2 转移网络**

您可以将组织内的网络转移给下级组织，操作步骤如下：

1. 进入小星云管家的【**网络**】页面；

2. 点击需要转移的网络的“**编辑**”按钮；

3. 选择目标组织，点击“保存”。

![1722396831839-307b9867-a4cb-494d-b695-99a798240237.png](./img/ejt7ppogIACxPqAy/1722396831839-307b9867-a4cb-494d-b695-99a798240237-410388.png)

**1.4.3 转移分组**

您可以将组织内的分组转移给下级组织，操作步骤如下：

1. 进入小星云管家的【**分组**】页面；

2. 点击需要转移的分组的“**编辑**”按钮；

3. 选择目标组织，点击“保存”。

![1722396832358-9d379994-4ec1-4905-8cbb-893d12c65fa2.png](./img/ejt7ppogIACxPqAy/1722396832358-9d379994-4ec1-4905-8cbb-893d12c65fa2-291298.png)

**1.4.4 转移用户**

您可以将组织内的用户转移给下级组织，操作步骤如下：

1. 进入映翰通云服务的【**用户管理**】页面；

2. 点击需要转移的用户的“**编辑**”按钮；

3. 选择目标组织，点击“保存”。

![1722396832462-0d0ae51a-a279-4590-bec0-0c1bf37c2630.png](./img/ejt7ppogIACxPqAy/1722396832462-0d0ae51a-a279-4590-bec0-0c1bf37c2630-323341.png)

### 跨机构的访问

当某个机构需要您提供技术支持时，除了该机构可以为您再创建一个账号或提供他的账号（内部用户）的方式外，如果您在小星云管家已注册邮箱账号，此时该机构的管理员可以通过添加“外部用户”的方式来邀请您加入该机构。既保障了客户账号的安全，又方便您提供支持。

![1722396832526-c0f858a9-b9f7-4d6a-b3cf-49ce0b72deff.png](./img/ejt7ppogIACxPqAy/1722396832526-c0f858a9-b9f7-4d6a-b3cf-49ce0b72deff-317724.png)

邀请外部用户的操作步骤如下（以上图中的机构A邀请机构C的用户甲到小星云管家提供支持为例）：

1. 机构A和B的管理员进入**映翰通云服务**的【**用户管理**】页面；

2. 在用户列表点击“**添加**”；

3. 选择用户类型为“**外部用户**”；

4. 登录邮箱填写用户甲在小星云管家的邮箱；

5. 所属组织选择需要用户甲提供支持的组织，如：三级组织A；

6. 选择应用为“小星云管家”；

7. 选择用户角色并保存。

![1722397334100-089909ae-2e3f-4cf4-9cc1-c6a5f56088bf.png](./img/ejt7ppogIACxPqAy/1722397334100-089909ae-2e3f-4cf4-9cc1-c6a5f56088bf-574848.png)

用户甲的邮箱将收到一封邀请邮件，接受邀请后，用户甲即可切换到机构A提供技术支持。

切换机构的操作步骤如下：

1. 登录到小星云管家；

2. 点击页面右上角个人信息，点击“**切换组织**”；

3. 在切换组织弹窗中，选择机构A，并点击“**切换**”，即可成功切换到机构A提供技术支持。

![1722396832537-514b0ff9-35e0-4ebc-aff9-e3a404187510.png](./img/ejt7ppogIACxPqAy/1722396832537-514b0ff9-35e0-4ebc-aff9-e3a404187510-809833.png)

## 操作手册

### 1.登录小星云管家

### 1.1 注册

使用浏览器访问小星云管家，平台地址：[https://star.inhandcloud.cn/](https://star.inhandcloud.cn/)，进入小星云管家注册登录页。（推荐使用 Google ）

注册成功后，即可使用账号密码登录，登录后进入到映翰通云服务的“控制台”页面，点击小星云管家下的**访问**，即可进入“小星云管家”。

**2.2.2手机绑定**

**2.2.3双因素身份验证**

通过二次身份验证来提高账户安全性。你可以同时设置多种身份验证方式，以便于在你丢失设备或无法获取APP验证码时，可以用另一种方式作为备份。

1. 短信验证
    1. 点击“设置”，按照操作提示绑定手机号码，绑定成功后再次登录时，会给绑定的手机号发送验证码以做二次登录验证。
    2. 当你不需要短信验证时，可以点击“关闭”来关闭短信验证。  

2. APP验证
    1. 点击“设置”，按照操作提示，在你的手机上下载安装Google Authenticator 或 Microsoft Authenticator APP。然后打开APP，扫描界面二维码来设置你的账号。
    2. 扫码后，输入APP上的6位验证码来验证身份，验证通过后，APP验证即绑定成功。绑定成功后在登录时，将需要你输入APP上的6位验证码来做二次登录验证。
    3. 当你不需要APP验证时，可以点击“关闭”来关闭APP验证。

3. 备份码验证
    1. 在首次设置双因素身份验证时，会同时生成一个备份码，备份码可用于当你无法获取短信验证码或APP验证码时，可用这个备份码来绕过二次验证。登录时选择“使用备份代码”。

### 3.1配置设备联网

要实现云端管理，必须确保设备已经联网。不同产品系列的配置方式存在差异，请参考同系列产品进行配置。各个设备因固件版本不同，界面可能存在差异，请以实际设备为准。以下以ER805 产品系列为例。

**方式一：以太网联网**

环境：用网线将PC与ER805的LAN1接口连接，WAN1接口连接Internet。

首先为PC配置一个192.168.2.X网段的IP地址（如192.168.2.100），掩码配置为255.255.255.0，也可以使用默认的DHCP自动获取IP地址。然后在谷歌浏览器中输入<https://192.168.2.1,访问页面如下：>

![1722396837056-0d252fd8-1a17-4245-9b3f-50df7e500a41.png](./img/ejt7ppogIACxPqAy/1722396837056-0d252fd8-1a17-4245-9b3f-50df7e500a41-973024.png)

- **DHCP：** 若WAN口可以通过DHCP自动获取IP地址，则设备无需任何操作即可上网。
- **静态IP：** 将“Internet”菜单下的WAN1配置为静态IP模式，并将IP地址配置为公网IP地址及掩码。
- **PPPoE：** 支持WAN口宽带接入，PPPoE拨号上网。

一般情况下，SIM卡插卡即可联网。有定制需求的用户可编辑 Internet->蜂窝，填写APN参数，实现联网。

**方式三：Wi-Fi（STA）联网**

在“Internet”菜单下点击“添加”按钮创建Wi-Fi(STA)，配置参数使其连接到现场的SSID实现联网。

### 5.2远程配置

在设备列表，支持对单个设备进行远程配置操作，包含编辑配置、查看配置、清除配置、复制配置等操作。

**5.2.1编辑配置**

设备列表支持编辑设备配置，编辑并提交配置后，会将新的配置下发给设备。此处编辑的配置为设备的个性化配置。

1. 选择目标设备；

2. 点击设备列表上方的**远程配置**，选择**编辑** 操作；

3. 在**编辑配置**弹窗中更改配置，更改完成后，点击**提交配置**。

![1722396839553-0be4f313-1a52-483d-bec2-2d9ef2ed7be2.png](./img/ejt7ppogIACxPqAy/1722396839553-0be4f313-1a52-483d-bec2-2d9ef2ed7be2-907074.png)

![1722396839665-20ff2c49-cd15-4deb-ad2a-09d3a738ea55.png](./img/ejt7ppogIACxPqAy/1722396839665-20ff2c49-cd15-4deb-ad2a-09d3a738ea55-272108.png)

编辑配置弹窗中，支持以下操作：

- 待执行配置：点击“待执行配置”按钮，此次会话修改的配置将保存至平台后台，但未下发至设备。
- 放弃更改：点击“放弃更改”，此次会话期间所做操作将被清空。
- 提交配置：点击“提交配置”按钮此次会话期间的配置将应用到设备中。

:::color3
┃**说明：**

①设备需在平台上过线且当前固件版本支持远程配置；

②设备的配置同步状态挂起时，在平台的配置将不会下发给设备。

:::

**5.2.2配置详情**

设备列表可以查看目标设备的配置信息，包含：运行配置、个性化配置、组配置、目标配置、待执行配置。

3. 在**配置详情** 弹窗中可以查看当前设备的所有配置信息。配置信息包含：

- 运行配置：设备当前运行的配置，包含用户在设备本地或云端对设备做的配置修改。
- 个性化配置：在单个设备的编辑界面或设备本地界面做的配置信息。设备的个性化配置优先于分组配置。
- 组配置：展示设备所在分组的配置信息。
- 目标配置：个性化配置与分组配置的组合，用户修改配置后期望设备执行的配置。目标配置可自定义显示/隐藏分组配置。
- 待执行配置：用户在平台修改的，但尚未与设备同步的配置。

### 6.2 SD-WAN网络

**背景：**

企业分支间往往需要相互访问以实现业务传输、视频会议等需求。传统的VPN实现方式配置复杂且排障困难。映翰通推出的SD-WAN网络功能，通过简单的操作界面，帮助用户实现分支间快速联网，提高链路灵活性的同时，大大降低了运维管理难度，给企业用户带来了良好的网络体验。

SD-WAN配置示例请参考：[https://inhandnetworks.yuque.com/unup4w/ntla5f/xv2c7yghipavsnyr](https://inhandnetworks.yuque.com/unup4w/ntla5f/xv2c7yghipavsnyr)

访问【SD-WAN网络】页面，点击添加，即可跳转至SD-WAN网络添加页面。

![1741760829660-b5d76136-7aec-47de-9a4f-c0b25446c14d.png](./img/ejt7ppogIACxPqAy/1741760829660-b5d76136-7aec-47de-9a4f-c0b25446c14d-184795.png)

【SD-WAN】网络支持Hub&Spoke组网类型，设备角色分为**中心设备、分支设备**。所有分支设备都与指定中心设备建立隧道，分支设备之间的流量通过中心设备转发。

**6.2.2添加设备**

在添加网络页面，点击（中心设备/分支设备）的“添加”按钮，选择要加到网络中的设备。

选中设备后，添加设备的公网IP映射信息。如需更改设备的网络配置，可点击本地网络的“编辑”按钮进行远程配置。

**【中心设备】**

- 中心设备需要一个公网IP地址。可以通过IP映射的方式解决公网IP不足的问题。
- 中心设备的上联设备需要在防火墙规则中放行2个端口号，并将其映射到ER系列路由器的上行接口。端口范围：1-65535。
- 一个SD-WAN网络最大支持添加5个中心设备。
- 可作为中心的设备型号：ER2000、ER805、ER605。

| **字段** | **说明** |
| --- | --- |
| 选择设备 | 需要作为中心节点的设备。 |
| 隧道端口 | 中心设备的上联设备在防火墙规则中放行的2个端口号。 |
| 公网IP映射 | 设备的公网IP。 |
| 子网 | + 默认展示当前设备本地配置的所有子网。<br/>+ 用户可根据实际需要，选择需要用于SD-WAN网络中的子网。<br/>+ 也可点击“编辑”按钮，重新配置/修改当前设备的子网。 |

![1741762281240-0e050eab-e650-408f-ada4-39f37a8d97b6.png](./img/ejt7ppogIACxPqAy/1741762281240-0e050eab-e650-408f-ada4-39f37a8d97b6-173259.png)

**【分支设备】**

- 分支设备的数量取决于中心设备的数量。例如：一个ER2000中心设备可支持500个分支设备。
- 分支设备无需公网IP。
- 可作为分支设备的型号：ER805、ER605。

| **字段** | **说明** |
| --- | --- |

### **13.3应用/取消许可**

设备需要应用许可后才能正常使用部分功能。在“**订阅 >> 设备**”页面，可以管理设备的许可应用情况，包含以下功能：  
![1722407040735-b6322209-545e-432d-8fd4-9acddea88b9f.png](./img/ejt7ppogIACxPqAy/1722407040735-b6322209-545e-432d-8fd4-9acddea88b9f-891998.png)

1、设备服务状态说明

在“**订阅 >> 设备**”可以查看到当前设备的服务状态，服务状态如下：

| 状态 | 说明 |
| --- | --- |
| 可用 | 设备能够使用连接到平台，并使用平台的功能（功能范围由设备应用的许可证决定）； |
| 不可用 | 由于设备的许可证过期或没有应用许可证，设备无法连接到平台做管理。需要为设备应用许可证。 |
| 异常 | 为设备应用的许可证到期后，设备仍能使用平台部分功能，但可能不包含您期望使用的功能。 |

2、单个应用许可

在“**订阅 >> 设备**”页面，支持为单个设备应用许可。操作步骤如下：

1. 点击设备列表操作列的**应用许可**（设备还未应用许可）；

2. 在应用许可弹窗，选择许可证类型及需要使用的**许可证**，点击**应用**。

![1722407174801-60f7a105-45df-43e1-9376-0e835e0cee8f.png](./img/ejt7ppogIACxPqAy/1722407174801-60f7a105-45df-43e1-9376-0e835e0cee8f-820557.png)

3、延长设备许可订阅时长

对于一个已有许可的设备，设备的许可证到期前，支持为设备延长许可时长，避免许可到期后服务终止，影响业务运行。平台支持通过叠加多个许可的时长来延长许可时长。由于设备已有一个许可，在叠加新许可时，只能叠加同类型的许可证。操作步骤如下：

1. 在“**订阅>>设备**”页面，点击操作列的**应用许可**（**设备已经应用许可**）；

2. 在叠加许可弹窗，选择许可证，并点击应用。

:::color3
┃**说明：**

①叠加后，原应用给设备的许可证的时长将延长；

②叠加操作不可逆，许可证叠加后，无法再拆分。

:::

![1722407243808-c2cff8f5-86e9-45a9-872e-c6dc5d382a87.png](./img/ejt7ppogIACxPqAy/1722407243808-c2cff8f5-86e9-45a9-872e-c6dc5d382a87-309314.png)

4、批量应用许可

当设备量较大时，平台支持批量快速的为多个设备应用许可证。操作步骤如下：

1. 在“**订阅>>设备**”页面，点击设备列表上方的批量许可；

2. 选择需要应用的**许可证类型及设备**，点击**批量许可**，平台将自动分配许可；

3. 确认许可结果，并点击**应用**，批量许可生效。

为了保证您账号下的许可得到最大化的利用，平台将优先分配已激活的许可，其他类型许可将不区分时长随机分配。

![1722407761706-b38b51b3-6e8b-410d-a8c8-ba1e19d363bd.png](./img/ejt7ppogIACxPqAy/1722407761706-b38b51b3-6e8b-410d-a8c8-ba1e19d363bd-048354.png)

:::color3
┃说明：如果设备已有在用的同类许可，再次应用后，许可时长将会叠加，此操作不可逆，请谨慎操作。

:::

5、单个取消许可

当某个设备业务用途变更，需要切换其他类型的许可证，或由于设备故障报修，需要把许可证应用到其他设备时，可以取消这个设备的许可证，取消后，被取消的许可证可以应用到其他设备，设备原许可范围内的服务将立即终止。操作步骤如下：

1. 在“**订阅>>设备**”页面，点击目标设备操作列的**取消许可**；

2. 二次确认后，许可将从设备取消应用。

![1722408897693-e42e6dcc-23b2-430a-a66f-45bde6fb0f87.png](./img/ejt7ppogIACxPqAy/1722408897693-e42e6dcc-23b2-430a-a66f-45bde6fb0f87-401282.png)

1. 在“**订阅>>设备**”页面，选择需要取消许可的设备；

2. 点击设备列表上方的**取消许可**，二次确认即可。

### **13.4升级许可证**

当设备绑定的低版本许可证功能不足以满足业务需求时，您可以升级这个设备的许可证。操作步骤如下：

2. 选择需要升级的**许可证类型**。选择后，平台将自动过滤绑定了此许可类型的设备；

3. 选择需要升级许可的**设备**；

4. 选择**预升级到的许可证类型**，点击提交；

5. 确认升级许可的结果，点击确认后，许可升级生效，设备的许可变更为高级版许可。如取消升级许可，则此次升级不生效。

:::color3
**┃说明：** 支持对已激活的同类型许可批量升级。升级后，许可证剩余时长将根据前后两个许可证类型的价格比重进行等比调整。

### 13.7统一许可到期日期

当您需要对齐设备许可服务的到期时间，以便于您更好的管理业务，统一财务对账结算。你可以执行以下操作：

1. 进入“**设备**”或“**许可证**”页面；
2. 选择需要对齐到期时间的许可证；
3. 点击“统一许可到期日期”；
4. 确认统一后的结果信息，点击“确定”，系统将自动将所选的许可证的到期时间统一到同一时间。

![1719487194489-6501dda6-f512-4d99-ba6f-f65a55466be3.png](./img/ejt7ppogIACxPqAy/1719487194489-6501dda6-f512-4d99-ba6f-f65a55466be3-155392.png)

:::color3
**注意：**

①系统会自动把所选的同一类型的许可证的过期日期统一到同一时间。例如：当前为2024-06-01，选择了2个剩余时长为1年、1个剩余时长为60天的基础版许可证，那么平均剩余时长为：（2*365+1*60）/ 3= 264。则这三个许可证的统一后过期日期为：2024-06-01 + 264天 = 2025-02-20。

②“统一许可到期日期”操作不可逆，请谨慎操作。

:::

2. 选择需要续订的许可证，点击**续订许可**;
3. 选择续订时长，点击提交；

## FAQ  

**1. APP支持本地配置哪些设备的联网配置？**

当前只支持本地配置ER805、ER605的联网配置。

**2. 扫码后无法连接设备是什么原因？**

扫描ER805设备铭牌上的二维码后无法正常连接设备通常为以下原因:

1. 设备未开启http访问;

2. 设备2.4G Wi-Fi的接入点功能已关闭;

3. 设备Wi-Fi的连接参数不是出厂设置;

4. Wi-Fi信号较弱，如距离设备较远或设备未连接Wi-Fi天线;

当无法扫码连接设备时请尝试手动连接设备Wi-Fi或使用网线配置设备联网参数。

**3.平台添加设备报错**

在平台添加设备报错，通常为以下原因：

1. 序列号已存在：该设备已被添加到平台，无法重复添加；
2. 设备信息无效：请确认设备的序列号及MAC地址与设备铭牌上的信息一致；

如仍存在问题，请联系技术支持。

**4.设备无法在平台上线**

设备添加到平台后无法上线，请排查：

1. 设备的网络是否正常；
2. 设备本地的云服务是否已启用；
3. 确认设备当前的固件版本是否支持连接云平台；
4. 设备如果通过蜂窝上网，请检查物联网卡的白名单地址是否包含：*.inhandcloud.cn，*.amazonaws.com.cn ；
5. 设备是否绑定了许可证，如未绑定，可进入“**映翰通云服务 >> 订阅**”进行许可证绑定。

**5.设备配置同步挂起**

“挂起”表示云端向设备同步配置的动作被中断。当设备的固件版本与设备所在分组的固件版本不一致时就会出现该状态。一旦变更为挂起状态，云端的任何配置都将不会下发给设备。您可以通过以下两种方式解决：

1. 升级设备/分组的固件版本，保证设备与其所在分组的固件版本保持一致；
2. 直接将设备移出分组。

![1722396851868-46cd45fd-abc4-47bb-8b05-44a035b29590.png](./img/ejt7ppogIACxPqAy/1722396851868-46cd45fd-abc4-47bb-8b05-44a035b29590-554334.png)

**6.设备配置同步失败**

配置失败的原因通常有以下原因：

1. 配置的内容有误；
2. 设备的网络质量不好，导致配置下发失败；

您也可以点击“配置失败”图标，查看配置失败的具体原因。也可点击“清除Pending配置”，以消除配置失败状态，且不会对设备现有配置有任何影响。![1739773702801-ea310ca4-8492-46bc-a0fb-68ec8cf828f3.png](./img/ejt7ppogIACxPqAy/1739773702801-ea310ca4-8492-46bc-a0fb-68ec8cf828f3-777123.png)

![1722396851875-c9e3e16c-8dcf-4eaf-9749-f7496edbf136.png](./img/ejt7ppogIACxPqAy/1722396851875-c9e3e16c-8dcf-4eaf-9749-f7496edbf136-929247.png)

**7.如何对设备进行恢复出厂？**

设备上线后，会重新应用一次平台上存的配置。如需对设备执行恢复出厂，请按以下步骤操作：

1. 确认设备是否存在分组配置，如果存在，需先将设备移出分组或清除分组的配置；
2. 到设备列表/设备详情，清除设备的个性化配置；
3. 选择设备，点击“**指令>>恢复出厂**”。

![1722396851900-3c4c2731-c8bd-44dd-b4dd-c5db6f90cb69.png](./img/ejt7ppogIACxPqAy/1722396851900-3c4c2731-c8bd-44dd-b4dd-c5db6f90cb69-145511.png)

**8.离线的设备是否支持固件远程升级？**

云平台的固件升级不区分在线状态，对离线的设备也能创建升级任务。当设备下次到平台上线后，设备将会收到平台的升级任务执行升级操作。

离线设备下发固件升级任务后，固件升级状态会变更为“等待”，如您不需要本次升级操作，可在任务列表取消。

![1722396852086-db4451b7-cd52-4b94-a989-6f0cf54934a7.png](./img/ejt7ppogIACxPqAy/1722396852086-db4451b7-cd52-4b94-a989-6f0cf54934a7-243040.png)

3.在切换组织弹窗中，选择需要协助的组织，并点击“**切换**”，即可成功切换到对应组织提供技术支持。

1. **在哪里能测速？页面上为什么没有展示测速功能？**

测速功能仅在小星云管家支持，设备本地不支持此功能。如需测速，请在设备列表点击设备名称，进入设备详情的“诊断工具”页面进行操作。

如果“诊断工具”页面未展示“测速”功能模块，请检查：

- 当前产品是否支持测速功能；
- 当前设备的固件版本是否支持此功能；
- 如以上两点都支持，请向您机构的系统管理员确认是否为您开启了测速权限。

1. **如何为其他账号开启测速权限？**

测速权限默认仅“系统管理员”角色的用户拥有。系统管理员可进入映翰通云服务页面，在“用户管理 >> 用户详情”页面为对应账号开放测速权限。![1756367687643-70824562-1f39-49fb-8a03-b8b6d1c878eb.png](./img/ejt7ppogIACxPqAy/1756367687643-70824562-1f39-49fb-8a03-b8b6d1c878eb-307110.png)
