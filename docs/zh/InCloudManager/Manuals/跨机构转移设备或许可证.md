# 跨机构转移设备或许可证

当您需要将自己机构下的设备或者许可证转移到其他机构时，您可以邀请这个机构成为您在映翰通云服务的客户，然后您就可以为这个机构转移资源。详细操作步骤请参考此文档。

该文档也适用于将多个机构下的设备统一放到一个机构管理，或者回收某个机构的设备的场景。

下文将以”机构A”转移设备与许可到”机构B”为例。

## 前置条件

- 机构A账户类型为MSP（请联系客户经理配置）；
- 机构A的操作用户角色为系统管理员。

## 步骤一：邀请客户

1. 机构A的系统管理员登录到[映翰通云服务](https://portal.inhandcloud.cn/)；
2. 在菜单栏选择 **客户**；
3. 点击 **邀请客户**，并填写以下信息：
    1. 自定义的 **客户名称**；
    2. 填写客户“机构B”在映翰通云服务的系统管理员**邮箱账号**；

![1741243578841-5c5425e2-dbed-4855-9ae6-6fe17be85ef8.png](./img/KBQIffbEtAMFTKcf/1741243578841-5c5425e2-dbed-4855-9ae6-6fe17be85ef8-155910.png)

1. 点击 **邀请客户**。

## 步骤二：客户同意邀请

1. 机构B的系统管理员收到**邀请通知邮件**；
2. 点击 **接受邀请**。![1741243765819-b2de8574-bcab-4fd6-98a1-9504c7d9ee09.png](./img/KBQIffbEtAMFTKcf/1741243765819-b2de8574-bcab-4fd6-98a1-9504c7d9ee09-377167.png)![1741243844194-282ac898-e5d9-452c-9aaa-683a9e7699bb.png](./img/KBQIffbEtAMFTKcf/1741243844194-282ac898-e5d9-452c-9aaa-683a9e7699bb-774175.png)

## 步骤三：转移设备或许可证

### 转移设备

1. 机构A的系统管理员登录到[映翰通云服务](https://portal.inhandcloud.cn/)；
2. 在菜单栏选择 **订阅**；
3. 进入 **设备** 菜单；
4. **选择**需要转移给机构B的**设备**，点击 **转移**；![1741244305841-112dba63-f606-46df-bc19-27960768978c.png](./img/KBQIffbEtAMFTKcf/1741244305841-112dba63-f606-46df-bc19-27960768978c-241021.png)
5. 在转移设备弹窗中，**选择客户**：机构B；
6. 点击 **转移**。![1741244205429-f37e0cea-719a-4efa-9247-2ddd7b43a532.png](./img/KBQIffbEtAMFTKcf/1741244205429-f37e0cea-719a-4efa-9247-2ddd7b43a532-614325.png)

:::color3
**注意：** 如果设备有许可证，则许可证将同设备一起转移给客户。如果不需要转移许可证，可以先对设备进行批量 **取消许可** 操作。

:::

### 转移许可证

1. 机构A的系统管理员登录到[映翰通云服务](https://portal.inhandcloud.cn/)；
2. 在菜单栏选择 **订阅**；
3. 进入 **许可证** 菜单；
4. **选择**需要转移给机构B的**许可证**，点击 **转移**；![1741244575825-c7cb84e1-dc70-4687-8f2a-621a5cd405c9.png](./img/KBQIffbEtAMFTKcf/1741244575825-c7cb84e1-dc70-4687-8f2a-621a5cd405c9-620508.png)
5. 在转移设备弹窗中，**选择客户**：机构B；
6. 点击 **转移**。![1741244608737-b2d57b4f-665b-40c6-8394-25b973a3bb93.png](./img/KBQIffbEtAMFTKcf/1741244608737-b2d57b4f-665b-40c6-8394-25b973a3bb93-501688.png)

## 验证结果

机构A转移设备或者许可证后，机构B用户可登录到[映翰通云服务](https://portal.inhandcloud.cn/)看到这些设备或者许可证。
