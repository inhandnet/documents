## 1. 平台简介

### 1.1 什么是 InConnect

InHand Connect Service 云平台（简称 InConnect）是面向企业客户提供的"一站式"联网服务。该平台通过人性化界面和简易流程，搭配映翰通联网设备，可助用户快速连接全球业务网点，实现各地区间设备、信息安全稳定的互联互通。

## 2. 注册登录

### 2.1 注册

使用浏览器访问 InConnect 平台地址：<https://ics.inhandiot.com/>，进入 InConnect 云平台注册页面。（推荐使用 Google Chrome 浏览器）

### 2.2 登录

注册完毕后，使用注册邮箱及密码登录云平台。登录后可在"设置"页面修改密码和绑定手机号码；绑定手机号码后，可使用手机号码登录云平台。

![1718101453827-680087bb-2505-4a3b-b232-07ce8e5ad727.png](./img/Fv9P5NgmDahP7rvI/1718101453827-680087bb-2505-4a3b-b232-07ce8e5ad727-310114.png)

### 2.3 使用映翰通云服务账号登录

映翰通云服务（包含小星云管家、DeviceLive、白鹰能源管家、ISCada、InLink）采用统一账号体系。映翰通还提供独立的 Device Manager 及 InConnect 设备管理平台。通过简单配置，可将现有映翰通云服务账号与 InConnect 关联，实现一键快速登录及平台间无缝切换。

#### 2.3.1 前置条件

- 已有映翰通云服务系统账号
- 已有 InConnect 账号

#### 2.3.2 配置步骤

1. 登录 InConnect 平台
2. 进入 **设置 〉 映翰通云服务** 页面
3. 开启 **允许通过映翰通云服务登入**

![1768804716416-a06dc44c-8b71-4d00-80c9-e8b6450179f6.png](./img/Fv9P5NgmDahP7rvI/1768804716416-a06dc44c-8b71-4d00-80c9-e8b6450179f6-615769.png)

#### 2.3.3 验证结果

**方式一：通过映翰通云服务直接访问 InConnect**

1. 登录 [映翰通云服务](https://portal.inhandcloud.cn/)
2. 进入 **控制台**
3. 点击 **访问** InConnect

![1768804751816-459048e0-6b75-4b92-acb0-ec66bf136293.png](./img/Fv9P5NgmDahP7rvI/1768804751816-459048e0-6b75-4b92-acb0-ec66bf136293-367338.png)

**方式二：在 InConnect 登录页面使用映翰通云服务账号登录**

1. 进入 InConnect 的登录页面
2. 点击 **使用映翰通云服务登录**
3. 输入映翰通云服务的登录账号及密码，点击 **登录**

![1768804777533-c2d303e9-3c7e-46d3-a131-2b1746577eb3.png](./img/Fv9P5NgmDahP7rvI/1768804777533-c2d303e9-3c7e-46d3-a131-2b1746577eb3-483490.png)

## 3. 路由器接入 InConnect

InConnect 平台会在添加路由器后自动下发运行配置来建立安全的网络连接。路由器接入步骤：

1. 平台添加路由器：在平台的"站点"添加路由器
2. 路由器接入配置：在路由器的 Web 界面配置接入 InConnect
3. 检查路由器与平台连接状态

### 3.1 InConnect 添加路由器

在路由器 Web 界面配置接入 InConnect 平台前，需先在 InConnect 平台新增此路由器的信息。

**步骤一：新增路由器**

1. 在 **站点 〉 路由器** 点击 **新增**
2. 在表单中输入：**自定义的路由器名称**、**路由器的序列号**、选择对应的 **路由器机型** 及 **所属网络**
3. 点击 **确认** 保存

![1731313401864-4a7a43bd-8b10-498b-b688-672715a01db2.png](./img/Fv9P5NgmDahP7rvI/1731313401864-4a7a43bd-8b10-498b-b688-672715a01db2-147612.png)

**注意：** 序列号可在路由器铭牌或路由器 Web 管理页面查看；机型需选择正确，否则会导致路由器无法成功建立连接。

**步骤二：确认添加成功**

在"路由器"列中出现该设备即表示添加成功。InConnect 会为添加成功的路由器自动分配一个虚拟 IP 地址；InConnect 客户端软件安装配置完毕后，可直接根据此虚拟 IP 访问远端路由器。

![1718101453858-902a11dc-101b-49c4-8244-5db80b4dd06a.png](./img/Fv9P5NgmDahP7rvI/1718101453858-902a11dc-101b-49c4-8244-5db80b4dd06a-721389.png)

### 3.2 配置路由器接入 InConnect

在路由器 Web 界面配置接入 InConnect 平台，配置前需确保路由器已联网。不同产品系列的配置方式会有差异，请对照设备参考同系列产品进行配置。各设备因固件版本不同，界面可能存在差异，以实际设备为准。

#### 3.2.1 IR300 产品系列

1. 路由器和 PC 连接后，使用浏览器访问 **192.168.2.1**
2. 在登录页面输入初始登录信息
3. 登录到设备 Web 页面后，进入 **服务 〉 设备远程管理平台**：
   - 选择"服务类型"为 **InConnect Service**
   - 选择"服务器"为 **ics.inhandiot.com**
   - 在"注册账户"输入 InConnect 的注册邮箱账号（无账号可点击"注册/登录"到平台注册后再录入）
4. 点击 **应用** 后，在 **状态 〉 远程管理平台** 查看状态为 **已连接** 表示设备已正常接入

## 4. 客户端软件使用

云平台会自动为接入的路由器建立安全连接通道。用户在本地环境（PC 或手机）安装并配置客户端接入安全连接通道后，即可用虚拟 IP 远程访问路由器和路由器下端设备。

### 4.1 Windows

**步骤一：下载并安装 OpenVPN 客户端**

1. 在 InConnect 平台下载 OpenVPN 客户端并安装
2. 在 InConnect 云平台的"用户"菜单 **下载配置文件**

![1718101455763-fdbcfbd5-5424-4bb9-ae33-920642d3ca82.png](./img/Fv9P5NgmDahP7rvI/1718101455763-fdbcfbd5-5424-4bb9-ae33-920642d3ca82-278918.png)

**步骤二：导入配置文件**

以管理员身份运行 OpenVPN GUI 快捷方式，在桌面右下角找到对应图标右击，点击 **导入配置文件**，导入配置文件到 OpenVPN 客户端。

## 5. 连接路由器下端设备

将路由器下端设备添加到 InConnect 平台，可实现用户对远端设备的访问与控制。InConnect 平台会在添加下端设备时自动下发运行配置来建立安全的网络连接。

InConnect 支持两种访问方式：虚拟 IP 访问和真实 IP 访问。

### 5.1 虚拟 IP 访问

平台通过自动映射机制，为加入的 PLC 等设备自动映射虚拟 IP，可自动避免访问冲突。当使用场景对实 IP 管理和使用的要求不高时，建议使用此方式。虚拟 IP 访问为系统默认支持的访问方式。

**操作方式一：**

1. 在"站点"页面，点击 **终端 〉 新增**
2. 输入下端设备 **名称**、真实的 **IP 地址** 并搜索选择 **所属站点**（对应站点中添加的设备名称）
3. 点击 **提交** 按钮并查看。平台会自动为下端设备映射一个虚拟 IP 地址，单台路由器下最多可添加 254 个下端设备

![1718101457263-d0f230f6-ff86-476e-8f57-917be534b8c9.png](./img/Fv9P5NgmDahP7rvI/1718101457263-d0f230f6-ff86-476e-8f57-917be534b8c9-416233.png)

![1718101457267-03ef0c20-9c51-4046-8e41-0841dac240cb.png](./img/Fv9P5NgmDahP7rvI/1718101457267-03ef0c20-9c51-4046-8e41-0841dac240cb-641388.png)

**操作方式二：**

1. 在站点页面点击需要添加下端设备的路由器名称
2. 进入路由器管理界面
3. 选择 **终端 〉 新增**，输入相应信息，点击 **提交** 按钮
4. 查看平台自动为下端设备映射的虚拟 IP 地址

![1718101457393-76e765ed-aecc-41ca-99a5-088905bf247e.png](./img/Fv9P5NgmDahP7rvI/1718101457393-76e765ed-aecc-41ca-99a5-088905bf247e-738040.png)

### 5.2 真实 IP 访问

5. 测试网络连接（虚拟 IP 访问和实 IP 访问均可）

## 6. 网络管理

### 6.1 对等网络配置

对等网络常应用于面向远程维护 PLC、HDMI、摄像头等设备连接。

初始所有新加入的路由器都在"默认网络"里，"默认网络"为一个对等网络。对等网络里的路由器都可以开启实 IP 访问，开启时会检测 IP 冲突。若存在相同的 IP，需新建一个对等网络来配置路由器的实 IP。用户也可通过"开启/关闭"实 IP 来避免冲突，达到实 IP 的按需访问。"网络"模块也可实现网络访问的权限控制。

#### 6.1.1 添加对等网络

1. 选择"网络"点击 **+** 号，添加网络
2. 输入网络名称，网络类型勾选 **对等网络**
3. 设置 **设备互访** 开关，点击 **提交** 按钮

**设备互访开关说明：** 当需要网络内的路由器/终端设备互相访问时，可开启设备互访功能；出于安全要求，也可选择关闭。关闭后，该网络中的路由器之间、路由器与另一路由器的终端之间将不能互访，但网络中的用户仍可访问该网络中的所有路由器和终端。默认网络中的路由器和终端默认可互访，暂不支持修改。

![1731313607492-80116276-4b58-4c4c-9f57-198bbb0a0e53.png](./img/Fv9P5NgmDahP7rvI/1731313607492-80116276-4b58-4c4c-9f57-198bbb0a0e53-956104.png)

4. 选择相应网络管理人员及需要管理的站点，点击 **提交** 按钮

![1718101458785-eb4f768e-0218-485e-ba34-995caca481cf.png](./img/Fv9P5NgmDahP7rvI/1718101458785-eb4f768e-0218-485e-ba34-995caca481cf-462782.png)

**注意：** "对等网络"隔离了网络访问，即仅本网络内的用户和站点设备之间可相互访问；本网络内的用户无法访问网络外的站点设备，网络外的用户也无法访问本网络内的站点设备。"默认网络"为一个对等网络，初始新加入的路由器站点和用户都在默认网络内。

#### 6.1.2 对等网络添加站点/用户

1. 选中相关对等网络，点击 **添加成员** 按钮
2. 勾选相应的 **用户名、站点名称**，点击 **提交** 按钮。若开启实 IP 的路由器存在 IP 冲突，不能添加到此对等网络中
3. 选择"站点"可查看对等网络下的所有站点
4. 选择"用户"可查看对等网络下的所有用户

**注意：** 开启实 IP 的路由器若存在 IP 冲突，不能添加到对等网络中。

#### 6.1.3 移动站点/用户

- **移动站点：** 网络管理页面中选择 **站点 〉 移动站点**，选择期望移动到的网络并确认
- **移动用户：** 选择 **用户 〉 移动用户**，选择期望移动到的网络并确认

**注意：** 开启实 IP 访问的路由器若存在 IP 冲突，则不能从网络中移出。当机构存在多个 VPN Server 时，移动用户到其他网络后，可能需要重新下载和使用新的配置文件。

#### 6.1.4 删除对等网络

1. 删除对等网络前，需先删除对等网络中的所有用户及站点
2. 点击"删除"，二次确认后，网络删除成功

![1731315201201-75bab277-ea1e-46c7-ad14-00ca79167782.png](./img/Fv9P5NgmDahP7rvI/1731315201201-75bab277-ea1e-46c7-ad14-00ca79167782-822018.png)

### 6.2 星型网络配置

星型网络应用于远程办公，使用户在家或外地出差时能通过 InConnect 通道访问公司内网的服务器资源。

**注意：** 星型网络仅适用于远程办公场景，若需远程访问现场的 PLC 等设备，请选择"对等网络"。

#### 6.2.1 新增星型网络

1. 选择 **网络 〉 +**，输入网络名称，选择 **星型网络**，选择相应的中心设备，输入 **中心 IP 地址**，点击"提交"按钮

5. 客户端通过 OpenVPN 访问路由器 WAN 口的网段（需开启 OpenVPN 连接）

#### 6.2.2 星型网络添加站点/用户

2. 勾选要增加的 **用户、站点**，点击"提交"按钮
3. 查看添加的网络信息

#### 6.2.3 删除星型网络

1. 选择"网络"，点击网络管理中对应的网络名称
2. 选择 **站点 〉 移除站点**
3. 选择对应的网络名称，点击 **删除** 按钮，删除网络

## 8. 路由器/网关管理

### 8.1 路由器列表

路由器列表支持用户自定义列字段，用于给路由器设备打标签（如：所属客户等）。

1. 点击列表右上方的 **自定义字段**

添加、导入、编辑设备时可为设备备注自定义信息，也支持使用这些自定义列字段进行查询。

2. 用户可对在线的路由器进行远程 Web 管理、远程配置、强制下线和升级固件操作

### 8.3 远程 Web 管理

通过 InConnect 平台远程访问路由器 Web 管理界面；点击图标登录路由器 Web 管理页面，可实现对路由器的远程管理。

### 8.4 连接配置下发

InConnect 平台会在添加路由器和下端设备时 **自动下发运行配置** 来建立安全的网络连接。当路由器在线但 VPN 状态为登出时，可点击运行配置下发按钮来重新建立连接。

### 8.5 远程更新路由器配置

此功能用于对平台接入的路由器批量更新配置。

**步骤一：新增配置模板**

1. 点击 **配置管理 〉 +**，选择配置文件或输入配置内容
2. 可在路由器本地管理界面下载原始配置文件，根据需要做部分修改后再上传到 InConnect 平台来批量更新路由器配置

**步骤二：选择路由器**

选择需要下发配置的路由器，即可更新配置。在线的设备收到配置指令会立即更新配置；离线的设备会在恢复在线时更新配置。

**步骤三：查看进度**

在配置详情页可查看应用了该配置的所有路由器的更新进度及状态。在配置列表可查看、删除配置，批量更新配置。

### 8.6 远程升级路由器固件

**步骤一：新增固件**

点击 **固件升级 〉 +**，选择用于升级的固件，点击 **确定**，上传固件到云平台。

**步骤二：选择路由器**

在设备列表勾选需要升级固件的路由器，点击"确定"按钮，即对设备进行更新。在线的设备在收到升级指令时会立即执行升级任务；离线的设备当恢复在线时会执行升级任务。

**步骤三：查看进度**

在固件详情列表可查看应用了该固件的路由器的升级进度及状态。在固件列表可 **查看、删除固件，批量升级固件**。

### 8.7 任务

查看平台对路由器的任务下发情况，可对这些任务进行 **重新下发和取消** 操作。

## 10. 结算

InConnect 平台对所有客户端和用户之间安全连接产生的流量进行计费。可在"**结算**"页面为 InConnect 提供的联网服务进行充值，支付前需要同意《支付协议》。

**注意：** 为保证服务的正常连接，平台提供了 10GB 的流量可供预使用。当信用额度内的流量使用完毕且仍未进行充值时，平台连接服务将被中止；充值补足信用额度后，服务会立刻恢复。

充值完毕后，可在"**充值记录**"里查看充值信息，或下载收据查看充值订单详情。平台根据使用情况会定时在"消费记录"里更新使用日志。每月 1 日会通过邮件的方式发送月账单，记录上月流量和资费的使用情况。

为避免欠费导致业务受到影响，可配置 **流量阈值提醒**。

## 11. 服务器证书

当在 InConnect 平台注册了一个机构时，系统会为该机构签发一个有效期为 30 年的证书，以保证能使用平台的 OpenVPN 功能。

当证书即将过期时，会收到来自系统的证书更新提醒。可按以下步骤操作更新：

1. 登录 InConnect 平台，点击"服务证书过期提醒"弹窗的"更新证书"或上方功能菜单栏的"证书管理"
2. 选择所有用户并点击列表右上方的"批量更新证书"按钮，批量更新用户证书
3. 点击"更新服务器证书并部署"并等待部署完成。部署和更新时间一般为 5～30 分钟

**注意：** 更新用户证书后，需更新和部署服务器证书，才能使此页面下载的新的 OpenVPN 配置文件生效。更新服务器证书前，原来的 OpenVPN 配置文件仍然可用。更新完成后，请重新下载和使用用户的配置文件；设备的 VPN 相关配置，系统将自动下发。

## 12. 常见问题

**Q1：一个 InConnect 注册账号可以添加多个用户账号和路由器？**

最大支持 4000 个用户账号和路由器设备，单个路由器下最大支持接入 254 个下端设备。若设备无法正常添加或无法正常分配虚拟 IP，请邮件联系：ics-service@inhand.com.cn。

**Q2：InConnect 平台的计费模式？**

按照平台内的路由器、终端及用户间使用 VPN 通道通信产生的数据流量收费，费用标准 5 元/GB。

**Q3：一个路由器在不产生任何业务数据，并且网络正常情况下，每月最低的流量是多少？**

大概 30～40MB。

**Q4：云平台部署在什么地方？美国亚马逊和中国亚马逊？**

国内服务部署在中国亚马逊，国外服务部署在美国亚马逊和德国。

**Q5：服务器证书是什么？怎么更新？**

详见"11. 服务器证书"章节。

**Q6：服务器证书更新后，设备的 VPN 一直连接不上。**

请先下载设备本地的完整运行日志，将日志发给技术支持工程师。然后可在平台上手动下发一次配置，若配置仍无法成功下发，请联系技术支持。

**Q7：设备详情页面看不到开启实 IP 功能。**

实 IP 的开与关会影响同一网络的其他设备的 VPN 连接。为避免误操作，实 IP 功能默认不展示。可在 **系统管理 〉 公司信息 〉 实IP功能** 开启，然后即可在设备详情配置实 IP 信息。

**Q8：机构的用户和设备的 VPN 功能都无法使用。**

请检查：是否欠费；VPN 功能是否开启。系统检测到长时间未使用 VPN 功能后，会暂停 VPN 功能。可在 **系统管理 〉 公司信息 〉 VPN功能** 重新启用。

**Q9：有没有账号即将欠费的通知？**

当账号欠费后，系统会邮件告知。平台提供 10GB 的信用额度。也可在 **系统管理 〉 结算** 页面配置流量阈值提醒，以便流量监测与充值。

**Q10：证书更新成功后，为什么电脑上的 VPN 客户端一直连接失败？**

证书更新后，更新前的 VPN 配置文件已经失效。需要在用户列表下载新的 VPN 配置文件，并重新导入 VPN 客户端才能正常使用。

**Q11：使用映翰通云服务账号登录的常见问题**

- **（1）在映翰通云服务访问 InConnect 失败：** 请检查映翰通云服务的邮箱账号是否在 InConnect 注册过；InConnect 平台是否已开启"允许通过映翰通云服务登入"。
- **（2）管理员已开启"允许通过映翰通云服务登入"，我的账号能直接通过映翰通云服务登录 InConnect 吗？** 不可以。开启"允许通过映翰通云服务登入"属于个人账号维度的设置，需要每个账号单独配置。
- **（3）InConnect 的设备数据会同步到映翰通云服务吗？** 不会。InConnect 与映翰通云服务的设备数据互相隔离。
- **（4）映翰通云服务的用户账号会同步到 InConnect 平台吗？** 不会。InConnect 与映翰通云服务的用户账号互相隔离。若需该账号能访问 InConnect，需到 InConnect 的用户列表进行添加。若需经常同时在这两个平台添加用户，可反馈至：incloud@inhand.com.cn

## 附录 路由器联网

### 方式一：拨号上网 / SIM 卡上网

以 IR915 为例，其他设备的联网方式请查看硬件使用手册或访问映翰通官网：<https://www.inhand.com.cn/>

1. 将 SIM 卡插入卡槽 1，拧紧 4G LTE 天线到 ANT 接线柱

**注意：** 插拔 SIM 卡操作时必须拔掉电源，以免造成数据丢失或设备损坏。

2. 打开网页，登录设备（参见方式二：以太网上网）
3. 点击 **网络 〉 拨号接口 〉 启用**，待网络连接状态显示为"已连接"并显示已分配 IP 地址时，说明 SIM 卡已成功上网。若是专网卡还需填写"拨号参数集"
4. 当设备插入双 SIM 卡时，需勾选"**启用双选 SIM 卡**"

### 方式二：以太网

以 IR915 为例。不使用 SIM 方式上网时，请禁用"**拨号接口**"，否则设备会循环拨号干扰网络。

1. LAN（FE 1/1 或 GE1/1）口连接 PC，WAN 口连接外网
2. 设置 PC 与路由器设备的 IP 地址在同一网段：方法一为自动获取 IP 地址（推荐）；方法二为使用固定 IP 地址，设置 PC 和路由器 EF/GF 口在同一网段。路由器初始 IP 地址为 192.168.2.1，子网掩码均为 255.255.255.0
3. 在浏览器地址栏输入 **192.168.2.1**，进入网关 Web 登录页面
4. 在登录页面输入初始登录信息
5. 点击 **网络 〉 新建 WAN**，给 WAN 口配置 IP 地址，使路由器接入 Internet
6. 推荐使用"**动态地址（DHCP）**"，若选择"静态 IP"，在手动配好网络参数后保存即可
7. 点击 **工具 〉 PING 探测**，在"主机"中填入国内常用网址，测试路由器是否成功接入互联网

### 方式三：Wi-Fi

以 IR915 为例。不使用 SIM 方式上网时，请禁用"拨号接口"。

1. 用网线使 PC 和网关 LAN 口连接，Wi-Fi 天线与 WLAN1/2 口连接
2. 设置 PC 的 IP 地址与路由器在同一网段（参见方式二）
3. 登录设备 Web 管理页面（参见方式二）
4. 点击 **网络 〉 WLAN 接口**，接口类型选择客户端，填写 Wi-Fi 网络参数，点击应用并保存。随后进入"状态"栏，可查看到网络连接状态为已连接
5. 点击 **快速向导 〉 新建 WLAN** 配置参数
6. 点击 **防火墙 〉 网络地址转换（NAT）**，若出现"dot11radio 1"连接，表示 Wi-Fi 连接成功
