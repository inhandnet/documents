## 第1章 认识设备

### 概述

VT200系列车辆跟踪网关是一款面向物流运输、工程车辆监控等行业的资产跟踪产品。该设备集成GNSS（Global Navigation Satellite System，可以理解为：全球卫星导航系统，用于精确定位设备所在位置）定位功能，支持车辆及驾驶员状态的实时跟踪与监视、历史轨迹回放、地理围栏（可以理解为：在电子地图上划定的虚拟边界，当设备进出该区域时可触发告警）以及异常警报等功能。VT200结合车载网络云平台，可实现远程车辆管理、资产跟踪和预防性维护，有助于车队运营降低成本、提高效率。该设备提供多种子型号，支持LTE CatM1、Cat1、Cat4等不同速率的无线网络接入。

#### 接口说明

| 接口 | 功能说明 |
| :--- | :--- |
| RS232串行端口 | 用于数据传输，不用于设备配置。配置设备需使用USB Type-C接口。 |
| USB Type-C | 位于设备侧边（橡胶塞下方），用于连接计算机配置设备，功能等同于RS232串行端口。 |
| DI（Digital Input，数字输入，可以理解为：检测开关状态的"传感器"，判断是否有信号输入） | 用于检测开关值（如按钮按下/弹起、开关开/关状态）。VT200提供可配置的上拉功能，DI默认含10kΩ下拉至GND的电阻；配置为上拉时，含20kΩ上拉至电源电压的电阻。 |
| DO（Digital Output，数字输出，可以理解为：控制开关的"遥控器"，用于控制外部设备的开关状态） | 用于输出直流电压，为开漏输出，支持300mA电流，通常与继电器配合使用。 |
| AI（Analog Input，模拟输入，可以理解为：测量连续变化的"仪表"，如温度、压力等模拟量信号） | 用于检测直流电压，可直接接入电压模拟量。 |
| 1-Wire | 用于连接小型通信设备（如DS18B20数字温度计、iButton设备）。需将1-Wire设备的DQ引脚连接至VT200 PIN13，VDD和GND引脚连接至VT200的GND。 |
| IGT（点火感应） | 用于连接车辆点火开关，VT200可检测车辆是否点火。测试时需将IGT线缆和V+线缆连接至直流电源。 |

### 指示灯说明

VT200设备配有GNSS状态灯（绿色）和Cellular状态灯（橙黄色），各指示灯状态含义如下：

| 指示灯 | 状态 | 含义 |
|--------|------|------|
| GNSS状态灯（绿色） | 长期熄灭 | 设备未启动或GNSS模组工作不正常 |
|  | 快速闪烁（频率: 0.5Hz） | GNSS对时服务成功 |
|  | 慢速闪烁（频率: 1Hz） | GNSS功能已启用 |
|  | 常亮 | 定位成功 |
| Cellular状态灯（橙黄色） | 长期熄灭 | 设备被禁用或拨号模组工作不正常 |
|  | 快速闪烁（频率: 0.5Hz） | 拨号成功 |
|  | 慢速闪烁（频率: 1Hz） | 拨号已启用 |

### 恢复出厂设置

由于设备配置中往往包含设备证书文件，通过硬件方式恢复出厂设置时，仅恢复用户名和密码为默认值（admin/123456），其余配置不受影响。

操作步骤如下：

1. 找到设备上的Reset按键。
2. 使用螺丝刀或其他尖细工具按住Reset按键，持续超过8秒。
3. 松开按键，设备将恢复默认用户名和密码。

## 第2章 安装与首次使用

### 2.1 安装前准备

- **环境要求**：为避免运输过程中消耗电池电量，设备在出厂状态下处于运输模式。VT200 需通过外部电源或车辆诊断接口进行激活。
- **工具与配件准备**：
  - VT200 主设备
  - 20PIN 电缆或测试线缆（如 SCAB000381）
  - 电源适配器（如需办公室测试）
  - Micro-SIM 卡
  - 4G 与 GNSS 外部天线（若为外部天线型号）

### 2.2 安装与启动指南

1. 将 20PIN 母头（P1）插入 VT200 设备接口。
2. 将 PIN20 `CONN-x-v-` 和 PIN10 `CONN-x-v+` 分别连接到电源适配器的负极和正极。将 PIN9 `CONN-X-IGT` 与 `V+` 共同连接到电源的正极侧。

<p align="center"><img src="./img/wITXdMzOj-UCDayd/1690879897831-efa428b6-d806-428a-8e1a-9ba9fab56137-663739.webp" alt="电源连接"></p>

<p align="center"><strong>图 2-1 电源与接口线缆连接</strong></p>

3. 通过 USB-Type C 接口连接计算机进行调试与配置。需事先下载 VTConfig Tool 配置工具，并通过 USB-Type C 线缆连接计算机与 VT200 设备。

<p align="center"><img src="./img/wITXdMzOj-UCDayd/1677481412377-95c691a9-f98e-468b-b95e-2a9c657fa5ca-660511.webp" alt="USB-Type C接口位置"></p>

<p align="center"><strong>图 2-3 USB 线缆连接计算机</strong></p>

4. 插入 Micro-SIM 卡，需确保 Micro-SIM 卡的截止角指向插槽内侧。

<p align="center"><img src="./img/wITXdMzOj-UCDayd/1702957498478-98859d30-7065-45e4-905a-89df1182c676-494954.webp" alt="安装SIM卡"></p>

6. 配置及连线完成后，将设备顶盖与底盖重新连接复位。

#### 2.4.2 移动网络参数

移动网络参数页面用于查看无线网络链接状态。未插入 SIM 卡时显示为默认参数。当设备通过 SIM 卡拨号联网成功后，可获得 IP 地址并进行数据传输。

| 参数名称 | 参数说明 |
| --- | --- |
| 信号值 | 代表无线网络的信号强度，取值范围 0～31 |

### 场景1：休眠模式配置

**目标**：配置设备休眠模式，使车辆熄火后设备自动进入低功耗休眠状态，延长电池续航，并支持定时唤醒上报数据。

**前提**：设备已上电并完成基本网络配置，已通过配置工具登录设备。

**预计用时**：约5分钟。

#### 工作原理

休眠模式通过状态机控制设备在三种运行状态之间切换，保障车辆熄火后的电池续航能力。状态机如下：

<p align="center"><img src="./img/wITXdMzOj-UCDayd/1681197221579-b4cbfbf8-7105-4980-83d2-568ec6168e1f-663786.webp" alt="休眠模式状态机示意图"></p>

<p align="center"><strong>图 3-1 休眠模式状态机</strong></p>

#### 状态机说明

设备包含三种运行状态：**Run**（正常运行）、**Sleep**（休眠）和 **Temp run**（临时运行）。各状态之间的转换条件如下：

1. **Run → Temp run（转换 ①）**：当满足以下任一条件时，设备从正常运行状态进入临时运行状态：
   - 外接电源断开（电源电压低于休眠电压阈值）；
   - IGT 信号变为 OFF（需在配置中使能 IGT 功能）。
   首次进入休眠流程时，设备默认保持 10 秒的延时休眠时间（用于上报信息），随后进入 Sleep 状态。

2. **Temp run → Sleep（转换 ②）**：当满足以下任一条件时，设备从临时运行状态进入休眠状态：
   - 设备在 Temp run 状态下运行满一个"唤醒运行时间"周期；
   - 设备从 Run 状态经过 15 秒 Temp run 后自动转入。

3. **Sleep → Run（转换 ③）**：当同时满足以下条件时，设备从休眠状态恢复为正常运行状态：
   - 外接电源恢复（电源电压高于休眠电压阈值）；
   - IGT 信号变为 ON（需在配置中使能 IGT 功能）。

4. **Sleep → Temp run（转换 ④）**：设备在 Sleep 状态下运行满一个"唤醒间隔"周期后，自动进入 Temp run 状态（用于定时唤醒上报数据）。

5. **Temp run → Run（转换 ⑤）**：当同时满足以下条件时，设备从临时运行状态恢复为正常运行状态：
   - 外接电源恢复（电源电压高于休眠电压阈值）；
   - IGT 信号变为 ON（需在配置中使能 IGT 功能）。

#### 操作步骤

1. 登录配置工具，进入【系统设置】→【休眠模式】页面。
2. 根据实际需求配置以下参数：

| 参数 | 说明 |
| :--- | :--- |
| 使能IGT | 使能后，设备将 IGT 状态作为进入和退出休眠的判断条件之一。默认不勾选。 |
| 唤醒间隔 | 设备进入休眠后，自动唤醒的时间间隔。默认值为 120 分钟。 |
| 唤醒运行时间 | 设备唤醒后，维持运行状态的持续时间（超过该时间后重新进入休眠）。默认值为 5 分钟。 |
| 延时休眠时间 | 设备进入休眠前，用于上报数据的等待时间。 |

3. 配置完成后，点击"保存配置"使设置生效。

### 场景4：车载诊断接口配置

| 参数 | 说明 | 备注 |
| :--- | :--- | :--- |
| J1939/J1979 | CAN2 接口协议，对应物理层 PIN CAN_2L (PIN 12) 和 CAN_2H (PIN 25) | OBD 默认配置 |
| J1939 | CAN2 接口协议，对应物理层 PIN CAN_2L (PIN 12) 和 CAN_2H (PIN 25) | |
| J1979 | CAN2 接口协议，对应物理层 PIN CAN_2L (PIN 12) 和 CAN_2H (PIN 25) | |
| J1708 | J1708 接口协议，对应物理层 PIN J1708_B (PIN 13) 和 J1708_A (PIN 26) | |
| Auto | 自动轮询模式。设备依次发送上述协议的数据包进行连接测试，收到有效响应后自动锁定该协议进行通讯。 | 使用 Auto 模式时，CAN1 接口和 J1708 接口将同时被打开。 |
| Disable | 关闭 CAN2 接口和 J1708 接口 | |

**预计用时**：约5分钟。

### 4.2 日志获取（CLI模式）

VT200 支持通过 CLI（命令行接口）模式获取设备运行日志，便于故障排查与状态分析。用户需通过 USB 转串口配置线将设备与电脑相连，并借助串口调试软件完成操作。

#### 4.2.1 连接设备与配置终端

使用 USB 转串口配置线将电脑与 VT200 连接。在电脑上打开一款串口调试软件（例如可从 Microsoft Store 下载的串口调试工具）。

<p align="center"><img src="./img/wITXdMzOj-UCDayd/1681197251874-59abe9ea-bc9a-4cc9-86cd-d808d0848a4e-331822.webp" alt="串口调试软件"></p>

1. 打开串口调试软件，选择对应的串口端口。串口默认参数为 **115200/8/N/1**。
2. 将字符编码模式（Character encoding）设置为 **ASCII**，断行模式（Line break）设置为 **\n(LF)**。
3. 点击 **Open serial port** 打开串口连接。

### 车载诊断协议介绍

#### J1939 协议

J1939 是专门针对重型车辆（卡车和客车）开发的一种 SAE 规范。该协议旨在推广用于带微控制器的模块之间通信的标准，以更经济的方式在设备之间进行数据传送。J1939 主要定义了物理层，常见的运行在其上的高层协议包括 SAE J1587（TMC/SAE 联合的供电子数据交换的微机系统在重型车）和 SAE J1922（用于中型和重型公路用柴油车电控传动接口）。

#### J1708 协议

J1708 是专门针对重型车辆（卡车和客车）开发的一种 SAE 规范。该协议旨在推广用于带微控制器的模块之间通信的标准，以更经济的方式在设备之间进行数据传送。J1708 主要定义了物理层，常见的运行在 J1708 之上的高层协议包括 SAE J1587（TMC/SAE 联合的供电子数据交换的微机系统在重型车）和 SAE J1922（用于中型和重型公路用柴油车电控传动接口）。
