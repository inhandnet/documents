## 第一章产品介绍和准备

## 1.导言

VT200系列车辆跟踪网关是一款具有成本效益、接口丰富、性能强的资产跟踪产品。适用于物流运输、工程车辆监控等行业。它提供了与GNSS的精确定位，跟踪和监视车辆和驾驶员的状态，历史跟踪，地理围栏，异常警报等功能，并结合了车载网络云平台，可以实现远程车辆管理，资产跟踪，预防性维护，帮助车队运营商节省成本，提高效率。该设备提供支持各种速度的无线网络访问的子型号，例如LTE CatM1，Cat1，Cat4等。

### RS232 Serial Port

VT200的RS232串行端口用于仅数据传输，不用于配置设备。配置设备需要USB-Type C接口。连接VT200的USB类型C可以像RS232串行端口一样配置设备。在设备侧边打开橡胶塞，可以看到USB-Type C接口。

### 数字输入 (DI)

DI可以检测开关值，例如按钮是否被按下或反弹，以及开关是否打开或关闭。VT200提供可配置的上拉。DI有一个默认的10k&Omega; 电阻下拉到GND。当DI被配置为上拉时，有一个20k&Omega; 电阻上拉至电源电压。使用DI时，有必要区分上拉和无上拉。

### 1-Wire

1-Wire类型设备通常用于小型通信设备，例如数字温度计和iButton设备。使用前，客户需要将1线设备的DQ引脚 (信号线) 连接到VT200 PIN13，并将1线设备的VDD和GND引脚连接到vt200的GND。传感器是DS18B20型。下图为32数字式温度传感器探头的水温检测线。

### 点火信号

IGT (点火感应): IGT用于连接车辆的点火开关。VT200可以检测连接的车辆是否点火。使用20针电缆进行测试时，将IGT电缆和V + 电缆连接到DC电源。

## 启动VT200

客户按照上述步骤完成安装后，即可启动设备进行调试。可以通过状态指示器告知设备的状况。为了避免在运输过程中消耗电池电量，该设备在出厂状态下处于运输模式。VT200需要通过外部电源或车辆诊断接口激活。

1. 将P1的20PIN母头插入VT200；
2. 将PIN20 CONN-x-v-和PIN10 CONN-x-v + 分别连接到电源适配器的负极和正极。PIN9 CONN-X-IGT和V + 都连接到电源的正极侧；

1. 连接USB-Type C进行调试和配置vt200。下载配置工具，用USB-Type C连接计算机和VT200。第2章 登录和设备配置” 章节进行配置。

## 3.系统设置

### 3.1 休眠模式

休眠模式保证熄火后保持续航，为特殊环境提供持续保障。状态机如下：

![1681197221579-b4cbfbf8-7105-4980-83d2-568ec6168e1f.png](./img/wITXdMzOj-UCDayd/1681197221579-b4cbfbf8-7105-4980-83d2-568ec6168e1f-663786.png)

状态机说明：

Run、Sleep、Temp run分别代表正常运行状态、休眠状态和临时运行状态。

① 对应状态机，从Run进入Temp run条件为没有外接电源（电源电压小于休眠电压）或者IGT OFF（需在配置中使能IGT），第一次进入休眠时，设备会默认持续10s的延时休眠时间（用于上报信息）后进入Sleep；

②对应状态机，从Temp run进入Sleep条件为设备在Temp run运行一个唤醒运行时间周期后或者设备从Run运行15sTemp run后，会进入Sleep；

③对应状态机，从Sleep进入Run条件为外接电源（电源电压大于休眠电压）和IGT ON（需在配置中使能IGT）；

④对应状态机，从Sleep进入Temp run条件为设备在Sleep运行一个唤醒间隔周期后，会进入Temp run；

⑤对应状态机，从Temp run进入Run条件为外接电源（电源电压大于休眠电压）和IGT ON（需在配置中使能IGT）；

配置休眠模式：

| 参数 | 参数说明 |
| :--- | :--- |
| 使能IGT | 使能IGT后，设备会将IGT状态作为进入、退出休眠条件，默认不勾选 |
| 唤醒间隔 | 进入休眠后，设备自动唤醒的时间间隔，默认120分钟 |
| 唤醒运行时间 | 休眠唤醒后，设备进入下一次休眠的时间间隔，默认5分钟 |
| 延时休眠时间 | 进入休眠后，用于上报数据的等待时间 |

## 4. 配置蜂窝网络

点击“蜂窝网络”进入配置页面，通常情况下客户仅需要配置“网络接入点名称（APN）”、“网络拨号用户名”、“网络拨号密码”以及“认证模式”并点击“保存配置”，设备重启后生效。

如果客户试用场景特殊，可以点击“显示高级选型”展开隐藏配置项。根据需求配置“网络拨号号码”、“PIN”以及“默认承载APN”。如下图所示：

![1681197223242-1ce5ae9e-e286-4c1d-a89b-39e44efdd02a.png](./img/wITXdMzOj-UCDayd/1681197223242-1ce5ae9e-e286-4c1d-a89b-39e44efdd02a-880472.png)

| :--- | :--- |
| 网络接入点名称（APN） | APN专网接入移动网络时需要运营商提供此参数。大部分公网业务的SIM卡拨号时不对APN进行认证。 |
| 网络拨号用户名 | 默认参数“gprs”，专网AAA认证时，移动网络运营商需要提供此参数。 |
| 网络拨号密码 | 默认参数“gprs”，专网AAA认证时，移动网络运营商需要提供此参数。 |
| 认证模式 | 自动/CHAP/PAP，专网AAA认证时，运营商需要提供此参数。自动：轮流使用 PAP 和 CHAP 认证方式来拨号(上电第 一次使用 pap 认证，如果拨号失败，再次拨号时就使用 chap 认证，再下一次拨号又使用 pap 认证，以此类推)。如果认证模式不选择“自动”，而是选择 PAP 或 CHAP， 那就仅仅使用 PAP 或 CHAP 认证方式来拨号。 |

### 6.6 MQTT 协议平台链接配置

MQTT 遥测传输 (MQTT) 是轻量级基于代理的发布/订阅的消息传输协议，设计思想是开放、简单、轻量、易于实现。这些特点使它适用于受限环境。例如，但不仅限于此：网络代价昂贵，带宽低、不可靠。在嵌入设备中运行，处理器和内存资源有限。该协议的特点有：使用发布/订阅消息模式，提供一对多的消息发布，解除应用程序耦合。对负载内容屏蔽的消息传输，使用 TCP/IP 提供网络连接。支持MQTT的开源软件ThingsBoard、EMQ等，客户可以基于这些开源软件开发物联网平台。

### 2. J1939 协议介绍

J1708是专门针对重型车辆（卡车和客车）开发的一种SAE规范。协议旨在推广用于带微控制器的模块之间通信的标准。标准规范以更经济的方式在设备之间进行数据传送。J1708主要定义了物理层，常见的运行在J1708之上的高层协议是SAE J1587（TMC/SAE 联合的供电子数据交换的微机系统在重型车）和SAE J1922（用于中型和重型公路用柴油车电控传动接口）。
