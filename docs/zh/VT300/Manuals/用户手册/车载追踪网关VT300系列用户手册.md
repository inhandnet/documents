# 第1章 认识设备

## 1.1 概述

VT300系列车载追踪网关是一款面向物流运输、工程车辆监控等行业的资产追踪产品，具备接口丰富、性能稳定的特点。该设备支持GNSS（全球导航卫星系统，可以理解为：提供高精度定位和授时服务的卫星系统）精准定位，可实现车辆与驾驶人员状态追踪监控、历史轨迹记录、电子围栏及异常报警等功能。设备提供支持LTE CatM1、Cat1、Cat4等多种速率的无线网络接入子型号。搭配车联网云平台，可实现远程车辆管理、资产追踪及预防式维护，有助于车队运营者降低成本、提升运营效率。

## 1.4 指示灯说明

设备出厂状态下处于运输模式，需外接电源或通过车辆诊断接口供电激活VT310后，方可通过状态灯判断设备运行情况。

| 指示灯 | 状态 | 含义 |
|--------|------|------|
| GNSS | 长灭 | 设备未启动或GNSS功能关闭 |
|  | 慢闪（频率1Hz） | GNSS功能开启 |
|  | 快闪（频率0.5Hz） | GNSS授时成功 |
|  | 长亮 | 定位成功 |
| Cellular（蜂窝网络，可以理解为：通过运营商基站提供无线数据连接的移动通信网络） | 长灭 | 设备未启动或拨号功能关闭 |
|  | 慢闪（频率1Hz） | 拨号功能开启 |
|  | 快闪（频率0.5Hz） | 拨号成功 |

**操作步骤**：

1. 找到设备上的 Reset 按键。
2. 使用螺丝刀或其他工具按住 Reset 按键，持续超过 8 秒。
3. 松开按键，设备将恢复默认账号密码。

# 第2章 安装与首次使用

## 2.1 安装前准备

在安装 VT310 设备之前，需准备以下物品和工具：

- 已激活的 SIM 卡（用于蜂窝拨号上网）
- 匹配的线缆（根据应用场景选择，详见 [2.3 线缆连接](#23-线缆连接)）
- 安装螺栓（用于将设备固定在车辆上）
- 9-48V 直流电源适配器（办公室测试场景）
- RS232 转 USB 线缆（用于串口调试和登录设备）

> **注意**：安装前需确认供电电压在 9-48V 直流范围内，超出范围可能导致设备损坏。安装位置应便于接收 GPS 信号，推荐安装在车辆前挡风玻璃下方。

> **注意**：连接线缆时需区分电源正极（V+）与负极（V-），接反可能造成设备损坏。

### 2.2.4 连接 I/O 接口

I/O 接口集成在 26PIN 车载接口中，提供以下资源：

- 3 路 DO（Digital Output，数字输出，可以理解为：控制外部设备开关状态的输出），最大电流 300mA
- 4 路 DI（Digital Input，数字输入，可以理解为：检测开关状态的传感器）
- 1 路 AI（Analog Input，模拟输入，可以理解为：测量连续变化的模拟量信号）
- 1 路 1-Wire
- 1 路 RS232 串口
- 1 路点火信号

**RS232 串口**

RS232 串口用于调试。使用前需将 VT310 的 RS232_RX、RS232_TX、GND 分别与 DB-9 串口免焊头的 TXD、RXD、GND 相连，再通过 RS232 转 USB 线与 DB-9 串口免焊头连接。

# 第3章 常用场景配置

## 场景1：蜂窝网络配置

**目标**：通过配置蜂窝网络参数，使设备能够接入移动运营商的网络并连接互联网。

**前提**：已完成SIM卡安装并登录设备Web管理页面。

**预计用时**：约5分钟。

**操作步骤**：

1. 在导航菜单中点击 **蜂窝网络**，进入蜂窝网络配置页面。

2. 根据运营商提供的信息，填写以下基本参数：

   | 参数 | 参数说明 |
   | :--- | :--- |
   | 网络接入点名称（APN） | APN（Access Point Name，可以理解为：运营商网络的"门牌号"）。专网接入移动网络时需要运营商提供此参数。大部分公网业务的SIM卡拨号时不对APN进行认证。 |
   | 网络拨号用户名 | 默认参数"gprs"，专网AAA认证时，移动网络运营商需要提供此参数。 |
   | 网络拨号密码 | 默认参数"gprs"，专网AAA认证时，移动网络运营商需要提供此参数。 |
   | 认证模式 | 自动/CHAP/PAP，专网AAA认证时，运营商需要提供此参数。**自动**：轮流使用 PAP 和 CHAP 认证方式来拨号（上电第一次使用 PAP 认证，如果拨号失败，再次拨号时使用 CHAP 认证，再下一次拨号又使用 PAP 认证，以此类推）。如果认证模式选择 **PAP** 或 **CHAP**，则仅使用对应的认证方式进行拨号。 |

3. 如果使用场景有特殊需求，可点击 **显示高级选项** 展开隐藏配置项，根据运营商提供的信息填写以下高级参数：

   | 参数 | 参数说明 |
   | :--- | :--- |
   | 网络拨号号码 | 默认参数"\*99\*\*\*1#"，移动网络运营商需要提供此参数。 |
   | PIN | PIN（Personal Identification Number）是指SIM卡的个人识别密码。当SIM卡开启PIN验证时，需填入对应的PIN码。移动网络运营商需要提供此参数。 |
   | 默认承载APN | 运营商提供此参数。 |

   > **注意**：默认承载设置是针对某些运营商的网络数据传输特殊情况设置的功能，一般情况下无需配置。如需配置，请向运营商咨询默认承载的相关参数。

   <p align="center"><img src="./img/LasY-pyLehrIerMK/1681197223242-1ce5ae9e-e286-4c1d-a89b-39e44efdd02a-943174.webp" alt="蜂窝网络配置页面"></p>
   <p align="center"><strong>图 3-1 蜂窝网络配置页面</strong></p>

4. 参数填写完成后，点击 **保存配置**。设备重启后，配置生效。

| J1939 | CAN2 接口协议，对应物理层PIN CAN_2L(PIN 12)和CAN_2H(PIN25) |  |
| J1939 | CAN2 接口协议，对应物理层PIN CAN_2L(PIN 12)和CAN_2H(PIN25) |  |
| J1708 | J1708 接口协议，对应物理层PIN J1708_B（PIN13）和J1708_A (PIN 26) | |

### 4.2.1 休眠模式

休眠模式用于在车辆熄火后维持设备续航能力，为特殊应用环境提供持续运行保障。其状态机如下所示：

<p align="center"><img src="./img/LasY-pyLehrIerMK/1681197221579-b4cbfbf8-7105-4980-83d2-568ec6168e1f-750651.webp" alt="休眠模式状态机"></p>
<p align="center"><strong>图 4-5 休眠模式状态机</strong></p>

**状态机说明**

Run、Sleep、Temp run 分别代表正常运行状态、休眠状态和临时运行状态。

- **状态转换 1（Run → Temp run）**：当设备没有外接电源（电源电压小于休眠电压）或 IGT 为 OFF（需在配置中使能 IGT）时，设备从 Run 进入 Temp run。首次进入休眠时，设备默认持续 10 秒的延时休眠时间（用于上报信息），随后进入 Sleep。
- **状态转换 2（Temp run → Sleep）**：设备在 Temp run 状态下运行满一个唤醒运行时间周期后，或从 Run 状态运行 15 秒进入 Temp run 后，将进入 Sleep。
- **状态转换 3（Sleep → Run）**：当外接电源恢复（电源电压大于休眠电压）且 IGT 为 ON（需在配置中使能 IGT）时，设备从 Sleep 恢复至 Run。
- **状态转换 4（Sleep → Temp run）**：设备在 Sleep 状态下运行满一个唤醒间隔周期后，将自动进入 Temp run。
- **状态转换 5（Temp run → Run）**：当外接电源恢复（电源电压大于休眠电压）且 IGT 为 ON（需在配置中使能 IGT）时，设备从 Temp run 恢复至 Run。

**配置休眠模式**

| 参数 | 参数说明 |
| :--- | :--- |
| 使能IGT | 使能 IGT 后，设备将 IGT 状态作为进入和退出休眠的条件，默认不勾选 |
| 唤醒间隔 | 进入休眠后，设备自动唤醒的时间间隔，默认 120 分钟 |
| 唤醒运行时间 | 休眠唤醒后，设备进入下一次休眠的时间间隔，默认 5 分钟 |
| 延时休眠时间 | 进入休眠后，用于上报数据的等待时间 |
