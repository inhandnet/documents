# 白鹰能源管家用户手册-V1.1

## 简介

全球各地的能源供应面临着多种挑战。与此同时，随着气候变化和环境污染问题日益严峻，节能降碳已成为全球普遍共识的目标。为了有效地应对这些挑战，迫切需要寻求高效的节能降碳解决方案。

白鹰能源管家专为光伏储能管理、配电房监控、生产能耗监测和办公空间节能等关键领域设计。提供客制化的关键指标看板、高效的节能策略设置、直观的WEB组态界面、深度的能效分析、全面的运维管理以及灵活的权限管理。搭配映翰通能源网关等智能硬件，帮助企业快速实现能耗监控和分析，助力节能减排，确保设备安全可靠运行。

## 名词解释

- **租户**

租户即一个独立的企业，每个租户之间的设备、用户等数据完全隔离。

- **组织**

组织通常代表企业中的子公司、部门等单位，比如长沙子公司或生产部门。通过定义不同的组织，可以区分每个组织具备的监控设备等数据权限，实现不同组织和用户的数据权限划分。

- **站点组织**

站点组织是一种特殊的组织类型，除了具备一般组织的数据权限划分功能外，还额外具备站点属性。例如，站点组织具备站点大屏功能，支持配置站点电价等操作。

- **站点**

一个站点组织对应一个站点，通常表示一个工厂，一栋建筑或其他能源管理单元。

- **通信设备**

通信设备是指实现数据采集、数据处理与数据上云的通信网关设备，用于连接监控设备与白鹰平台。

- **监控设备**

监控设备是需要在平台中监控的各类设备，例如空调、电表、照明设备等。

- **变量**

变量是监控设备中需要监控的具体数据，例如电压、电流、温度、湿度等。

- **变量类型**

变量类型指的是变量的分类，平台内置了各种变量类型，如电压、电流、功率因数等，用于统计分析和数据管理，需要准确配置。

- **设备类型**

设备类型指监控设备的分类，例如变压器、水表、电表等。

- **管理维度**

管理维度是指用于能效管理的各种维度，例如区域、部门、楼层等。

- **节点**

节点是指管理维度中的每个具体单位，例如区域管理维度中的生产区就是一个节点。

- **全局角色**

全局角色是指所有组织均可以查看和分配的角色。

- **组织角色**

全局角色是指特定组织才能查看和分配的角色。

- **计费线路**

计费线路用于设置哪些监控设备的用能数据作为站点的用能统计数据，用于计算站点每月的电费。

- **分时电价**

分时电价即按高峰用电和低谷用电分别计算电费的一种电价制度。

- **不分时电价**

不分时电价即不区分高峰和低谷用电，电费按统一标准计算的电价制度。

- **DeviceSupervisor Agent**

DeviceSupervisor Agent简称DSA，是运行在通信设备中实现数据采集、处理和上云的数据采集软件。

- **控制器**

控制器为被采集数据的设备，如电表、PLC、逆变器等。

- **测点**

测点即设备的业务数据，如A相电流、A相电压等。

## 快速入门

以下以EC322为例，介绍如何配置网关联网、采集电表数据并上传至云端，最终实现云端监控电表数据。详细的DSA配置操作，请参考[DSA用户手册](https://help.inhand.com.cn/portal/zh/kb/articles/dsa%E4%B8%AD%E6%96%87)。

### 1. 设备联网

要实现云端监控，必须确保设备已经联网。不同产品系列的配置方式存在差异，请参考同系列产品进行配置。各个设备因固件版本不同，界面可能存在差异，请以实际设备为准。

- 方式一：以太网联网

环境：用网线将PC与EC322的ETH1（默认192.168.3.100）接口连接，ETH2（默认192.168.4.100）接口连接Internet。

首先为PC配置一个192.168.3.X网段的IP地址（如192.168.3.110），掩码配置为255.255.255.0。然后在谷歌浏览器中输入[https://192.168.3.100:9100,](https://192.168.3.100:9100,)访问页面如下：

![1717739348372-d3aeeb8d-9b35-4ea1-9934-3cf61b14b3cd.png](./img/RPTlnNabsvN7n1k6/1717739348372-d3aeeb8d-9b35-4ea1-9934-3cf61b14b3cd-052034.png)

进入“系统 >> 网络管理>>接口”页面，为ETH2配置DHCP或静态IP。

![1717739348604-a823b214-6322-4760-868f-275683708065.png](./img/RPTlnNabsvN7n1k6/1717739348604-a823b214-6322-4760-868f-275683708065-924807.png)

配置完成后进入“系统 >> 网络管理>>静态路由”，添加相应的静态路由完成联网。

![1717739348864-b2226af7-f476-420d-a940-cd5d05a5db40.png](./img/RPTlnNabsvN7n1k6/1717739348864-b2226af7-f476-420d-a940-cd5d05a5db40-935781.png)

- 方式二：蜂窝联网

一般情况下，SIM卡插卡即可联网。如果您有定制需求，可在“系统 >> 网络管理>>蜂窝网络”页面配置APN等参数，实现联网。

![1717739349089-41f04c2e-6be5-4dac-9997-ead349106e68.png](./img/RPTlnNabsvN7n1k6/1717739349089-41f04c2e-6be5-4dac-9997-ead349106e68-256457.png)

- 方式三：Wi-Fi（STA）联网

在“系统 >> 网络管理 >> WiFi”菜单下点击“启用WiFi”，配置相关参数，使其连接到现场的SSID实现联网。

![1717739349290-fd548a9e-a5d2-4ae0-87eb-a2fb1eb9c080.png](./img/RPTlnNabsvN7n1k6/1717739349290-fd548a9e-a5d2-4ae0-87eb-a2fb1eb9c080-000293.png)

### 2. 数采上云配置

### 2.1 管理DSA

EC322出厂默认安装了DSA。在“系统 >> 插件”页面，可以点击“添加”按钮并选择新的DSA版本文件以更新DSA版本（版本在官网的资源中心获取）。点击“暂停”或“启动”按钮以启动或暂停DSA。

![1717739349488-3a939a99-675a-48b7-a3a6-01406db325cf.png](./img/RPTlnNabsvN7n1k6/1717739349488-3a939a99-675a-48b7-a3a6-01406db325cf-043107.png)

### 2.2 数据采集配置

进入“设备监控 >> 测点监控 >> 监控列表”页面。在该界面可配置和监控控制器以及测点状态。

点击控制器列表的“添加”按钮，选择通信协议并配置相应的通信参数。

![1717739349702-ee35203d-85f5-4b2f-b59b-25ca6f0f2856.png](./img/RPTlnNabsvN7n1k6/1717739349702-ee35203d-85f5-4b2f-b59b-25ca6f0f2856-663205.png)

控制器添加完成后，点击测点列表的“添加”按钮，配置相应的采集信息。

![1717739349892-06367582-1ba1-4546-bc17-6ebadb9e95d3.png](./img/RPTlnNabsvN7n1k6/1717739349892-06367582-1ba1-4546-bc17-6ebadb9e95d3-820349.png)

配置完成后可在该页面监控控制器和测点的状态。

![1717739350149-9245030f-f6ff-43c6-b263-4231d172f85c.png](./img/RPTlnNabsvN7n1k6/1717739350149-9245030f-f6ff-43c6-b263-4231d172f85c-825244.png)

### 2.3 数据上云配置

进入“设备监控 >> 云服务”页面。在该界面可配置测点数据传输到白鹰能源管家。

启用云服务后，选择云平台类型为“白鹰能源管家”，选择服务器地址并点击“提交”即可。成功连接平台后，云服务状态为“连接成功”。

![1717739350388-f93de605-87b3-4bac-9e45-c30deab5f9c0.png](./img/RPTlnNabsvN7n1k6/1717739350388-f93de605-87b3-4bac-9e45-c30deab5f9c0-362562.png)

### 3.1 添加通信设备

进入白鹰能源管家的“系统管理 >> 设备管理 >> 监控设备管理”页面。点击“添加”按钮，配置并添加一台监控设备。

点击“新增”按钮，添加需要在云平台中监控的变量。

### 站点大屏

站点大屏页面支持直观展示的站点状态信息。展示的信息如下：

- 站点信息：站点组织的“站点信息”页面中配置的电压等级、变压器装机容量和供电线路
- 设备统计：统计站点组织下的通信设备、监控设备和变量点
- 用能趋势：统计站点下水电气的年月日维度的用能趋势
- 场站图：站点组织的“站点信息”页面中配置的场站图像。推荐图片比例为3:2
- 负荷趋势：展示站点的实时负荷和负荷趋势。实时负荷为站点组织的“站点信息”页面中用电功率关联的变量值。负荷趋势每小时1条数据，展示0~24小时今日与昨日的负荷趋势曲线
- 能耗统计：统计站点的本月和本年总能耗
- 用能统计：统计站点下水电气的年月日维度的用能环比信息
- 用电排名：统计站点下用电排名TOP3的能源节点（不统计根能源节点）

### 安全运行

### 设备监控

设备监控页面提供单个监控设备详细的状态展示和查询。在“系统管理 >> 监控设备管理”添加设备后，用户可在此页面查看监控设备的实时状态、历史趋势和告警记录等信息。在左侧的设备列表页面可通过组织、标签和设备名称筛选过滤需要查看的监控设备。

在概览页面，用户可以快速掌握监控设备的运行情况。包括监控设备的实时运行状态和历史运行趋势，还可以查看该监控设备的用能和尖峰平谷用电占比情况。

#### 空调策略管理

空调策略管理页面支持管理空调的节能策略模板和设置空调的节能策略，实现空调节能。

在空调策略管理页面中，点击“空调策略模板”按钮进入空调策略模板管理页面。点击“新增”按钮可增加一个空调策略模板。一个模板支持多种模式，比如冬季模式和夏季模式。具体参数说明如下：

- 模板名称：空调策略模板的名称
- 模式名称：模板中单个模式的名称
- 排除时间：不执行策略的时间，支持以下选项：
  - 不排除：一直执行
  - 周末：周末时不执行
  - 指定时间：用户自行指定哪些日期不执行策略
- 控制配置：勾选启用指定节能模式
  - 环境温度：可设置环境温度大于等于或者小于等于某个温度值时才能开机
  - 关机条件
    - 无人之后：检测到无人后一定时间内，自动将空调设定为关机状态。当空调配置了变量类型为空调→人体微动的变量，并且所有人体微动变量的数值都维持在0（即表示无人），并保持该状态一段指定的时间后，将自动触发空调关机。
    - 开窗之后：检测到开窗后一定时间内，自动将空调设定为关机状态。当空调配置了变量类型为空调→窗户状态的变量，并且任一窗户状态变量的数值都维持在1（即表示窗户打开），并保持该状态一段指定的时间后，将自动触发空调关机。
    - 定时关机：可以指定一定时间段内空调处于不可开机。如果设置了多个时间段，则多个时间段内均不可开机。
  - 设定温度：可以设置空调的最低和最高设定温度，当设定温度超过或低于最低和最高设定温度时，会修改为最低或最高设定温度。

![1717739359598-0054a9e3-fc9f-44b0-bf22-735a338bb2ca.png](./img/RPTlnNabsvN7n1k6/1717739359598-0054a9e3-fc9f-44b0-bf22-735a338bb2ca-817728.png)

在空调策略管理页面中，左侧可勾选组织筛选展示选中组织对应的空调节能策略，右侧可编辑和删除空调节能策略。点击“新增”按钮可新增空调节能策略，选择相应的空调和节能策略并提交即可按照设定的策略自动控制空调的运行状态。

  - 所选组织的所有变量：对多个组织下的所有变量均执行一键校值操作
  - 所选组织的所有用能变量：对多个组织下的所有用能变量（即变量类型为正向有功电度和反向有功电度的变量，以及在站点组织的“费率管理 >> 其他能源单价”页面中配置的变量类型对应的变量）均执行一键校值操作
  - 所选变量：对当前选择的变量执行一键校值操作
- 选择组织：选择需要批量校值的组织项
- 校值方式
  - 按前后有效值等差分布：当某个时间点的变量没有数值时，根据前一个和后一个有值的时间点的数值差，将这个差值均匀分布到没有数值的时间点上，以此来计算该时间点的数据值。
  - 与上一个有效值一致：当某个时间点的变量没有数值时，以前一个有值的时间点的数值作为该时间点的数据值。

#### 监控设备管理

监控设备管理页面支持管理平台中的监控设备（比如空调、电表和变压器等设备）。

点击“新增”按钮，弹窗中的参数如下：

- 名称：设备名称
- 类型：设备类型
- 标签：自定义设备标签，可用于分类和快速筛选设备
- 编码：设备的唯一标识，为空时系统自动生成
- 所属组织：设备所属的组织名称
- 额定负荷：该监控设备的额定负荷，用于计算分析设备的负荷率
- 设备能力：勾选以对该设备进行电能质量分析
  - 额定电压：设备的额定电压，用于评估电能质量是否合格
  - 额定电流：设备的额定电压，用于评估电能质量是否合格
  - 电流负载率门限：电流负载率门限，用于电能质量是否合格。为空时则表示电流负载率一定合格

![1717739376283-a3fb582e-c998-4a1e-b393-037cc051d44d.png](./img/RPTlnNabsvN7n1k6/1717739376283-a3fb582e-c998-4a1e-b393-037cc051d44d-868340.png)

复制设备时，弹窗中的参数如下：

- 名称：设备名称
- 类型：设备类型
- 标签：自定义设备标签，可用于分类和快速筛选设备
- 编码：设备的唯一标识，为空时系统自动生成
- 所属组织：设备所属的组织名称
- 测点来源：定义监控设备下的变量所关联的测点是来自哪个通信设备下的哪个控制器的测点。在选择时，需要从某个通信设备的DSA中配置的指定控制器获取。系统会根据原始变量关联的测点名称，在选定的控制器下寻找是否存在同名的测点。如果找不到匹配的测点，则该变量将不会与任何测点关联。

##### 变量管理

在“监控设备管理”页面点击相应监控设备的“变量管理”进入该监控设备的“变量管理”页面，在此页面可管理该监控设备的变量信息。

![1717739377878-29198a77-1d07-4b4e-87d9-84b31c4d8d65.png](./img/RPTlnNabsvN7n1k6/1717739377878-29198a77-1d07-4b4e-87d9-84b31c4d8d65-578353.png)

点击“新增”可新增变量，弹窗中的参数如下：

- 变量名称：变量的名称
- 变量类型：变量的类型，系统内置的变量类型涉及设备的用能分析和展示，请准确填写
- 变量单位：变量单位可根据需求自行修改。部分变量类型的变量涉及计算，无法修改单位
- 变量来源：变量可以直接来源于通信设备，也可以是一个由通信设备的数据计算后的虚拟变量。配置虚拟变量时，该监控设备的所属组织需要具备计算虚拟变量涉及的所有变量对应的监控设备的访问权限
- 通信设备：变量所关联的测点的所属网关
- 关联测点：变量关联的测点。关联后，测点的数值即变量的数值
- 上报周期：测点的上报周期
- 基值：变量的基础值，常用于电表超量程后从0重新计数，可为变量配置电表的量程作为该变量的基值。后续每次上报数据时将自动加上基值
- 依赖变量值无效：仅适用于虚拟变量。当计算虚拟变量所需要的变量数据无效（例如通信设备离线）时如何计算
- 数据类型：仅适用于虚拟变量。设置计算后的数据值为模拟量还是状态量
- 存盘周期：变量的存盘周期
- 连续添加：勾选后将在添加完成后再次打开“新增”弹窗，同时弹窗中的通信设备、上报周期和存盘周期将与上次的配置保持一致

![1717739378194-408fe2b5-7b44-4545-8605-c9d7037f1250.png](./img/RPTlnNabsvN7n1k6/1717739378194-408fe2b5-7b44-4545-8605-c9d7037f1250-155547.png)

修改监控设备的变量配置后，需要“应用”后才能生效，点击“应用”后可以按照最新的配置监控数据。

![1717739378442-b3701ee8-a25f-498a-a60e-708b7bbf731d.png](./img/RPTlnNabsvN7n1k6/1717739378442-b3701ee8-a25f-498a-a60e-708b7bbf731d-451188.png)

### 电价管理

电价管理页面支持配置电价信息，以便于站点在定义电价时可以直接选择使用。点击“新增”按钮，弹窗中的参数如下：

- 名称：费率名称
- 电价：支持配置多条电价方案，比如1~6月的电价和7~12月的电价不一样
  - 有效月份：该条电价配置生效的月份，可以选择多个
  - 是否分时：选择电价是分时电价还是不分时电价
  - 单价：分时电价时，需要配置尖峰平谷用电的电价；非分时用电时需要配置用电单价
  - 设置时间段：电价为分时电价，需要配置尖峰平谷用电对应的时间段

添加后的费率配置可在站点组织的费率管理中选择使用。在计算费用时，将采用选择费率配置时的设置，而不是最新的费率配置。即如果费率配置有更新，站点组织的费率配置不会自动更新。你需手动点击“同步配置到站点”，以将最新的费率配置更新至站点组织。
